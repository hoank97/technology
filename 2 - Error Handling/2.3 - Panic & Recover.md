## When to Panic

### Dùng được khi:

1. **Programmer error** — violated invariants
   ```go
   if index < 0 || index >= len(slice) {
       panic("index out of bounds") // Bug nghiêm trọng
   }
   ```

2. **Init failure** — không thể tiếp tục
   ```go
   if os.Getenv("API_KEY") == "" {
       panic("API_KEY must be set") // App không thể chạy
   }
   ```

3. **Truly unrecoverable** — memory corruption, stack overflow

### Không dùng khi:

- Expected errors (user input, network)
- Library code

## Recover

`recover()` **bắt panic** và ngăn crash:

```go
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered:", r)
        }
    }()
    panic("oops!")
}
```

### Chỉ hoạt động trong defer

```go
// ❌ Không work — panic xảy ra trước
recover()
panic("oops!")
```

### Recover in goroutine

```go
go func() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered in goroutine:", r)
        }
    }()
    panic("oops")
}()
```

---

## HTTP Server Pattern

Nơi recover hợp lý:

```go
func handleRequest(w http.ResponseWriter, r *http.Request) {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("Panic recovered: %v", r)
            http.Error(w, "Internal Server Error", 500)
        }
    }()
    // handler code
}
```

---

## ⚠️ Cautions

1. **Không dùng recover thay error handling**
2. **State sau panic có thể corrupted**
3. **Chỉ recover có chủ đích** — trong framework/server
