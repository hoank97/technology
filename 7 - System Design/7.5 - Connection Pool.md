## Solution: Connection Pool

A **set of connections already opened** and reusable:

```
With connection pool:
Request 1 → Get from pool (0ms) → Query → Return to pool
Request 2 → Get from pool (0ms) → Query → Return to pool
Request 3 → Get from pool (0ms) → Query → Return to pool
```

---

## How it works

```go
import "github.com/jmoiron/sqlx"

db, _ := sqlx.Connect("postgres", "user=postgres dbname=myapp sslmode=disable")

// Configure pool
db.SetMaxOpenConns(25)              // Max connections
db.SetMaxIdleConns(5)                // Keep idle connections
db.SetConnMaxLifetime(time.Hour)     // Recycle after 1 hour

// Application code
rows, _ := db.Queryx("SELECT * FROM users WHERE id = $1", userID)
for rows.Next() {
    var u User
    rows.StructScan(&u)
}
```

---

## Pool Settings

| Setting | Mô tả |
|---------|--------|
| **MaxOpenConns** | Tổng số connections tối đa |
| **MaxIdleConns** | Số connections giữ idle |
| **ConnMaxLifetime** | Recycle sau X time |
| **ConnMaxIdleTime** | Close idle connection sau X time |

### Sizing pool

```
Formula:
pool_size = (number_of_cores × 2) + effective_spindle_count

Ví dụ: 4 cores + SSD (no spindle limit)
→ pool_size = (4 × 2) + 0 = 8
```

## Problems Without Connection Pool

| Problem | Without Pool | With Pool |
|---------|-------------|-----------|
| Connection overhead | ~10-20ms mỗi request | ~0ms (reused) |
| DB running out of connections | ✅ Dễ xảy ra | ❌ Tránh được |
| Throughput | Thấp | Cao |
