## NoSQL

### Đặc điểm

| | |
|---|---|
| **Schema** | Flexible hoặc schema-less |
| **Language** | Varied (key-value, document, etc.) |
| **Relationships** | Denormalized, embedded |
| **BASE** | Eventual consistency |
| **Scaling** | Horizontal (tự nhiên) |

### Types of NoSQL

| Type | Description | Examples |
|------|-------------|----------|
| **Document** | JSON-like documents | MongoDB, CouchDB |
| **Key-Value** | Simple key-value pairs | Redis, DynamoDB |
| **Column** | Wide columns | Cassandra, HBase |
| **Graph** | Nodes và edges | Neo4j |

### Use cases

- **Flexible schema** — data không cố định
- **Easy horizontal scaling** — write throughput cao
- **High write throughput** — log, metrics, IoT
- **Examples**: MongoDB, Cassandra, DynamoDB, Redis

---

## So sánh

| | SQL | NoSQL |
|---|---|---|
| Schema | Fixed | Flexible |
| JOINs | ✅ Native | ❌ Manual denormalization |
| ACID | ✅ Full | BASE (eventual) |
| Scaling | Vertical | Horizontal |
| Query | Declarative (SQL) | Varied |
| Consistency | Strong | Eventual |

---

## Decision Guide

```
Chọn SQL khi:
├── Structured data có relationships rõ ràng
├── Cần ACID transactions
└── Complex queries + JOINs

Chọn NoSQL khi:
├── Schema không cố định
├── Cần horizontal scaling
├── Write throughput rất cao
└── Eventual consistency chấp nhận được
```
