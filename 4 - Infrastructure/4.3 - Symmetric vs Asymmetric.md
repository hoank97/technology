## Asymmetric Encryption

**Public key để encrypt**, **private key để decrypt**.

```
┌──────────┐                    ┌──────────┐
│  Sender   │                    │ Receiver │
│          │                    │          │
│ Encrypt  │  ──── data ────→  │ Decrypt  │
│ w/ public │                    │ w/ priv  │
└──────────┘                    └──────────┘
```

### Ví dụ

- **RSA**: encrypt, digital signature
- **ECC** (Elliptic Curve Cryptography)

| Ưu điểm | Nhược điểm |
|---------|-----------|
| Không cần share private key | Chậm hơn symmetric |
| Dùng cho key exchange | |

---

## So sánh

| | Symmetric | Asymmetric |
|---|---|---|
| Keys | 1 shared key | 2 keys (public + private) |
| Speed | Nhanh | Chậm (~100-1000x) |
| Use case | Bulk data encryption | Key exchange, signatures |
| Algorithm | AES, ChaCha20 | RSA, ECC |
| Key exchange | Cần secure channel | Public key để share |

---

## Hybrid (thực tế dùng)

**Dùng asymmetric để exchange key → symmetric để encrypt data:**

```
1. Client lấy server's public key
2. Client encrypt random session key bằng public key → gửi server
3. Server decrypt bằng private key → lấy session key
4. Cả 2 dùng session key (symmetric) để encrypt communication
```

→ Kết hợp ưu điểm của cả 2: secure key exchange + fast encryption
