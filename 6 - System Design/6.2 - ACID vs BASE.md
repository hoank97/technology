## BASE

**NoSQL / distributed database model** — Availability + Eventual consistency.

### Properties

| | Mô tả |
|---|---|
| **Basically Available** | System đảm bảo available |
| **Soft state** | State có thể thay đổi theo thời gian |
| **Eventually Consistent** | System sẽ consistent sau một thời gian |

### Soft state vs Hard state

```
ACID:  Strong consistency ←→ ACID is "hard state"
BASE:  Eventually consistent ←→ BASE is "soft state"
```

→ NoSQL **không đảm bảo** data consistent ngay lập tức

---

## So sánh

| | ACID | BASE |
|---|---|---|
| Consistency | Immediate/Strong | Eventual |
| Availability | May sacrifice | Prioritized |
| Transactions | Full ACID | Limited |
| Use case | Financial, critical data | High scale, flexible |
| Examples | PostgreSQL, MySQL | MongoDB, Cassandra |

---

## CAP Theorem Recap

```
ACID ←→ Strong Consistency
BASE ←→ Eventual Consistency

ACID databases → CP systems (CP trong CAP)
BASE databases → AP systems (AP trong CAP)
```
