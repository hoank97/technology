## Approaches

### 1. Choreography (Event-driven, Decentralized)

```
Order Service ──→ Payment Service ──→ Inventory Service
      ↑                   │
      └───────────────────┘
         (events trigger next steps)
```

**Tốt cho**: Simple workflows, few services
**Xấu cho**: Complex coordination, many services

### 2. Orchestration (Centralized Coordinator)

```
Saga Orchestrator
    │
    ├── Step 1: CreateOrder
    ├── Step 2: ReserveInventory   (rollback: ReleaseInventory)
    ├── Step 3: ChargePayment      (rollback: RefundPayment)
    └── Step 4: ConfirmOrder
```

**Tốt cho**: Complex workflows, many services
**Xấu cho**: Simple workflows (overkill)

## Outbox Pattern

Tránh **lost event** khi DB write success nhưng MQ publish fail.

```
Without Outbox:
1. DB write ✓ → 2. MQ publish ✗ → Event LOST!

With Outbox:
1. DB write + outbox record ✓ (atomic, cùng transaction)
2. Outbox poller đọc → 3. MQ publish ✓ → 4. Mark sent ✓
```

→ **At-least-once delivery**, không mất event.
