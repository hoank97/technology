## Impact

| Users | Without N+1 | With N+1 (N=1) |
|-------|-----------|----------------|
| 10 | 1 query | 11 queries |
| 100 | 1 query | 101 queries |
| 10,000 | 1 query | 10,001 queries |

→ **Performance degrades linear** theo data size

---

## Solutions

### 1. JOIN (SQL)

```sql
-- 1 query thay vì N+1
SELECT u.*, o.* FROM users u
LEFT JOIN orders o ON u.id = o.user_id
```

→ Tốt cho 1:N relationships

### 2. WHERE IN (Batch)

```go
// Query 1: Lấy users
users, _ := db.Query("SELECT * FROM users")

// Query 2: Batch lấy orders cho tất cả users
userIDs := extractIDs(users)
orders, _ := db.Query("SELECT * FROM orders WHERE user_id IN (?)", userIDs)
```

→ 2 queries bất kể bao nhiêu users

### 3. Eager Loading (ORM)

```go
// GORM
var users []User
db.Preload("Orders").Find(&users)
// → 2 queries: SELECT users + SELECT orders WHERE user_id IN (...)
```
