### 2. Sharding

**Data is split based on shard key.**

```
Users 1-1000    вҶ’ Shard 1
Users 1001-2000 вҶ’ Shard 2
Users 2001-3000 вҶ’ Shard 3
```

| | |
|---|---|
| вң… | Both read and write scale |
| вң… | Can scale infinitely |
| вқҢ | Cross-shard queries complex, slow |
| вқҢ | Shard key choice is critical |
| вқҢ | Rebalancing when adding shards is hard |

### Sharding Strategies

```go
// Hash-based sharding
shardID := hash(userID) % numShards

// Range-based sharding

// Geographic sharding

```
