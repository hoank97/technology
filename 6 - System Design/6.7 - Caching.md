## Cache Patterns

### 1. Cache Aside (Lazy Loading)

```go
// Application reads from cache first
val, err := redis.Get("user:123")
if err == nil {
    return unmarshal(val)
}

// Cache miss → read from DB
user, err := db.GetUser("123")

// Write back to cache
redis.Set("user:123", marshal(user), 5*time.Minute)
return user
```

→ **Pro**: Cache chỉ có data thực sự cần
→ **Con**: Cache miss = 2 round trips

### 2. Write Through

```go
// Write → DB + Cache đồng thời
db.SaveUser(user)
redis.Set("user:123", marshal(user), 5*time.Minute)
```

→ **Pro**: Cache luôn consistent với DB
→ **Con**: Write chậm hơn

### 3. Read Through

```go
// Chỉ đọc từ cache, cache tự load từ DB nếu miss
val := cache.Get("user:123") // tự động load nếu miss
```

### 4. Write Aside

```go
// Write → DB, cache được invalidates sau
db.SaveUser(user)
redis.Del("user:123") // không write to cache
```

→ **Pro**: Write nhanh, đơn giản
→ **Con**: Cache miss sau write

## Cache Problems

### Thundering Herd

**Nhiều requests cùng hit cache miss → tất cả query DB cùng lúc.**

```
Without lock:
Request 1-1000 ──→ Cache miss ──→ 1000 DB queries cùng lúc!
```

**Solution: Distributed lock hoặc probabilistic early expiration**

```go
// Probabilistic Early Expiration
val := cache.Get("user:123")
if val != nil && shouldRefresh(val) {
    go func() {
        cache.Set("user:123", db.GetUser("123"))
    }()
}
```

### Cache Warming

**Điền cache trước khi production** để tránh cold cache:

```bash
# Chạy script warm cache trước khi deploy
./scripts/warm-cache.sh
```

## TTL (Time To Live)

```
Short TTL:   1-30s    → Fresh data, high DB load
Medium TTL:  1-30min   → Balance
Long TTL:    1-24h     → Static data, low freshness needed
```
