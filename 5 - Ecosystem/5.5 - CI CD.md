# 5.5 CI/CD

## Overview

CI/CD = **Continuous Integration** + **Continuous Delivery/Deployment** — pipeline tự động hóa việc build, test, và deploy code.

```
Code Commit → Build → Test → (Staging) → Deploy to Production
     ↑______________CI______________|_______CD______________|
```

### CI (Continuous Integration)

- Developer push code lên shared repo
- Pipeline tự động trigger: build + run tests
- Mục tiêu: phát hiện lỗi sớm, không merge broken code

### CD (Continuous Delivery)

- Sau CI pass → tự động deploy lên staging/pre-prod
- Cần manual approval trước khi vào production

### CD (Continuous Deployment)

- Không cần manual approval
- Mỗi commit pass CI → tự động deploy lên production
- Yêu cầu: test coverage cao, rollback strategy rõ ràng

## GitHub Actions

### Basic Structure

```yaml
name: CI Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: '1.22'

      - name: Cache dependencies
        uses: actions/cache@v4
        with:
          path: ~/go/pkg/mod
          key: ${{ runner.os }}-go-${{ hashFiles('**/go.sum') }}
          restore-keys: |
            ${{ runner.os }}-go-

      - name: Build
        run: go build -v ./...

      - name: Test
        run: go test -v -race -coverprofile=coverage.out ./...

      - name: Upload coverage
        uses: actions/upload-artifact@v4
        with:
          name: coverage
          path: coverage.out
```

### Go Specific Pipeline

```yaml
name: Go CI

on:
  push:
    branches: [main, release/**]
  pull_request:
    branches: [main]

env:
  GO_VERSION: '1.22'
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  # ─── 1. Lint & Vet ─────────────────────────────────────────────
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-go@v5
        with:
          go-version: ${{ env.GO_VERSION }}
          cache: false  # Use golangci-lint own caching

      - name: Run golangci-lint
        uses: golangci/golangci-lint-action@v6
        with:
          version: latest
          args: --timeout=5m

  # ─── 2. Test & Coverage ────────────────────────────────────────
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-go@v5
        with:
          go-version: ${{ env.GO_VERSION }}

      - name: Test
        run: go test -v -race -covermode=atomic -coverprofile=coverage.out ./...

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v4
        with:
          files: ./coverage.out
          fail_ci_if_error: true

  # ─── 3. Build Docker Image ──────────────────────────────────────
  build:
    runs-on: ubuntu-latest
    needs: [lint, test]
    permissions:
      contents: read
      packages: write

    steps:
      - uses: actions/checkout@v4

      - name: Meta
        id: meta
        run: |
          VERSION=${GITHUB_REF#refs/tags/v}
          [ -z "$VERSION" ] && VERSION=latest
          echo "version=$VERSION" >> $GITHUB_OUTPUT
          echo "sha=${{ github.sha }}" >> $GITHUB_OUTPUT

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ steps.meta.outputs.version }}
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ steps.meta.outputs.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
          provenance: true
          sbom: true  # Software Bill of Materials

  # ─── 4. Multi-Platform Build ───────────────────────────────────
  build-multi:
    runs-on: ubuntu-latest
    needs: [lint, test]
    permissions:
      packages: write

    strategy:
      matrix:
        platform: [linux/amd64, linux/arm64]

    steps:
      - uses: actions/checkout@v4

      - name: Set up QEMU
        uses: docker/setup-qemu-action@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push (multi-arch)
        uses: docker/build-push-action@v5
        with:
          context: .
          platforms: ${{ matrix.platform }}
          push: true
          tags: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ─── 5. Release (on tag) ────────────────────────────────────────
  release:
    runs-on: ubuntu-latest
    needs: [build]
    if: startsWith(github.ref, 'refs/tags/v')

    steps:
      - uses: actions/checkout@v4

      - name: Download binary
        run: |
          VERSION=${GITHUB_REF#refs/tags/v}
          ARCH=$(go env GOARCH)
          OS=$(go env GOOS)
          curl -sL "https://github.com/${{ github.repository }}/releases/download/v${VERSION}/app_${VERSION}_${OS}_${ARCH}.tar.gz" -o release.tar.gz

      - name: Create checksums
        run: sha256sum release.tar.gz > checksums.txt

      - name: Upload to GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: |
            release.tar.gz
            checksums.txt
          generate_release_notes: true
```

## Multi-Stage Pipeline

### Staging → Production Flow

```yaml
jobs:
  # Stage 1: Quality Gates
  quality-gates:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Run tests
        run: go test -v ./...

      - name: Security scan
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: 'fs'
          severity: 'HIGH,CRITICAL'
          exit-code: '1'  # Fail if vulnerabilities found

  # Stage 2: Deploy to Staging
  deploy-staging:
    runs-on: ubuntu-latest
    needs: quality-gates
    environment: staging

    steps:
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.STAGING_HOST }}
          username: deploy
          key: ${{ secrets.STAGING_SSH_KEY }}
          script: |
            cd /app && docker compose pull
            docker compose up -d
            docker image prune -f

  # Stage 3: Smoke Test
  smoke-test:
    runs-on: ubuntu-latest
    needs: deploy-staging

    steps:
      - name: Health check
        run: |
          for i in {1..30}; do
            HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" https://staging.example.com/health)
            if [ "$HTTP_CODE" = "200" ]; then
              echo "✅ Service is healthy"
              exit 0
            fi
            echo "Waiting... attempt $i/30"
            sleep 10
          done
          echo "❌ Service failed health check"
          exit 1

  # Stage 4: Deploy to Production
  deploy-production:
    runs-on: ubuntu-latest
    needs: smoke-test
    environment: production
    if: github.ref == 'refs/heads/main'  # Only on main branch

    steps:
      - name: Deploy to production
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.PROD_HOST }}
          username: deploy
          key: ${{ secrets.PROD_SSH_KEY }}
          script: |
            cd /app && docker compose pull
            docker compose up -d --remove-orphans
            docker image prune -f
```

### Matrix Jobs

```yaml
jobs:
  test-matrix:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        go-version: ['1.21', '1.22', '1.23']
        platform: [ubuntu-latest, macos-latest]
      fail-fast: false  # Don't cancel all if one fails

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-go@v5
        with:
          go-version: ${{ matrix.go-version }}

      - name: Test on ${{ matrix.platform }}
        run: go test -v ./...
```

## Caching Strategy

### Dependency Cache

```yaml
# ❌ Bad: cache key chỉ dựa trên OS
- uses: actions/cache@v4
  with:
    path: ~/go/pkg/mod
    key: go-modules  # Luôn miss vì không có hash

# ✅ Good: cache key dựa trên go.sum hash
- uses: actions/cache@v4
  with:
    path: ~/go/pkg/mod
    key: go-${{ runner.os }}-${{ hashFiles('**/go.sum') }}
    restore-keys: |
      go-${{ runner.os }}-

# ✅ Best: action setup-go tích hợp sẵn cache
- uses: actions/setup-go@v5
  with:
    go-version: '1.22'
    cache: true  # Tự động cache ~/go/pkg/mod
```

### Docker Layer Cache (BuildKit)

```yaml
- name: Build and push
  uses: docker/build-push-action@v5
  with:
    cache-from: type=gha  # Cache từ GitHub Actions cache
    cache-to: type=gha,mode=max  # Lưu tất cả layers
```

### Composite Cache (nhiều step)

```yaml
- name: Cache all
  uses: actions/cache@v4
  with:
    path: |
      ~/go/pkg/mod
      ~/.cache/go-build
      node_modules
    key: ${{ runner.os }}-deps-${{ hashFiles('**/go.sum', '**/package-lock.json') }}
```

## Reusable Workflows

### `workflows/go-ci.yml` — Reusable workflow cho Go projects

```yaml
# .github/workflows/go-ci.yml
name: Go CI Reusable

on:
  workflow_call:
    inputs:
      go-version:
        required: false
        default: '1.22'
        type: string
      lint-enabled:
        required: false
        default: true
        type: boolean

jobs:
  lint:
    runs-on: ubuntu-latest
    if: inputs.lint-enabled
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: ${{ inputs.go-version }}
      - uses: golangci/golangci-lint-action@v6
        with:
          version: latest

  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: ${{ inputs.go-version }}
          cache: true
      - run: go test -race ./...
```

### Gọi từ workflow khác

```yaml
# .github/workflows/my-app.yml
jobs:
  ci:
    uses: ./.github/workflows/go-ci.yml
    with:
      go-version: '1.22'
      lint-enabled: true
```

## Self-Hosted Runner

```yaml
# Setup self-hosted runner
jobs:
  build:
    runs-on: self-hosted  # Thay vì ubuntu-latest

    # Label để chọn runner cụ thể
    # runs-on: [self-hosted, linux, x64]

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Build
        run: go build -o myapp .
```

**Ưu điểm self-hosted:**
- Free (không tốn phí minute của GitHub)
- Access internal network/resource
- Cache lớn hơn

**Nhược điểm:**
- Tự quản lý infra và bảo mật
- Cần cleanup sau mỗi job

## Docker Compose for Local CI

```yaml
# compose.ci.yml
version: "3.8"
services:
  test:
    image: golang:1.22-alpine
    working_dir: /app
    volumes:
      - .:/app
    command: go test -v -race ./...

  lint:
    image: golangci/golangci-lint:latest
    working_dir: /app
    volumes:
      - .:/app
    command: golangci-lint run --timeout=5m

  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: testdb
      POSTGRES_USER: test
      POSTGRES_PASSWORD: test

  integration-test:
    image: golang:1.22-alpine
    working_dir: /app
    volumes:
      - .:/app
    depends_on:
      postgres:
        condition: service_healthy
    command: go test -v ./tests/integration/...
```

```bash
docker compose -f compose.ci.yml run --rm test
docker compose -f compose.ci.yml run --rm lint
docker compose -f compose.ci.yml run --rm integration-test
```

## Security Best Practices

### Pipeline Security

```yaml
jobs:
  security:
    runs-on: ubuntu-latest
    permissions:
      security-events: write
      contents: read

    steps:
      - uses: actions/checkout@v4
        with:
          persist-credentials: false  # Không lưu credentials

      # 1. Dependency scan
      - name: Run Trivy vulnerability scanner
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: 'fs'
          scan-ref: '.'
          format: 'sarif'
          output: 'trivy-results.sarif'
          severity: 'CRITICAL,HIGH'

      - name: Upload Trivy results to GitHub Security
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: 'trivy-results.sarif'

      # 2. Secret scanning (built-in GitHub)
      # Không cần config — tự động scan commits

      # 3. Supply chain security
      - name: Verify Go dependencies
        run: |
          go mod verify

      # 4. SBOM generation
      - name: Generate SBOM
        uses: anchore/sbom-action@v0
        with:
          image: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
          format: spdx-json
          output-file: sbom.spdx.json
```

### Quy tắc bảo mật pipeline

| Quy tắc | Lý do |
|---------|--------|
| `persist-credentials: false` | Không lưu long-lived credentials |
| `permissions: least-privilege` | Chỉ cấp quyền cần thiết |
| Pin action version (`@v4`) | Tránh supply chain attack |
| Không hardcode secrets | Dùng GitHub Secrets |
| Pin base image version | `golang:1.22` thay vì `golang:latest` |

## Go Release Pipeline

```yaml
name: Release

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Cần cho goreleaser

      - uses: actions/setup-go@v5
        with:
          go-version: '1.22'
          cache: true

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v5
        with:
          version: latest
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          # Các env var khác cho Docker Hub, etc.
```

### `.goreleaser.yaml`

```yaml
# .goreleaser.yaml
builds:
  - id: myapp
    binary: myapp
    goos:
      - linux
      - darwin
      - windows
    goarch:
      - amd64
      - arm64
    flags:
      - -trimpath
    ldflags:
      - -s -w
      - -X main.version={{.Version}}
      - -X main.commit={{.Commit}}
      - -X main.date={{.Date}}

archives:
  - format: tar.gz
    format_overrides:
      - goos: windows
        format: zip

checksums:
  name_template: 'checksums.txt'

snapshot:
  name_template: "{{ .Tag }}-next"

changelog:
  filters:
    exclude:
      - '^docs:'
      - '^test:'
      - '^chore:'
```

## 📌 Tóm tắt

```
CI/CD Pipeline
├── Trigger
│   ├── push / pull_request / tag / schedule
│   └── workflow_dispatch (manual)
│
├── Jobs
│   ├── lint ────────── golangci-lint, go vet
│   ├── test ────────── go test -race -coverprofile
│   ├── build ───────── docker build-push-action
│   ├── build-multi ──── multi-arch (amd64 + arm64)
│   ├── deploy-staging ── SSH / kubectl
│   ├── smoke-test ────── curl health check
│   └── deploy-prod ───── ArgoCD / kubectl / docker
│
├── Caching
│   ├── go mod ─────── actions/cache / setup-go
│   ├── docker ─────── type=gha
│   └── composite ──── nhiều paths cùng key
│
└── Security
    ├── Trivy ───────── vulnerability scan
    ├── SBOM ────────── Software Bill of Materials
    ├── Pin action ──── @v4 thay vì @main
    └── permissions ──── least-privilege
```
