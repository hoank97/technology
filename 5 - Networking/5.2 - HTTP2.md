## HTTP/2 Features

### 1. Multiplexing

**Multiple streams** over single TCP connection:

```
Stream 1: HEADERS + DATA
Stream 2: HEADERS + DATA
Stream 3: HEADERS + DATA
───────────────────────────
         Single TCP Connection
```

→ Không head-of-line blocking → parallel requests

### 2. Header Compression (HPACK)

```
HTTP/1.1:
GET / HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Accept: text/html

HTTP/2 (compressed):
// Chỉ gửi delta của headers
```

→ Giảm header size đáng kể

### 3. Server Push

Server chủ động push resources mà client chưa request:

```
Server Push:
GET /index.html
← push /style.css
← push /app.js
```

→ Client không cần request từng file riêng

### 4. Binary Protocol

```
HTTP/1.1: Text (ASCII)
HTTP/2:    Binary frames
```

→ Parse nhanh hơn, error ít hơn

---

## HTTP/2 vs HTTP/1.1

| | HTTP/1.1 | HTTP/2 |
|---|---|---|
| Multiplexing | ❌ 1 request/response | ✅ Multiple streams |
| Header compression | ❌ Plain text | ✅ HPACK |
| Server push | ❌ Không | ✅ Có |
| Protocol | Text | Binary |
| Connection | Multiple | Single |
| Head-of-line blocking | ✅ Có | ❌ Không |

---

## HTTP/3 (QUIC)

HTTP/3 dùng **QUIC** (UDP-based):

- **0-RTT** handshake → thậm chí nhanh hơn HTTP/2
- Không bị head-of-line blocking ở transport layer
- Built-in encryption (TLS 1.3)
