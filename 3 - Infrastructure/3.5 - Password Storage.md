## ❌ Bad: Simple Hash (MD5, SHA-1)

```sql
-- Vẫn không an toàn
INSERT INTO users (password) VALUES (SHA1('mypassword123'));
```

### Tại sao không?

- **Rainbow table**: precomputed hash → password lookup
- **GPU attack**: hash rate ~10B MD5/sec

---

## ✅ Good: salted hash

### Salt

**Random data** thêm vào trước khi hash:

```go
salt := make([]byte, 16)
rand.Read(salt)

hash := sha256.Sum256(append(salt, password...))
// Lưu: salt + hash
```

→ Mỗi user có salt khác nhau → rainbow table không work.

---

## ✅ Best: Password Hashing Functions

### Bcrypt

```go
import "golang.org/x/crypto/bcrypt"

hash, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
// Cost = 2^12 rounds = 4096 iterations

err = bcrypt.CompareHashAndPassword(hash, []byte(password))
```

- **Adaptive**: có thể tăng cost theo thời gian
- **Built-in salt**: không cần tự thêm salt
- **Default cost**: 10 (có thể tăng lên 12+)

### Argon2 (recommended mới nhất)

```go
import "crypto/argon2"

func hashPassword(password string) string {
    salt := make([]byte, 16)
    rand.Read(salt)
    return argon2.IDKey([]byte(password), salt, 1, 64*1024, 4, 32)
}
```

- Winner of **Password Hashing Competition 2015**
- Resistant against GPU/ASIC attacks

## So sánh

| | MD5/SHA | Bcrypt | Argon2 |
|---|---|---|---|
| Speed | Very fast | Slow | Slow + tunable |
| GPU resistant | ❌ | ✅ | ✅✅ |
| Salt | Manual | Built-in | Built-in |
| Recommended | ❌ | ✅ | ✅✅ |
| Adaptive | ❌ | ✅ | ✅ |

## Checklist

```
Password Storage Checklist
│
├── ✅ Dùng Bcrypt hoặc Argon2
├── ✅ Không bao giờ lưu plain text
├── ✅ Không dùng MD5/SHA cho password
├── ✅ Mỗi user có salt khác nhau
├── ✅ Adaptive cost (tăng theo thời gian)
└── ❌ Không dùng user password trực tiếp làm key
```
