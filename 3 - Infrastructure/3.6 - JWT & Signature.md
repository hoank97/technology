## JWT vs Session

| | JWT | Session |
|---|---|---|
| Storage | Client-side (token) | Server-side (in memory/db) |
| Stateless | ✅ | ❌ |
| Revocation | Khó (stateless) | Dễ (invalidate session) |
| Size | Lớn hơn | Nhỏ |
| Use case | API, microservices | Web apps |

## Digital Signature

Xác minh **integrity** và **authenticity** của message.

### Dùng Asymmetric

```
Sender:
1. Hash(message)
2. Encrypt hash bằng Private Key → Signature
3. Gửi: message + signature

Receiver:
1. Decrypt signature bằng Public Key → original hash
2. Hash(message) → compare
3. Match = valid, tamper-free
```

## JWT Security Tips

| | |
|---|---|
| ✅ Dùng **HTTPS** | Không gửi token qua HTTP |
| ✅ Set **short expiry** | 15min - 1h |
| ✅ Dùng **refresh token** | Để prolong session |
| ✅ Verify **signature** trước khi trust payload |
| ✅ **Don't store sensitive data** trong JWT payload (base64, not encrypted) |
| ❌ Không dùng JWT làm session ID thay vì real session |
