## Hybrid (thực tế dùng)

**Dùng asymmetric để exchange key → symmetric để encrypt data:**

```
1. Client lấy server's public key
2. Client encrypt random session key bằng public key → gửi server
3. Server decrypt bằng private key → lấy session key
4. Cả 2 dùng session key (symmetric) để encrypt communication
```

→ Kết hợp ưu điểm của cả 2: secure key exchange + fast encryption
