## TLS (Transport Layer Security)

TLS = protocol bảo mật communication trên network.

### Mục đích

1. **Encryption**: Data không bị đọc bởi attacker
2. **Integrity**: Data không bị sửa đổi
3. **Authentication**: Xác minh identity của server (qua certificate)

---

## TLS Handshake (Simplified)

```
Client                          Server
  │                               │
  │  ─── ClientHello ──────────→  │  (supported TLS versions, ciphers)
  │                               │
  │  ←── ServerHello + Cert ────  │  (chosen cipher, certificate)
  │  ←── ServerKeyExchange ──────  │
  │                               │
  │  ─── ClientKeyExchange ─────→  │  (premaster secret)
  │  ─── ChangeCipherSpec ───────→ │
  │  ─── Finished ──────────────→  │
  │                               │
  │  ←── ChangeCipherSpec ──────  │
  │  ←── Finished ──────────────  │
  │                               │
  │  ═══ Encrypted Data ═══════════│
```

### Key Exchange

Dùng **asymmetric** để exchange key → **symmetric** để encrypt data (hybrid approach).

## Certificate

### SSL/TLS Certificate

```
Certificate
├── Subject: example.com
├── Issuer: Let's Encrypt (CA)
├── Public Key: ...
├── Validity: 2024-01-01 → 2025-01-01
└── Signature: CA's private key signing
```

### Certificate Authority (CA)

Trusted organization signing certificates:

- **Root CA**: Self-signed (DigiCert, Comodo, Let's Encrypt)
- **Intermediate CA**: Signed by Root CA
- **End-entity**: Server certificates
