# 7.2 Helm

## Overview

Helm = **package manager cho Kubernetes**. Đóng gói manifests (Deployment, Service, ConfigMap...) thành một **Chart** — có thể version, share, và deploy lại một cách deterministic.

```
┌─────────────────────────────────────────────┐
│                 Helm Chart                   │
│                                               │
│  myapp/                                       │
│  ├── Chart.yaml        ← metadata            │
│  ├── values.yaml       ← default config       │
│  ├── values.schema.json ← validation        │
│  ├── templates/         ← K8s manifests       │
│  │   ├── deployment.yaml                     │
│  │   ├── service.yaml                        │
│  │   ├── ingress.yaml                        │
│  │   ├── configmap.yaml                     │
│  │   └── _helpers.tpl  ← named template      │
│  ├── charts/           ← subcharts          │
│  └── README.md         ← documentation       │
└─────────────────────────────────────────────┘
```

**Tại sao Helm?**

| Không dùng Helm | Dùng Helm |
|----------------|-----------|
| `kubectl apply -f app.yaml` → stateless, không có version history | ✅ Release history — `helm history` → rollback được |
| Copy-paste manifests qua nhiều env (prod/staging/dev) | ✅ `values.yaml` per env, template engine |
| Config thủ công qua `kubectl edit` → không tái hiện được | ✅ Tái deploy từ Chart = deterministic |
| Không có dependency management | ✅ Subcharts (postgres, redis...) |
| Không có validation | ✅ JSON Schema validation |

## Chart Structure

### `Chart.yaml`

```yaml
apiVersion: v2          # v2 = Helm 3, v1 = Helm 2 (deprecated)
name: myapp
description: My application Helm chart
type: application
version: 1.2.3          # Chart version (semver)
appVersion: "1.2.0"    # Application version (app's version)
keywords:
  - web
  - api
home: https://github.com/myorg/myapp
sources:
  - https://github.com/myorg/myapp
maintainers:
  - name: DevOps Team
    email: devops@example.com
dependencies:           # Subcharts (v2 only)
  - name: postgresql
    version: "12.x.x"
    repository: "https://charts.bitnami.com/bitnami"
```

### `values.yaml`

```yaml
# Default values — applied to all environments

replicaCount: 3

image:
  repository: ghcr.io/myorg/myapp
  pullPolicy: IfNotPresent
  tag: "1.0.0"

service:
  type: ClusterIP
  port: 80

ingress:
  enabled: true
  className: nginx
  host: myapp.example.com
  path: /
  tls:
    enabled: true
    secretName: myapp-tls

resources:
  requests:
    cpu: 100m
    memory: 128Mi
  limits:
    cpu: 500m
    memory: 256Mi

autoscaling:
  enabled: true
  minReplicas: 2
  maxReplicas: 20
  targetCPUUtilizationPercentage: 70

env:
  APP_ENV: production
  LOG_LEVEL: info

config:
  pool_size: "10"
  timeout: "30s"

persistence:
  enabled: false
  size: 1Gi
  storageClass: standard

podSecurityContext:
  runAsNonRoot: true
  runAsUser: 1000
  fsGroup: 1000

containerSecurityContext:
  readOnlyRootFilesystem: true
  allowPrivilegeEscalation: false
```

### `templates/_helpers.tpl`

```tpl
{{/* Expand name to fullname */}}
{{- define "myapp.fullname" -}}
{{- if .Values.fullnameOverride }}
{{- .Values.fullnameOverride | trunc 63 }}
{{- else }}
{{- $name := default .Chart.Name .Values.nameOverride }}
{{- if contains $name .Release.Name }}
{{- .Release.Name | trunc 63 }}
{{- else }}
{{- printf "%s-%s" .Release.Name $name | trunc 63 }}
{{- end }}
{{- end }}
{{- end }}

{{/* Standard labels */}}
{{- define "myapp.labels" -}}
app.kubernetes.io/name: {{ include "myapp.fullname" . }}
app.kubernetes.io/instance: {{ .Release.Name }}
app.kubernetes.io/version: {{ .Chart.AppVersion | quote }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
app.kubernetes.io/component: backend
{{- end }}

{{/* Selector labels (immutable — pod selector phải match) */}}
{{- define "myapp.selectorLabels" -}}
app.kubernetes.io/name: {{ include "myapp.fullname" . }}
app.kubernetes.io/instance: {{ .Release.Name }}
{{- end }}

{{/* image full tag */}}
{{- define "myapp.imageTag" -}}
{{- default .Chart.AppVersion .Values.image.tag | quote }}
{{- end }}
```

### `templates/deployment.yaml`

```tpl
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ include "myapp.fullname" . }}
  labels:
    {{- include "myapp.labels" . | nindent 4 }}
spec:
  replicas: {{ .Values.replicaCount }}
  selector:
    matchLabels:
      {{- include "myapp.selectorLabels" . | nindent 6 }}
  strategy:
    type: {{ .Values.strategy | default "RollingUpdate" }}
  template:
    metadata:
      labels:
        {{- include "myapp.labels" . | nindent 8 }}
    spec:
      securityContext:
        {{- toYaml .Values.podSecurityContext | nindent 8 }}
      containers:
        - name: {{ .Chart.Name }}
          securityContext:
            {{- toYaml .Values.containerSecurityContext | nindent 12 }}
          image: "{{ .Values.image.repository }}:{{ include "myapp.imageTag" . }}"
          imagePullPolicy: {{ .Values.image.pullPolicy }}
          ports:
            - name: http
              containerPort: {{ .Values.service.port }}
              protocol: TCP
          livenessProbe:
            httpGet:
              path: /healthz
              port: http
            initialDelaySeconds: 5
            periodSeconds: 10
          readinessProbe:
            httpGet:
              path: /ready
              port: http
            initialDelaySeconds: 5
            periodSeconds: 5
          resources:
            {{- toYaml .Values.resources | nindent 10 }}
          env:
            {{- range $key, $value := .Values.env }}
            - name: {{ $key }}
              value: {{ $value | quote }}
            {{- end }}
            - name: RELEASE_NAME
              value: {{ .Release.Name | quote }}
            - name: RELEASE_NAMESPACE
              value: {{ .Release.Namespace | quote }}
            - name: RELEASE_REVISION
              value: {{ .Release.Revision | quote }}
          envFrom:
            - configMapRef:
                name: {{ include "myapp.fullname" . }}-config
```

### `templates/ingress.yaml`

```tpl
{{- if .Values.ingress.enabled -}}
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: {{ include "myapp.fullname" . }}
  labels:
    {{- include "myapp.labels" . | nindent 4 }}
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: {{ .Values.ingress.className }}
  {{- if .Values.ingress.tls.enabled }}
  tls:
    - hosts:
        - {{ .Values.ingress.host }}
      secretName: {{ .Values.ingress.tls.secretName }}
  {{- end }}
  rules:
    - host: {{ .Values.ingress.host | quote }}
      http:
        paths:
          - path: {{ .Values.ingress.path }}
            pathType: Prefix
            backend:
              service:
                name: {{ include "myapp.fullname" . }}

```

```tpl

    {{- include "myapp.labels" . | nindent 4 }}

        target:
          type: Utilization
```

## Template Functions Reference

### Common functions

```tpl
# String
{{ .Values.image.tag }}                  # Get value
{{ "myapp" | upper }}                    # "MYAPP"
{{ "hello world" | title }}              # "Hello World"
{{ "myapp" | lower }}                    # "myapp"

# Conditional
{{- if .Values.ingress.enabled }}
apiVersion: networking.k8s.io/v1
{{- end }}

{{- if eq .Release.Namespace "production" }}
  replicaCount: 5
{{- else }}
  replicaCount: 1
{{- end }}

# Default value
replicas: {{ .Values.replicaCount | default 3 }}

# Lookup (runtime values — NOT available during templates/ validation)
{{- range (lookup "v1" "Namespace" "" "").items }}
{{ .metadata.name }}
{{- end }}

# toYaml
{{- toYaml .Values.resources | nindent 8 }}

# include (call named template)
{{- include "myapp.labels" . | nindent 4 }}

# required (fail if missing)
image: "{{ .Values.image.repository }}/{{ required "image.tag is required" .Values.image.tag }}"
```

### Sprig functions (available in Helm templates)

```tpl
# Date/time
{{ now | date "2006-01-02" }}

# String manipulation
{{ "hello" | b64enc }}          # base64 encode
{{ "aGVsbG8=" | b64dec }}      # base64 decode
{{ "hello" | sha256sum }}       # hash
{{ "hello" | quote }}           # "\"hello\""
{{ "hello" | squote }}          # "'hello'"

# List/collection
{{- range $i, $v := .Values.env }}
{{ $i }}: {{ $v }}
{{- end }}

{{- range .Values.replicas }}
- {{ . }}
{{- end }}

# Default
replicas: {{ .Values.replicaCount | default 3 }}

# Ternary
{{ .Values.podSecurityContext.runAsNonRoot | default true | ternary "runAsNonRoot" "runAsRoot" }}
```

## Environment-Specific Values

### Per-environment overrides

```bash
# Development
helm install myapp ./myapp \
  --namespace dev \
  --set image.tag=dev-$(git rev-parse --short HEAD) \
  --set replicaCount=1 \
  --set autoscaling.enabled=false \
  --values values.dev.yaml

# Staging
helm install myapp ./myapp \
  --namespace staging \
  --set image.tag=staging-latest \
  --set replicaCount=2 \
  --set autoscaling.enabled=true \
  --values values.staging.yaml

# Production
helm install myapp ./myapp \
  --namespace production \
  --set image.tag=1.2.3 \
  --set replicaCount=5 \
  --values values.prod.yaml
```

## Helm Hooks

Hooks cho phép execute logic ở certain points trong release lifecycle.

```yaml
# templates/job-migrations.yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: {{ include "myapp.fullname" . }}-migrations
  annotations:
    "helm.sh/hook": pre-upgrade, pre-install   # ⚠️ CRITICAL: run before upgrade/install
    "helm.sh/hook-weight": "1"                 # Lower = run first
    "helm.sh/hook-delete-policy": before-hook-creation, hook-failed
spec:
  template:
    spec:
      restartPolicy: OnFailure
      containers:
        - name: migrations
          image: "{{ .Values.image.repository }}:{{ include "myapp.imageTag" . }}"
          command: ["migrate", "up"]
```

### Hook Types & Weights

| Annotation | Khi nào chạy | Use case |
|-----------|-------------|----------|
| `pre-install` | Trước khi install, trước render templates | Init resources |
| `post-install` | Sau khi install | Post-install setup |
| `pre-upgrade` | Trước khi upgrade | DB migrations |
| `post-upgrade` | Sau khi upgrade | Notify, verify |
| `pre-rollback` | Trước khi rollback | Cleanup |
| `post-rollback` | Sau khi rollback | Notify |
| `pre-delete` | Trước khi delete | Cleanup resources |
| `post-delete` | Sau khi delete | Cleanup external resources |

### Hook Weights

```yaml
annotations:
  "helm.sh/hook-weight": "-5"   # Run early (negative = very early)
  # or
  "helm.sh/hook-weight": "10"    # Run later (higher = later)
```

**Best practice:** Dùng weighted hooks cho multi-step operations (VD: migration → validate → notify).

## Helm Release Management

### Basic Commands

```bash
# Install
helm install myapp ./myapp --namespace production
helm install myapp ./myapp --namespace production --create-namespace

# Dry-run (xem rendered manifests)
helm install myapp ./myapp --namespace production --dry-run
helm install myapp ./myapp --namespace production --dry-run --debug

# Template only (render, không apply)
helm template myapp ./myapp --namespace production > rendered.yaml

# Upgrade
helm upgrade myapp ./myapp --namespace production
helm upgrade myapp ./myapp --namespace production --atomic   # Auto rollback on failure

# Upgrade with values
helm upgrade myapp ./myapp --namespace production \
  --set replicaCount=10 \
  --set image.tag=1.3.0

# Rollback
helm rollback myapp 1 --namespace production     # Rollback to revision 1
helm rollback myapp --namespace production        # Rollback 1 version

# Release history
helm history myapp --namespace production
helm rollback myapp 3 --namespace production

# Uninstall
helm uninstall myapp --namespace production

# List all releases
helm list --namespace production
helm list --all-namespaces
```

### Atomic & Wait Flags

```bash
# --atomic: auto-rollback nếu upgrade fail
helm upgrade myapp ./myapp --namespace production --atomic --timeout 5m

# --timeout: max time cho upgrade operation
helm upgrade myapp ./myapp --namespace production --timeout 3m

# --wait: đợi tất cả pods ready trước khi complete
helm upgrade myapp ./myapp --namespace production --wait --timeout 5m

# --wait-for-jobs: đợi Jobs complete
helm upgrade myapp ./myapp --namespace production --wait --wait-for-jobs --timeout 10m
```

### Test Release


## Helm Testing in CI

### Test Helm chart locally

```bash
# Lint chart
helm lint ./myapp
helm lint ./myapp --values values.prod.yaml

# Kubeval (validate against K8s schema)
helm template myapp ./myapp | kubeval

# Chart-testing (ct)
# .github/workflows/helm-lint.yaml
name: Helm Lint and Test

on:
  pull_request:
    paths:
      - 'myapp/**'

jobs:
  helm-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Helm
        uses: azure/setup-helm@v4
        with:
          version: v3.14.0

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install chart-testing
        run: pip install chart-testing

      - name: Run chart-testing
        run: ct lint --charts-dir . --charts myapp --validate-maintainers false
```

### Promote across environments

```bash
#!/bin/bash
# scripts/helm-promote.sh

set -e

CHART_DIR="./myapp"
RELEASE_NAME="myapp"
IMG_TAG="${1:-latest}"

# 1. Test in dev
helm upgrade --install "$RELEASE_NAME" "$CHART_DIR" \
  --namespace dev \
  --values "$CHART_DIR/values.dev.yaml" \
  --set image.tag="$IMG_TAG" \
  --atomic --timeout 5m --wait

echo "✅ Dev deployed: $IMG_TAG"

# 2. Smoke test
sleep 30
curl -sf "https://dev.myapp.example.com/ready" || exit 1
echo "✅ Dev smoke test passed"

# 3. Promote to staging
read -p "Promote $IMG_TAG to staging? (y/n): " confirm
if [ "$confirm" = "y" ]; then
  helm upgrade --install "$RELEASE_NAME" "$CHART_DIR" \
    --namespace staging \
    --values "$CHART_DIR/values.staging.yaml" \
    --set image.tag="$IMG_TAG" \
    --atomic --timeout 5m --wait

  echo "✅ Staging deployed: $IMG_TAG"
fi
```

```yaml

  name: {{ .Release.Name }}

    {{- include "common.labels" . | nindent 4 }}

    matchLabels:
      {{- include "common.selectorLabels" . | nindent 6 }}
```

## Helm Security

```bash
# Sign a chart (package)
helm sign myapp-1.0.0.tgz --key mykey --keyring ~/.gnupg/secring.gpg

# Verify on install
helm install myapp myapp-1.0.0.tgz --verify --keyring ~/.gnupg/pubring.gpg

# OCI Registry (Helm 3.8+)
helm chart save myapp ghcr.io/myorg/myapp:1.0.0
helm chart push ghcr.io/myorg/myapp:1.0.0
helm chart pull ghcr.io/myorg/myapp:1.0.0

# Sign and push
helm registry login ghcr.io
helm chart save myapp ghcr.io/myorg/myapp:1.0.0
helm chart push --sign ghcr.io/myorg/myapp:1.0.0 --keyring ~/.gnupg/secring.gpg
```

### OCI-based workflow

```bash
# Build and push chart
helm package myapp
helm push myapp-1.0.0.tgz oci://ghcr.io/myorg/

# Pull and install from registry
helm pull oci://ghcr.io/myorg/myapp --version 1.0.0
helm upgrade --install myapp oci://ghcr.io/myorg/myapp \
  --version 1.0.0 \
  --namespace production \
  --values values.prod.yaml
```

## 📌 Tóm tắt

```
Helm
│
├── Chart Structure
│   ├── Chart.yaml ──────── Metadata, deps, version
│   ├── values.yaml ─────── Default config (all envs)
│   ├── values.prod.yaml ── Prod overrides
│   ├── values.schema.json ─ JSON validation
│   └── templates/ ──────── K8s manifests + _helpers.tpl
│
├── Template Functions
│   ├── {{ .Values.x }} ── Access values
│   ├── {{ include }} ──── Named templates
│   ├── {{- if }} ───────── Conditional
│   ├── {{- range }} ───── Loops
│   ├── toYaml / nindent ─ Formatting
│   └── Sprig funcs ────── b64enc, sha256sum, date...
│
├── Hooks
│   ├── pre-install / post-install
│   ├── pre-upgrade / post-upgrade
│   ├── pre-rollback / post-rollback
│   ├── hook-weight: control execution order
│   └── hook-delete-policy: cleanup on fail
│
├── Commands
│   ├── install / upgrade / rollback / uninstall
│   ├── template ───────── Render without apply
│   ├── lint ───────────── Validate syntax
│   ├── test ───────────── Run test hooks
│   ├── history ─────────── Release revisions
│   └── --atomic ───────── Auto-rollback on failure
│
└── Registry
    ├── OCI push/pull ───── ghcr.io/myorg/myapp:1.0.0
    └── Sign & verify ─────── helm sign / --verify
```
