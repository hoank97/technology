# 7.3 GitOps

## Overview

GitOps = **"Git is the single source of truth"** cho infrastructure và application deployments.

```
┌─────────────────────────────────────────────────────────────────┐
│  Git Repository (Single Source of Truth)                        │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ infra/                                                  │  │
│  │ ├── helm/myapp/values.prod.yaml                        │  │
│  │ ├── helm/myapp/values.staging.yaml                     │  │
│  │ ├── k8s/namespace.yaml                                │  │
│  │ └── terraform/ (IaC)                                   │  │
│  └─────────────────────────────────────────────────────────┘  │
│                              │                                │
│                    git push / merge                           │
│                              ▼                                │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │           ArgoCD / Flux (GitOps Controller)              │  │
│  │  1. Watch git (detect change)                          │  │
│  │  2. Compare: git state vs cluster state                │  │
│  │  3. Sync: apply diff to cluster                       │  │
│  │  4. Report: status back to Git (Commit Status / PR)   │  │
│  └─────────────────────────────────────────────────────────┘  │
│                              │                                │
│                              ▼                                │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │           Kubernetes Cluster                             │  │
│  │  myapp deployed ✓                                       │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### GitOps Principles (WeaveWorks)

1. **The entire system is described declaratively** — Tất cả configs ở dạng YAML/JSON, không manual `kubectl apply`
2. **The desired state is versioned in Git** — Immutable, có history, có rollback
3. **Approved changes are automatically pushed to the system** — Controller sync loop
4. **Software agents ensure correctness and alert on drift** — Reconciliation loop

### Application CRD

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: myapp
  namespace: argocd
  finalizers:
    - resources-finalizer.argocd.argoproj.io  # Cleanup on delete
spec:
  project: production
  source:
    repoURL: https://github.com/myorg/k8s-config.git
    targetRevision: main
    path: helm/myapp
    helm:
      valueFiles:
        - values.prod.yaml
      parameters:
        - name: image.tag
          value: "1.2.3"
        - name: replicaCount
          value: "5"

  destination:
    server: https://kubernetes.default.svc
    namespace: production

  # Sync policy
  syncPolicy:
    automated:
      selfHeal: true     # Auto sync on drift (cluster → git)
      prune: true        # Auto delete resources removed from git
      allowEmpty: false  # Allow application with no resources
    syncOptions:
      - CreateNamespace=true
      - PruneLast=true   # Prune after apply
    retry:
      limit: 5
      backoff:
        duration: 5s
        factor: 2
        maxDuration: 3m

  # Ignore differences (functional fields that K8s manages)
  ignoreDifferences:
    - group: apps
      kind: Deployment
      jsonPointers:
        - /spec/replicas  # ArgoCD ignores this field during diff
```

### RBAC (ArgoCD)

```yaml
# argo-rbac.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: argocd-rbac-cm
data:
  policy.csv: |
    # Policy format: p, <subject>, <resource>, <action>, <object>

    # Dev team: full access to staging project
    p, dev-team, applications, *, staging/*           # allow

    # Dev team: read-only on production project
    p, dev-team, applications, get, production/*     # allow
    p, dev-team, applications, sync, production/*    # deny

    # CI: can sync any app
    p, ci-system, applications, sync, *             # allow

    # On-call: full access
    p, on-call, *, *, *                             # allow

  # Default role: readonly
  policy.default: role:readonly
```

## Drift Detection & Reconciliation

### What is Drift?

```
Git says:                  Cluster says:
replicas: 5               replicas: 3         ← DRIFT!
image: myapp:1.2.3        image: myapp:1.2.3   ← OK
```

**Drift xảy ra khi:**
- Someone run `kubectl scale` mà không qua GitOps
- Cluster auto-scaled replicas (HPA thay đổi)
- Cloud provider changed managed resources
- Manual edits không qua ArgoCD

### ArgoCD: Drift Handling

```yaml
spec:
  syncPolicy:
    automated:
      selfHeal: true    # ✅ Cluster → Git: auto sync back
      prune: true      # ✅ Remove resources not in git
```

```bash
# Manual sync (override auto-sync)
argocd app sync myapp --force

# Refresh app state from git
argocd app get myapp --refresh

# Show diff
argocd app diff myapp

# Sync only specific resources
argocd app sync myapp --resource APPLICATION/myapp/Deployment/myapp
```

### Flux: Drift Handling

```yaml
spec:
  path: ./k8s
  prune: true          # Remove resources not in git
  # Flux does NOT auto-heal by default — intentional choice
  # Manual: flux reconcile kustomization myapp-infra --with-source
```

```bash
flux reconcile kustomization myapp-infra --with-source
flux events --kind=Kustomization --name=myapp-infra
flux get kustomization myapp-infra
```

## Progressive Delivery (Canary)

### ArgoCD Rollouts

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: myapp
spec:
  replicas: 5
  strategy:
    canary:
      # Traffic management (Istio/NGINX/ALB)
      trafficRouting:
        - plugin: argocd-nft
        # or Istio:
        - istio:
            virtualService:
              name: myapp-vsvc
              routes:
                - primary
      # Stable pod analysis
      canaryMetadata:
        labels:
          role: canary
      stableMetadata:
        labels:
          role: stable
      # Steps: progressive rollout
      steps:
        - setWeight: 5     # 5% traffic to canary
        - pause: {}         # Wait for manual approval (or time)
          duration: 5m
        - setWeight: 20    # 20%
        - analysis:
            templates:
              - templateName: myapp-analysis
            args:
              - name: rollout-name
                value: "{{rollout.metadata.name}}"
        - setWeight: 50    # 50%
        - pause: {}         # Final check
          duration: 10m
        - setWeight: 80    # 80%
        - setWeight: 100   # Full switch
      # Auto rollback on failure
      analysis:
        successfulRunHistoryLimit: 3
        failedRunHistoryLimit: 3
        templates:
          - templateName: myapp-analysis

---
# AnalysisTemplate (Prometheus-based)
apiVersion: argoproj.io/v1alpha1
kind: AnalysisTemplate
metadata:
  name: myapp-analysis
spec:
  args:
    - name: rollout-name
  metrics:
    - name: error-rate
      interval: 1m
      successCondition: result[0] < 0.01
      failureLimit: 2
      provider:
        prometheus:
          address: http://prometheus:9090
          query: |
            sum(rate(http_requests_total{status=~"5..", job="{{args.rollout-name}}"}[3m]))
            /
            sum(rate(http_requests_total{job="{{args.rollout-name}}"}[3m]))

    - name: latency
      interval: 1m
      successCondition: result[0] < 2
      provider:
        prometheus:
          address: http://prometheus:90
          query: |
            histogram_quantile(0.99,
              sum(rate(http_request_duration_seconds_bucket{job="{{args.rollout-name}}"}[3m])) by (le)
            )
```

## Image Update Automation

### Flux Image Automation

```yaml
apiVersion: image.toolkit.fluxcd.io/v1beta2
kind: ImageRepository
metadata:
  name: myapp
spec:
  image: ghcr.io/myorg/myapp
  interval: 1m
  secretRef:
    name: ghcr-credentials
  certSecretRef:
    name: ghcr-cert
---
apiVersion: image.toolkit.fluxcd.io/v1beta1
kind: ImagePolicy
metadata:
  name: myapp
spec:
  imageRepositoryRef:
    name: myapp
  policy:
    semver:
      range: 1.2.x       # Only patch versions
    # or: latest (always newest tag)
    # or: numeric (newest by number)
---
apiVersion: image.toolkit.fluxcd.io/v1beta2
kind: ImageUpdateAutomation
metadata:
  name: myapp
spec:
  sourceRef:
    kind: GitRepository
    name: myapp
  git:
    localSecretRef:
      name: git-credentials
    branch: main
    commit:
      author:
        email: flux@example.com
        name: flux
      message: |
        ci: automated image update
          {{range .Updated.Images}}{{println .}}{{end}}
  push:
    branch: main
  update:
    strategy: Setters   # Uses {{ .Values.image.tag }} setter
    path: k8s/overlays/production
```

**Result:** Khi có `myapp:1.2.4` tag mới → Flux tự update git → ArgoCD tự sync → cluster updated.

## CI/CD vs GitOps

| | CI/CD Pipeline | GitOps |
|---|---|---|
| **Trigger** | On code commit | On git commit |
| **Who applies?** | CI system (`kubectl apply`) | GitOps controller (in cluster) |
| **Credentials** | CI needs cluster access | Only controller needs cluster access |
| **Audit** | CI logs | Git history (immutable) |
| **Drift** | Pipeline must re-run | Controller auto-heals |
| **Rollback** | `git revert` + pipeline | `git revert` → auto-sync |
| **Best for** | Fast feedback, complex transforms | Production, declarative infra |

**GitOps = CI/CD + Declarative Infrastructure**
> GitOps không replace CI — CI vẫn build + push image. GitOps thay thế phần **deploy** của CI.

## GitOps Workflow (Complete)

```
1. Developer push code
        │
2. CI Pipeline (GitHub Actions)
        ├── go test -race ./...
        ├── go build -o myapp
        ├── docker build -t myapp:1.2.4 .
        ├── docker push ghcr.io/myorg/myapp:1.2.4
        │
3. Image Tag Update (Flux Image Automation)
        └── Update values.prod.yaml: image.tag: "1.2.4"
                │
4. ArgoCD/Flux Sync
        └── Detect change in git
            └── Compare git vs cluster state
                └── Diff detected: replicas: 5 vs 3
                    └── Apply: kubectl apply -f manifests
                        └── ✅ Cluster matches git

5. Status Report
        └── ArgoCD: Application Status = Synced ✓
```
