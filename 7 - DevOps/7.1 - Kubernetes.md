# 7.1 Kubernetes

## Overview

Kubernetes (K8s) là **container orchestrator** — tự động deploy, scale, và manage containerized applications trên nhiều nodes.

```
┌─────────────────────────────────────────────┐
│                  Kubernetes Cluster           │
│                                               │
│  ┌──────────┐    ┌──────────┐              │
│  │  Node 1   │    │  Node 2   │              │
│  │ ┌──────┐ │    │ ┌──────┐ │              │
│  │ │ Pod A │ │    │ │ Pod C │ │              │
│  │ └──────┘ │    │ └──────┘ │              │
│  │ ┌──────┐ │    │ ┌──────┐ │              │
│  │ │ Pod B │ │    │ │ Pod D │ │              │
│  │ └──────┘ │    │ └──────┘ │              │
│  └──────────┘    └──────────┘              │
│         │               │                   │
│  ┌──────────────────────────────────────┐   │
│  │          Control Plane               │   │
│  │  API Server │ etcd │ Scheduler │ CM  │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

## Core Objects

### Pod

Pod = **smallest deployable unit**. Chứa một hoặc nhiều containers chia sẻ network namespace.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    app: nginx
    version: v1
spec:
  containers:
    - name: nginx
      image: nginx:1.25-alpine
      ports:
        - containerPort: 80
      resources:
        requests:
          memory: "64Mi"
          cpu: "250m"
        limits:
          memory: "128Mi"
          cpu: "500m"
      livenessProbe:
        httpGet:
          path: /healthz
          port: 80
        initialDelaySeconds: 3
        periodSeconds: 10
      readinessProbe:
        httpGet:
          path: /ready
          port: 80
        initialDelaySeconds: 5
        periodSeconds: 5
```

**Pod lifecycle:**
```
Pending → Running → Succeeded/Failed
         ↗ Terminating (graceful shutdown)
```

### ReplicaSet

Đảm bảo số lượng pod replicas luôn chạy. Thường **không dùng trực tiếp** — dùng Deployment.

```yaml
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: nginx-replicas
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.25-alpine
```

### Deployment

Quản lý ReplicaSet, hỗ trợ **rolling update** và **rollback**.

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
  labels:
    app: myapp
spec:
  replicas: 3
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1        # Thêm tối đa bao nhiêu pod mới
      maxUnavailable: 0  # Không giảm pod cũ
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
        version: v1.0.0
    spec:
      containers:
        - name: myapp
          image: myapp:1.0.0
          ports:
            - containerPort: 8080
          env:
            - name: DATABASE_URL
              valueFrom:
                secretKeyRef:
                  name: myapp-secrets
                  key: database-url
```

**Rolling update strategy:**

| Strategy | maxSurge | maxUnavailable | Use case |
|----------|----------|----------------|----------|
| `RollingUpdate` | 1 | 0 | Zero-downtime, gradual |
| `RollingUpdate` | 0 | 1 | Cost-sensitive |
| `Recreate` | — | — | Database migration |

**Commands:**

```bash
kubectl apply -f deployment.yaml
kubectl rollout status deployment/myapp
kubectl rollout history deployment/myapp
kubectl rollout undo deployment/myapp              # Rollback 1 version
kubectl rollout undo deployment/myapp --to-revision=3  # Rollback to specific
kubectl set image deployment/myapp myapp=myapp:1.1.0
kubectl scale deployment/myapp --replicas=10
kubectl rollout restart deployment/myapp  # Restart without image change
```

## Service

### Service Types

```yaml
# ClusterIP (default) — chỉ accessible trong cluster
apiVersion: v1
kind: Service
metadata:
  name: myapp-clusterip
spec:
  type: ClusterIP
  selector:
    app: myapp
  ports:
    - port: 80          # Service port
      targetPort: 8080  # Container port

---

# NodePort — expose ra external qua <NodeIP>:<NodePort>
apiVersion: v1
kind: Service
metadata:
  name: myapp-nodeport
spec:
  type: NodePort
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 8080
      nodePort: 30080  # Optional: specific port

---

# LoadBalancer — cloud provider load balancer
apiVersion: v1
kind: Service
metadata:
  name: myapp-lb
spec:
  type: LoadBalancer
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 8080

---

# Headless — không có ClusterIP, trả về pod IPs trực tiếp
apiVersion: v1
kind: Service
metadata:
  name: myapp-headless
spec:
  type: ClusterIP
  clusterIP: None  # Headless
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 8080
```

## Resource Management

### Requests vs Limits

```
        Node capacity
        ├─────────────────────────────────────┤
        │
  Request│  ┌─────────────────────────────────┐
  (guar) │  │  Guaranteed: Pod được đặt ở đây  │
         │  └─────────────────────────────────┘
         │
  Limit  │  ┌─────────────────────────────────┐
         │  │  Burst: Pod có thể dùng đến đây  │
         │  │  (nếu node còn resource)         │
         │  └─────────────────────────────────┘
```

```yaml
resources:
  requests:
    memory: "256Mi"  # Guaranteed
    cpu: "250m"      # 0.25 CPU cores
  limits:
    memory: "512Mi"  # OOMKill if exceeded
    cpu: "1000m"     # Throttled if exceeded
```

### ResourceQuota & LimitRange

```yaml
# Namespace-level quota
apiVersion: v1
kind: ResourceQuota
metadata:
  name: compute-quota
spec:
  hard:
    requests.cpu: "4"
    requests.memory: "8Gi"
    limits.cpu: "8"
    limits.memory: "16Gi"
    pods: "20"

---
# Default requests/limits cho Pod
apiVersion: v1
kind: LimitRange
metadata:
  name: default-limits
spec:
  limits:
    - type: Container
      default:
        memory: "256Mi"
        cpu: "200m"
      defaultRequest:
        memory: "128Mi"
        cpu: "100m"
      max:
        memory: "1Gi"
        cpu: "1"
```

## Scaling

### HorizontalPodAutoscaler (HPA)

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: myapp-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: myapp
  minReplicas: 2
  maxReplicas: 20
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
    - type: Resource
      resource:
        name: memory
        target:
          type: Utilization
          averageUtilization: 80
  behavior:
    scaleDown:
      stabilizationWindowSeconds: 300  # 5 phút trước khi scale down
      policies:
        - type: Percent
          value: 10
          periodSeconds: 60
    scaleUp:
      stabilizationWindowSeconds: 0  # Immediate scale up
      policies:
        - type: Percent
          value: 100
          periodSeconds: 15
```

```bash
kubectl autoscale deployment myapp --min=2 --max=20 --cpu-percent=70
kubectl get hpa
kubectl describe hpa myapp
```

## Storage

### PersistentVolume (PV) & PersistentVolumeClaim (PVC)

```yaml
# Static provisioning
apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv-data
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce      # 1 node read-write
    # - ReadOnlyMany      # Nhiều nodes read-only
    # - ReadWriteMany     # Nhiều nodes read-write (NFS, CephFS)
  storageClassName: standard
  hostPath:
    path: /data/pv-data

---
# PVC (Pod request PV)
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: data-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
  storageClassName: standard

---
# Pod use PVC
spec:
  containers:
    - name: myapp
      volumeMounts:
        - name: data
          mountPath: /data
  volumes:
    - name: data
      persistentVolumeClaim:
        claimName: data-pvc
```

## RBAC (Role-Based Access Control)

```yaml
# 1. ServiceAccount
apiVersion: v1
kind: ServiceAccount
metadata:
  name: myapp-sa
  namespace: production

---
# 2. Role (namespace-scoped)
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: production
  name: myapp-role
rules:
  - apiGroups: [""]
    resources: ["configmaps", "secrets"]
    verbs: ["get", "list", "watch"]
  - apiGroups: ["apps"]
    resources: ["deployments"]
    verbs: ["get", "list", "watch", "update", "patch"]
  - apiGroups: [""]
    resources: ["pods"]
    verbs: ["get", "list", "watch", "logs"]
    resourceNames: ["myapp*"]  # Chỉ pods bắt đầu bằng myapp

---
# 3. RoleBinding (gán Role cho ServiceAccount)
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: myapp-rolebinding
  namespace: production
subjects:
  - kind: ServiceAccount
    name: myapp-sa
    apiGroup: ""
roleRef:
  kind: Role
  name: myapp-role
  apiGroup: ""
```

**ClusterRole (cluster-wide scope):**


## Liveness, Readiness, Startup Probes

```yaml
spec:
  containers:
    - name: myapp
      # Liveness: container còn alive không?
      livenessProbe:
        httpGet:
          path: /healthz
          port: 8080
        failureThreshold: 3      # Fail 3 lần → restart
        periodSeconds: 10

      # Readiness: container sẵn sàng nhận traffic?
      readinessProbe:
        httpGet:
          path: /ready
          port: 8080
        failureThreshold: 1      # Fail 1 lần → remove from Service
        successThreshold: 1
        periodSeconds: 5

      # Startup: container cần time để init?
      startupProbe:
        httpGet:
          path: /healthz
          port: 8080
        failureThreshold: 30    # Thử 30 lần trong 60s
        periodSeconds: 2
        # → Nghĩa là container có 60s để start
```

**Khi nào dùng probe nào?**

| Probe | Mục đích | Failure action |
|-------|---------|----------------|
| `livenessProbe` | Container bị deadlock, không respond | **Restart** container |
| `readinessProbe` | Container chưa ready nhận traffic | **Remove** from Service endpoints |
| `startupProbe` | Container cần init time dài | **Delay** các probe khác |

## Scheduling

### Node Selector & Affinity

```yaml
spec:
  containers:
    - name: myapp
      image: myapp:v1
  # Chỉ chạy trên nodes có label disk-type=ssd
  nodeSelector:
    disk-type: ssd

---
# Pod Affinity (prefer / require)
spec:
  affinity:
    # Muốn chạy gần app=cache (cùng node hoặc cùng zone)
    podAffinity:
      preferredDuringSchedulingIgnoredDuringExecution:
        - weight: 100
          podAffinityTerm:
            labelSelector:
              matchLabels:
                app: cache
            topologyKey: topology.kubernetes.io/zone

    # Tránh chạy gần app=cache
    podAntiAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        - labelSelector:
            matchLabels:
              app: cache
          topologyKey: kubernetes.io/hostname

    # Node affinity
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
          - matchExpressions:
              - key: node-type
                operator: In
                values: [compute-optimized]
      preferredDuringSchedulingIgnoredDuringExecution:
        - weight: 50
          preference:
            matchExpressions:
              - key: gpu
                operator: Exists
```

### Taints & Tolerations

```bash
# Taint a node (không cho pod thường chạy)
kubectl taint nodes node1 gpu=true:NoSchedule

# Pod cần toleration mới được schedule vào
```

```yaml
spec:
  tolerations:
    - key: "gpu"
      operator: "Equal"
      value: "true"
      effect: "NoSchedule"
    - key: "node.kubernetes.io/not-ready"
      operator: "Exists"
      effect: "NoExecute"
      tolerationSeconds: 300  # Cho phép chờ 5 phút
```

## Jobs & CronJobs

### Job

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: data-migration
spec:
  backoffLimit: 3          # Số lần retry nếu fail
  activeDeadlineSeconds: 3600  # Timeout
  ttlSecondsAfterFinished: 300  # Cleanup sau 5 phút
  template:
    spec:
      restartPolicy: OnFailure
      containers:
        - name: migration
          image: myapp:1.0.0
          command: ["migrate", "--all"]
```

### CronJob

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: daily-report
spec:
  schedule: "0 2 * * *"  # 2 AM hàng ngày
  timeZone: "Asia/Ho_Chi_Minh"
  concurrencyPolicy: Forbid  # Forbid / Allow / Replace
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 1
  startingDeadlineSeconds: 200  # Nếu miss, vẫn chạy trong 200s
  jobTemplate:
    spec:
      template:
        spec:
          restartPolicy: OnFailure
          containers:
            - name: report
              image: myapp:1.0.0
              command: ["generate-report"]
              env:
                - name: REPORT_DATE
                  valueFrom:
                    fieldRef:
                      fieldPath: metadata.labels['date']
```

**Schedule format:**
```
# ┌───────────── minute (0-59)
# │ ┌───────────── hour (0-23)
# │ │ ┌───────────── day of month (1-31)
# │ │ │ ┌───────────── month (1-12)
# │ │ │ │ ┌───────────── day of week (0-6)
# │ │ │ │ │
  * * * * *
```

| Pattern | Ý nghĩa |
|---------|---------|
| `0 * * * *` | Mỗi giờ |
| `0 9 * * 1-5` | 9 AM, Thứ 2 - Thứ 6 |
| `*/15 * * * *` | Mỗi 15 phút |
| `0 0 1 * *` | Ngày 1 mỗi tháng |

## Multi-Container Pod Patterns

### Sidecar

```yaml
# Sidecar: helper container cùng Pod
spec:
  containers:
    - name: app
      image: myapp:v1

        - name: shared-logs

    - name: shared-logs
```

```yaml
# Init: Chạy trước main container
spec:
  initContainers:
    - name: wait-for-db
      image: busybox:1.36
      command:
        - sh
        - -c
        - |
          echo "Waiting for database..."
          until nc -z db-service 5432; do
            sleep 2
          done
          echo "Database is ready!"

    - name: migrations
      image: myapp-migrations:v1
      command: ["migrate"]

  containers:
    - name: app
      image: myapp:v1
```

## K8s for Go Developer

### Go app Kubernetes-ready checklist

```go
// 1. Health endpoints (required for probes)
http.HandleFunc("/healthz", func(w http.ResponseWriter, r *http.Request) {
    // ✅ Liveness: app đang chạy
    w.WriteHeader(http.StatusOK)
})

http.HandleFunc("/ready", func(w http.ResponseWriter, r *http.Request) {
    // ✅ Readiness: có thể nhận traffic (VD: DB connected)
    if !db.IsConnected() {
        http.Error(w, "DB not connected", http.StatusServiceUnavailable)
        return
    }
    w.WriteHeader(http.StatusOK)
})

// 2. Graceful shutdown (required for rolling update)
srv := &http.Server{Addr: ":8080"}
go srv.ListenAndServe()

sigCh := make(chan os.Signal, 1)
signal.Notify(sigCh, syscall.SIGTERM, syscall.SIGINT)
<-sigCh

// ✅ Grace period cho K8s drain
ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
defer cancel()
srv.Shutdown(ctx)
```

### Go Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
```

## Architecture Deep Dive

### Control Plane Components

```
┌────────────────────────────────────────────────────┐
│                 Control Plane                        │
│                                                     │
│  ┌────────────┐   ┌────────────┐                   │
│  │ API Server │◄──│   etcd     │  (persistent store)│
│  │ (gateway)  │   └────────────┘                   │
│  └──────┬─────┘                                    │
│         │                                          │
│    ┌────┴────┬────────────┐                        │
│    ▼         ▼            ▼                        │
│ ┌──────┐ ┌────────┐ ┌───────────┐                  │
│ │Sched │ │ CCM    │ │Controller │                  │
│ │      │ │(cloud) │ │ Manager   │                  │
│ └──────┘ └────────┘ └───────────┘                  │
└────────────────────────────────────────────────────┘
```

| Component | Chức năng |
|-----------|-----------|
| **API Server** | REST API gateway, authenticate, validate, persist to etcd |
| **etcd** | Distributed key-value store, holds all cluster state |
| **Scheduler** | Assign Pods to Nodes dựa trên resources, affinity, taints |
| **Controller Manager** | Run controller loops: ReplicaSet, Deployment, Namespace, etc. |
| **Cloud Controller Manager** | Cloud-specific: LB, node, route controllers |

## Go-to-Production Checklist

| Item | Description |
|------|-------------|
| ✅ Resource requests/limits | CPU, memory bounded |
| ✅ Liveness/Readiness probes | Không restart liên tục |
| ✅ Graceful shutdown | SIGTERM handler, 30s grace period |
| ✅ Anti-affinity | Không để all replicas cùng node |
| ✅ HPA | Scale theo CPU/memory |
| ✅ PodDisruptionBudget | Bảo vệ min replicas khi disruptions |
| ✅ Topology spread | Spread across zones |
| ✅ NetworkPolicy | Restrict traffic flow |
| ✅ SecurityContext | Non-root, read-only root filesystem |
| ✅ Secret encryption | etcd encryption at rest |
| ✅ ImagePullPolicy | `Always` for prod |

## 📌 Tóm tắt

```
Kubernetes Architecture
├── Control Plane
│   ├── API Server ─────── Gateway, authentication
│   ├── etcd ───────────── State store
│   ├── Scheduler ──────── Pod → Node assignment
│   └── Controller Manager ─── ReplicaSet, Deployment, etc.
│
├── Node
│   ├── Kubelet ────────── Pod lifecycle, health
│   ├── Kube-Proxy ──────── Service networking
│   └── Container Runtime ── containerd, runc
│
├── Workload Objects
│   ├── Pod ─────────────── Smallest unit (containers share net)
│   ├── ReplicaSet ──────── Ensure N replicas
│   ├── Deployment ──────── Rolling update, rollback
│   ├── StatefulSet ─────── Ordered deploy, stable identity
│   ├── DaemonSet ───────── 1 pod per node (logging, monitoring)
│   ├── Job ─────────────── One-time task
│   └── CronJob ─────────── Scheduled task
│
├── Networking
│   ├── Service ─────────── ClusterIP / NodePort / LoadBalancer
│   ├── Ingress ─────────── HTTP routing
│   └── NetworkPolicy ───── Firewall rules
│
├── Config & Secrets
│   ├── ConfigMap ───────── Non-sensitive config
│   └── Secret ──────────── Sensitive data (base64, unencrypted!)
│
└── Storage
    ├── PersistentVolume ──── Raw storage
    ├── PersistentVolumeClaim ── Storage request
    └── StorageClass ─────── Dynamic provision
```
