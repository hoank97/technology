## Concurrency — tổ chức để **xử lý nhiều task xen kẽ** nhau:

```
Task A ──────
            │
Task B ───  │
      │─────┘
```

- Task A bắt đầu, dừng → Task B chạy → A tiếp tục
- **Không cần nhiều CPU**
- **Cấu trúc code** (làm thế nào)

---

## Parallelism

**Thực thi vật lý** — chạy **đồng thời** trên nhiều CPU core:

```
Task A ──────────→  CPU 1
Task B ──────────→  CPU 2
```

- 2 task chạy **cùng lúc**
- **Cần nhiều CPU**
- **Hardware** (làm cái gì)

---

## So sánh

| | **Concurrency** | **Parallelism** |
|---|---|---|
| **Mục đích** | Quản lý nhiều task cùng lúc | Chạy nhiều task cùng lúc |
| **Yêu cầu** | 1 CPU cũng được | Cần nhiều CPU |
| **Trọng tâm** | Cấu trúc code | Hardware |
| **Ví dụ** | Goroutine, event loop | Multi-core processing |
| **Go** | `go func()` | `GOMAXPROCS > 1` |

---

## Ví dụ Go

### Concurrency (1 CPU)

```go
runtime.GOMAXPROCS(1) // 1 CPU

go func() { doA() }()
go func() { doB() }()
// A và B xen kẽ trên cùng CPU
```

### Parallelism (nhiều CPU)

```go
runtime.GOMAXPROCS(4) // 4 CPU

go func() { doA() }() // CPU 1
go func() { doB() }() // CPU 2
go func() { doC() }() // CPU 3
go func() { doD() }() // CPU 4
// A, B, C, D chạy cùng lúc
```

---

## Khi nào cần gì?

| Scenario | Cần |
|----------|-----|
| I/O-bound (network, disk) | **Concurrency** |
| CPU-bound (computation) | **Parallelism** |
| Combination | Cả hai |
