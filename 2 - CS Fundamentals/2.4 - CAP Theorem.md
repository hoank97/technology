## 3 Properties

### C — Consistency

> Tất cả nodes thấy **cùng data cùng lúc**

- Sau khi write → tất cả reads trả về data mới

### A — Availability

> Mọi request đều được **respond**

- Response không quan trọng đúng/sai

### P — Partition Tolerance

> System hoạt động **khi network partition xảy ra**

- Network partition = mạng bị chia cắt

---

## Chỉ chọn được 2/3

| System | C | A | P | Ví dụ |
|--------|---|---|---|-------|
| **CP** | ✅ | ❌ | ✅ | MongoDB, HBase, Zookeeper |
| **AP** | ❌ | ✅ | ✅ | Cassandra, DynamoDB |
| **CA** | ✅ | ✅ | ❌ | Single-node RDBMS |

> ⚠️ **P là bắt buộc** (network luôn có thể partition)
> → Chỉ có **CP vs AP**

---

## PACELC Model

Kể cả **không có partition**, vẫn có trade-off:

| | Latency | Consistency |
|---|---|---|
| **AP** | Low Latency (EL) | Eventual (EC) |
| **CP** | Higher Latency | Strong |

| Database | PACELC |
|----------|--------|
| DynamoDB (write) | PA/EL |
| DynamoDB (read) | PA/EC |
| MongoDB | PC/EC |

## 📌 Tóm tắt

```
CAP Theorem
│
├── Chỉ chọn 2/3
│   ├── CP: Consistency > Availability
│   └── AP: Availability > Consistency
│
├── P là bắt buộc (network partition xảy ra)
│
└── PACELC: Latency vs Consistency khi bình thường
```
