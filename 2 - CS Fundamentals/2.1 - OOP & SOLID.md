## SOLID Principles

### S — Single Responsibility

**A class should have only one reason to change.**

```go
// ❌ Bad: Nhiều trách nhiệm
type User struct {
    Save()      // DB
    Validate()  // Validation
    SendEmail() // Email
}

// ✅ Good: Tách thành các class riêng
type User struct{}
type UserRepo struct{}  // Chỉ DB
type EmailService struct{} // Chỉ email
```

### O — Open/Closed

**Open for extension, closed for modification.**

```go
// ❌ Bad: Thêm feature = sửa existing code
if p.Type == "credit" { /* ... */ }
if p.Type == "paypal" { /* ... */ }

// ✅ Good: Thêm type = thêm struct mới
type PaymentProcessor interface { Process(amount float64) error }
```

### L — Liskov Substitution

**Subtypes must be substitutable for their base types.**

```go
// ❌ Bad: Penguin không bay được!
type Penguin struct{}
func (Penguin) Fly() { panic("cannot fly") }

// ✅ Good: Tách interface
type FlyingBird interface { Fly() }
```

### I — Interface Segregation

**Clients should not be forced to depend on interfaces they do not use.**

```go
// ❌ Bad: Fat interface
type Machine interface {
    Print(); Scan(); Fax();
}

// ✅ Good: Small, focused interfaces
type Printer interface { Print() }
type Scanner interface { Scan() }
```

### D — Dependency Inversion

**Depend on abstractions, not on concrete implementations.**

```go
// ❌ Bad: Depend on concrete
func Save(store *MySQLStore, data string) {
    store.Save(data)
}

// ✅ Good: Depend on interface
type Storage interface { Save(data string) }
func Save(store Storage, data string) {
    store.Save(data)
}
```

## 📌 Tóm tắt

```
OOP & SOLID
│
├── 4 Pillars
│   ├── Encapsulation → private fields + public methods
│   ├── Abstraction   → interface hides complexity
│   ├── Inheritance   → Go: struct embedding
│   └── Polymorphism → interface + different implementations
│
└── SOLID
    ├── S: Single Responsibility  → 1 class = 1 reason
    ├── O: Open/Closed           → Extend, don't modify
    ├── L: Liskov Substitution   → Subclass replaceable
    ├── I: Interface Segregation  → Small > Big interface
    └── D: Dependency Inversion   → Depend on abstraction
```
