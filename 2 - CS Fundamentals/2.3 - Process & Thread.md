# 2.3 Process & Thread

## Process

**A running instance of a program:**

- Separate memory space
- Independent execution environment
- Giao tiếp qua IPC

```
┌──────────────────────┐
│      Process A       │
│  Memory Space A      │
│  ├── Stack A         │
│  ├── Heap A          │
│  └── Code A          │
└──────────────────────┘
```

---

## Thread

**Đơn vị thực thi trong process:**

- Share memory với các thread khác trong cùng process
- Nhẹ hơn process

```
┌──────────────────────┐
│      Process         │
│  ┌────┐ ┌────┐      │
│  │ Th1│ │ Th2│      │
│  └──┬─┘ └──┬─┘      │
│     │      │         │
│  ┌──┴──────┴──┐     │
│  │    Heap     │     │
│  │    Code     │     │
│  └────────────┘     │
└──────────────────────┘
```

---

## So sánh

| | **Process** | **Thread** |
|---|---|---|
| Isolation | Separate memory | Shared memory |
| Creation | Chậm (~ms) | Nhanh (~μs) |
| Communication | IPC | Direct memory access |
| Overhead | Cao | Thấp |
| Crash isolation | ✅ Có | ❌ Không |
| Example | Browser tab | Async task in tab |

---

## Core and Thread in CPU

- **Core**: Đơn vị xử lý vật lý trong CPU
- **Thread**: Đơn vị thực thi logic
- 1 Core có thể chạy nhiều threads (hyperthreading)

```
CPU
├── Core 1
│   ├── Thread A
│   └── Thread B (hyperthreading)
└── Core 2
    └── Thread C
```

---

## 📌 Tóm tắt

```
Process vs Thread
│
├── Process
│   ├── Separate memory, slow create
│   └── Crash isolated
│
└── Thread
    ├── Shared memory, fast create
    └── Crash affects process
```
