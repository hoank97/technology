# 05.1 Generics

## Overview

Generics (type parameters) được thêm vào Go từ **Go 1.18** — cho phép viết code reusable mà không cần `interface{}` boxing.

> Thường hỏi trong interview: "Go có generics không? So sánh với Java/C++ generics?"

---
## Syntax: Type Parameters

### Basic syntax

```go
// ✅ Function với type parameter T
func Max[T any](a, b T) T {
    if a > b {
        return a
    }
    return b
}

// Gọi: Max[int](3, 5) hoặc Max(3, 5) (compiler infers)
fmt.Println(Max(3, 5))           // int
fmt.Println(Max(3.14, 2.71))     // float64
fmt.Println(Max("a", "b"))       // string
```

### Inference

```go
// Compiler tự suy ra type → không cần chỉ định
Max(1, 2)        // T = int
Max(1.0, 2.0)    // T = float64
```

## Type Constraints

### `any` — không giới hạn

```go
func Print[T any](val T) {
    fmt.Println(val)
}
```

### `comparable` — phải so sánh được (==, !=)

```go
func Contains[T comparable](slice []T, target T) bool {
    for _, v := range slice {
        if v == target {
            return true
        }
    }
    return false
}

fmt.Println(Contains([]int{1, 2, 3}, 2))      // true
fmt.Println(Contains([]string{"a", "b"}, "c")) // false
```

### ⚠️ Không có `ordered` constraint

Go **không có built-in constraint cho >, <**. Bạn phải tự định nghĩa:

```go
// ❌ KHÔNG compile — "int" không comparable với constraint
func Min[T any](a, b T) T {
    if a < b { // error: invalid operation: a < b (T not comparable)
        return a
    }
    return b
}

// ✅ Dùng generic function với custom constraint
type Ordered interface {
    int | int8 | int16 | int32 | int64 |
    uint | uint8 | uint16 | uint32 | uint64 |
    float32 | float64
}

func Min[T Ordered](a, b T) T {
    if a < b {
        return a
    }
    return b
}
```

## Generic Types (Struct)

```go
// ✅ Generic struct
type Pair[K, V any] struct {
    Key   K
    Value V
}

p := Pair[string, int]{Key: "age", Value: 25}
fmt.Println(p.Key, p.Value)

// ✅ Generic method
type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(v T) {
    s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
    if len(s.items) == 0 {
        var zero T
        return zero, false
    }
    v := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return v, true
}

stack := Stack[int]{}
stack.Push(1)
stack.Push(2)
fmt.Println(stack.Pop()) // 2 true
```

## Generic Methods (trong generic struct)

```go
type Box[T any] struct {
    value T
}

func (b *Box[T]) Get() T    { return b.value }
func (b *Box[T]) Set(v T)   { b.value = v }

// ✅ Methods có thể dùng type param khác (Go 1.18+)
func (b *Box[T]) Map[U any](f func(T) U) Box[U] {
    return Box[U]{value: f(b.value)}
}

box := Box[int]{value: 10}
doubled := box.Map(func(v int) int { return v * 2 })
fmt.Println(doubled.Get()) // 20
```

## Custom Type Constraints

```go
// ✅ Intersection constraint
type Number interface {
    int | int64 | float64
}

func Sum[T Number](vals []T) T {
    var total T
    for _, v := range vals {
        total += v
    }
    return total
}

fmt.Println(Sum([]int{1, 2, 3}))          // 6
fmt.Println(Sum([]float64{1.1, 2.2}))   // 3.3
```

## Unique Function (áp dụng constraint)

```go
func Unique[T comparable](vals []T) []T {
    seen := make(map[T]struct{})
    result := make([]T, 0, len(vals))
    for _, v := range vals {
        if _, ok := seen[v]; !ok {
            seen[v] = struct{}{}
            result = append(result, v)
        }
    }
    return result
}

names := Unique([]string{"a", "b", "a", "c", "b"})
fmt.Println(names) // [a b c]
```

## Generics vs `interface{}`

Interface vs Generics comparison — liên quan đến [[01.1 - Data Types & Basics#Interface Internals (iface vs eface)]].

### Performance

```go
// ✅ Generics — NO boxing, NO runtime allocation
func Max[T comparable](a, b T) T { ... }
m := Max(1, 2) // T = int, compiled to direct int comparison

// ❌ interface{} — boxing, runtime allocation
func MaxInterface(a, b interface{}) interface{} { ... }
m := MaxInterface(1, 2) // boxing int → interface{}, heap allocation
```

### Benchmark so sánh

```
Generics:    10 ns/op, 0 allocs/op
interface{}: 50 ns/op, 2 allocs/op
```

### Khi nào dùng?

| | Generics | `interface{}` |
|---|---|---|
| Static type safety | ✅ | ❌ |
| Performance (no boxing) | ✅ | ❌ |
| Khi type khác nhau cần cùng logic | ✅ | ✅ (nhưng không type-safe) |
| Reflection / dynamic type | ❌ | ✅ |
| Khi chỉ cần 2-3 types cụ thể | ⚠️ Overkill | ✅ |

## Common Idiom: `slices` / `maps` / `cmp` (Go 1.21+)

Xem thêm [[05.2 - Generic Algorithms (std)]] để biết chi tiết standard library implementations.

### `slices`

```go
import "slices"

names := []string{"zhao", "alice", "bob"}

// Sort
slices.Sort(names) // ["alice", "bob", "zhao"]

// Custom sort
slices.SortFunc(names, func(a, b string) int {
    return strings.Compare(a, b)
})

// Contains
found := slices.Contains(names, "alice") // true

// Binary search (sorted)
idx := slices.BinarySearch(names, "bob") // 1, nil

// Delete (in-place)
slices.Delete(&names, 0, 1) // remove index 0

// Concat
merged := slices.Concat(names, []string{"charlie", "dave"})
```

### `maps`

```go
import "maps"

m1 := map[string]int{"a": 1, "b": 2}
m2 := map[string]int{"b": 3, "c": 4}

// Equal
maps.Equal(m1, m2) // false

// Clone
m3 := maps.Clone(m1)

// Copy
m4 := maps.Clone(m1)
maps.Copy(m4, m2) // m4: {"a": 1, "b": 3, "c": 4}

// DeleteFunc
maps.DeleteFunc(m1, func(k string, v int) bool {
    return v < 2
}) // m1: {"b": 2}
```

### `cmp`

```go
import "cmp"

type User struct {
    Name string
    Age  int
}

// Compare (Go 1.21+)
cmp.Compare(1, 2)   // -1
cmp.Compare(2, 1)   // 1
cmp.Compare(2, 2)   // 0

// Or — return first non-zero value (Go 1.22+)
cmp.Or(0, 42)        // 42
cmp.Or("", "default") // "default"
```

## When NOT to Use Generics

```go
// ❌ OVER-ENGINEERED — generics không cần thiết
type IntPair struct { A int; B int }

func (p IntPair) Sum() int { return p.A + p.B }
// → Đừng generic hóa khi chỉ có 1-2 types cụ thể
```

**Dùng khi**:
- ✅ Code được dùng bởi nhiều types (algorithms, data structures)
- ✅ Type safety quan trọng (không muốn `interface{}` casting)
- ✅ Performance (tránh boxing)

**Không dùng khi**:
- ❌ Chỉ có 1-2 types cụ thể — just write specific functions
- ❌ Cần reflection hoặc dynamic type handling
- ❌ Premature abstraction — "maybe later I'll need it"

## Generic with `interface{}` fallback

```go
// ✅ Khi cần cả generics + runtime flexibility
func Process[T any](val T) {
    if v, ok := any(val).(fmt.Stringer); ok {
        fmt.Println("Stringer:", v.String())
    } else {
        fmt.Println("Default:", val)
    }
}
```

## Generics Internals — How the Compiler Handles Them

GCShape stenciling liên quan đến memory layout và allocation — xem [[02.1 - Memory & Runtime#Memory Alignment & Padding]].

Go compiler triển khai generics theo **GCShape stenciling** — không fully monomorphize như C++, không fully erase như Java:

```
GCShape stenciling:
  - Tất cả pointer types dùng CÙNG 1 code (shared)
  - Primitive types (int, float64, ...) được stencil riêng
  - → Balance: ít code bloat, giữ được type safety

C++ approach (Full monomorphization):
  List<int> và List<string> → 2 separate compiled classes
  → Code bloat, nhưng maximum performance

Java approach (Type erasure):
  List<int> và List<string> → same List<Object> at runtime
  → No code bloat, nhưng boxing overhead + no runtime type info
```

## Type Inference — Khi nào Go tự suy ra type?

```go
func Map[T, U any](s []T, f func(T) U) []U {
    result := make([]U, len(s))
    for i, v := range s {
        result[i] = f(v)
    }
    return result
}

// ✅ Go inference — type parameters inferred from arguments
doubled := Map([]int{1, 2, 3}, func(n int) int { return n * 2 })
//  ↑ Compiler infers T=int, U=int from args

// ✅ Partial inference
words := Map([]int{1, 2}, func(n int) string { return fmt.Sprint(n) })
// T=int inferred, U=string inferred from return type of func

// ❌ Inference fails — must explicit
type Result[T any] struct{ Value T }
r := Result[int]{Value: 42}  // cannot infer T from struct literal
```

## Constraints — Advanced Patterns

```go
// Union type constraint
type Number interface {
    int | int8 | int16 | int32 | int64 |
    float32 | float64
}

// Underlying type constraint (~)
type Ordered interface {
    ~int | ~float64 | ~string  // ~ = includes types with these underlying types
}

type MyInt int
func sum[T Ordered](a, b T) T { return a + b }
sum(MyInt(1), MyInt(2))  // ✅ MyInt underlying type is int

// Combining constraints
type Printable interface {
    comparable  // supports == and !=
    fmt.Stringer // has String() string
}

func printUnique[T Printable](items []T) {
    seen := make(map[T]bool)
    for _, item := range items {
        if !seen[item] {
            fmt.Println(item.String())
            seen[item] = true
        }
    }
}
```

## Khi nào NÊN và KHÔNG NÊN dùng Generics

### ✅ NÊN — Generic algorithms (tốt nhất)

```go
// Functions that work on any slice of T — classic use case
func Filter[T any](s []T, pred func(T) bool) []T {
    var result []T
    for _, v := range s {
        if pred(v) { result = append(result, v) }
    }
    return result
}

// Generic data structures
type Set[T comparable] struct{ m map[T]struct{} }
func (s *Set[T]) Add(v T) { s.m[v] = struct{}{} }
func (s *Set[T]) Has(v T) bool { _, ok := s.m[v]; return ok }

// Container types: Stack, Queue, Pair, Result
type Result[T any] struct {
    Value T
    Err   error
}
```

### ❌ KHÔNG NÊN — Các trường hợp interface vẫn tốt hơn

```go
// ❌ Generic không giúp gì nếu behavior phụ thuộc type
// Dùng interface thay
func Process[T any](items []T) {
    for _, item := range items {
        // Muốn gọi item.Validate() → interface phù hợp hơn
    }
}

// ✅ Better: use interface
type Validator interface { Validate() error }
func Process(items []Validator) { ... }

// ❌ Generic struct với nhiều behavior phụ thuộc type
// → Code phức tạp, khó đọc

// ❌ Khi có thể dùng interface{}/any và runtime dispatch OK
// (ví dụ: JSON encoding — reflection-based là đủ)
```

## Generics vs io Alternatives — Trade-off Analysis

| | `interface{}` / `any` | `reflect` | Generics `[T any]` |
|---|---|---|---|
| Type safety | ❌ Runtime | ❌ Runtime | ✅ Compile-time |
| Performance | ⚠️ Boxing overhead | ❌ Very slow | ✅ Near-zero overhead |
| Code complexity | ✅ Simple | ❌ Complex | ⚠️ Moderate |
| Flexibility | ✅ Full | ✅ Full | ⚠️ Constrained by type set |
| Error messages | ❌ Runtime panic | ❌ Runtime panic | ✅ Compile-time error |
| Use case | Plugin systems, JSON | Meta-programming | Algorithms, containers |

## Generic Functions — Design Patterns

```go
// Pattern 1: Functional (slices.Map, Filter, Reduce)
func Reduce[T, Acc any](s []T, init Acc, fn func(Acc, T) Acc) Acc {
    result := init
    for _, v := range s {
        result = fn(result, v)
    }
    return result
}

total := Reduce([]int{1, 2, 3, 4}, 0, func(acc, n int) int { return acc + n })
// 10

// Pattern 2: Result type (functional error handling)
type Result[T any] struct{ value T; err error }

func Try[T any](v T, err error) Result[T] {
    return Result[T]{value: v, err: err}
}

func (r Result[T]) Must() T {
    if r.err != nil { panic(r.err) }
    return r.value
}

// Pattern 3: Generic cache
type Cache[K comparable, V any] struct {
    mu    sync.RWMutex
    items map[K]V
}

func (c *Cache[K, V]) Get(key K) (V, bool) {
    c.mu.RLock()
    defer c.mu.RUnlock()
    v, ok := c.items[key]
    return v, ok
}
```
