# 07.1 Testing & Benchmarking

## Overview

Go có `testing` package tích hợp sẵn — không cần library bên thứ 3 cho hầu hết use cases.

> Testing là first-class concern trong Go. Interview thường hỏi table tests, benchmarks, và mocking patterns.

---

## Testing.T — Basics

### Function signature

```go
// File: sum_test.go
func TestSum(t *testing.T) {
    // test logic
}
```

### Các methods cơ bản

| Method | Use case |
|--------|----------|
| `t.Fatal(...)` | Fail immediately — test không continue |
| `t.Fatalf("msg %d", val)` | Fail with formatted message |
| `t.Error(...)` | Fail but continue execution |
| `t.Errorf("msg %d", val)` | Fail with formatted, continue |
| `t.Skip("reason")` | Skip test (feature not implemented) |
| `t.SkipNow()` | Skip rest of test |

### Assertions pattern

```go
func AssertEqual[T any](t *testing.T, got, want T) {
    if got != want {
        t.Errorf("got %v, want %v", got, want)
    }
}

// Dùng
AssertEqual(t, Add(1, 2), 3)
```

## Table-Driven Tests

### Pattern chuẩn

```go
func TestAdd(t *testing.T) {
    tests := []struct {
        name string
        a, b int
        want int
    }{
        {"zero", 0, 0, 0},
        {"positive", 1, 2, 3},
        {"negative", -1, -1, -2},
        {"large", 1_000_000, 2_000_000, 3_000_000},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got := Add(tt.a, tt.b)
            if got != tt.want {
                t.Errorf("Add(%d, %d) = %d; want %d", tt.a, tt.b, got, tt.want)
            }
        })
    }
}
```

**Output mẫu**:
```
=== RUN   TestAdd/zero
--- PASS: TestAdd/zero (0.00s)
=== RUN   TestAdd/positive
--- PASS: TestAdd/positive (0.00s)
=== RUN   TestAdd/negative
--- PASS: TestAdd/negative (0.00s)
=== RUN   TestAdd/large
--- PASS: TestAdd/large (0.00s)
--- PASS: TestAdd (4 subtests)
```

### Tại sao table tests?

| | Individual tests | Table tests |
|---|---|---|
| Thêm test case | Thêm function mới | Thêm row vào slice |
| Shared setup | Copy-paste | Dùng chung setup trong loop |
| Consistency | Tùy developer | Uniform pattern |
| Scalability | ❌ | ✅ |

## Subtests với `t.Run()`

```go
func TestDatabase(t *testing.T) {
    // Setup chung
    db := setupDB(t) // helper with cleanup

    t.Run("insert", func(t *testing.T) {
        err := db.Insert("key", "value")
        AssertNoError(t, err)
    })

    t.Run("query", func(t *testing.T) {
        val, err := db.Query("key")
        AssertEqual(t, val, "value")
    })

    t.Run("delete", func(t *testing.T) {
        err := db.Delete("key")
        AssertNoError(t, err)
    })
}
```

### Cleanup sau subtest

```go
func TestFile(t *testing.T) {
    // per subtest cleanup
    t.Run("write", func(t *testing.T) {
        f := createTempFile(t, "test.txt")
        defer os.Remove(f.Name())
        // test
    })
}
```

## HTTP Testing với `httptest`

### Test HTTP Handler

```go
func TestHandler(t *testing.T) {
    // Tạo HTTP request giả
    req := httptest.NewRequest(http.MethodGet, "/hello", nil)
    // Tạo ResponseRecorder để capture response
    rec := httptest.NewRecorder()

    // Gọi handler
    Handler(rec, req)

    // Assertions
    AssertEqual(t, rec.Code, http.StatusOK)
    AssertEqual(t, rec.Body.String(), "Hello!")
}
```

### Test với body và headers

```go
func TestPostJSON(t *testing.T) {
    body := strings.NewReader(`{"name": "Alice"}`)
    req := httptest.NewRequest(http.MethodPost, "/users", body)
    req.Header.Set("Content-Type", "application/json")

    rec := httptest.NewRecorder()
    CreateUser(rec, req)

    AssertEqual(t, rec.Code, http.StatusCreated)
}
```

## Testing Errors

```go
func TestErrors(t *testing.T) {
    t.Run("not found", func(t *testing.T) {
        err := db.Query("nonexistent")
        AssertEqual(t, err, ErrNotFound)
    })

    t.Run("invalid input", func(t *testing.T) {
        err := validate("")
        AssertError(t, err, "empty input")
    })
}

// Helper
func AssertError(t *testing.T, err error, substr string) {
    if err == nil {
        t.Fatal("expected error, got nil")
    }
    if !strings.Contains(err.Error(), substr) {
        t.Errorf("error %q does not contain %q", err.Error(), substr)
    }
}
```

## Benchmarking với `testing.B`

### Basic benchmark

```go
func BenchmarkFibonacci(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Fibonacci(20)
    }
}
```

### Chạy benchmarks

```bash
go test -bench=. -benchmem ./...
go test -bench=BenchmarkFibonacci -benchmem ./...
go test -bench=BenchmarkFibonacci -bench=10 -count=5 ./...
```

### Output mẫu

```
BenchmarkFibonacci-8    100000000    12.3 ns/op    0 B/op    0 allocs/op
```

| Column | Ý nghĩa |
|--------|---------|
| `100000000` | Test chạy `b.N` lần — tự động scale |
| `12.3 ns/op` | Trung bình mỗi operation |
| `0 B/op` | Memory allocate mỗi operation |
| `0 allocs/op` | Số allocations mỗi operation |

### Báo cáo allocations

```go
func BenchmarkLargeAlloc(b *testing.B) {
    b.ReportAllocs()
    for i := 0; i < b.N; i++ {
        _ = make([]byte, 1024)
    }
}
```

### So sánh benchmarks

```bash
go test -bench=. -benchmem -cpuprofile=cpu.out ./...
go test -bench=. -benchmem -memprofile=mem.out ./...

# Visualize
go tool pprof -http=:9090 cpu.out
```

### Realistic iteration

```go
// ❌ Sai — benchmarks quá nhanh
func BadBenchmark(b *testing.B) {
    for i := 0; i < 1000; i++ {
        Fibonacci(20) // benchmark mất <1ms
    }
}

// ✅ Đúng — để framework quyết định số iterations
func GoodBenchmark(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Fibonacci(20) // b.N tự động scale
    }
}
```

## TestMain — Custom Test Orchestration

```go
func TestMain(m *testing.M) {
    // Setup global (DB connection, mock server, etc.)
    setup()
    code := m.Run()
    // Teardown
    teardown()
    os.Exit(code)
}
```

### Hook setup cho tất cả tests

```go
var testDB *sql.DB

func TestMain(m *testing.M) {
    var err error
    testDB, err = sql.Open("postgres", os.Getenv("DATABASE_URL"))
    if err != nil {
        log.Fatal(err)
    }
    defer testDB.Close()

    os.Exit(m.Run())
}

// Dùng testDB trong mọi test
func TestUserQuery(t *testing.T) {
    rows, err := testDB.Query("SELECT * FROM users")
    // ...
}
```

## Mocking Patterns

### Interface-based mocking

```go
// Interface defines contract
type UserStore interface {
    GetUser(id string) (*User, error)
    SaveUser(u *User) error
}

// Real implementation
type PostgresStore struct{ db *sql.DB }

func (s *PostgresStore) GetUser(id string) (*User, error) {
    // ...
}

// Mock implementation
type MockStore struct {
    users map[string]*User
}

func (m *MockStore) GetUser(id string) (*User, error) {
    return m.users[id], nil
}
func (m *MockStore) SaveUser(u *User) error {
    m.users[u.ID] = u
    return nil
}

// Trong test
func TestService(t *testing.T) {
    store := &MockStore{users: map[string]*User{
        "1": {ID: "1", Name: "Alice"},
    }}
    svc := NewService(store)
    // ...
}
```
