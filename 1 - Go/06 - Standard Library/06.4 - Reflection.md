# 06.4 Reflection

## Overview

**Reflection** = khả năng của chương trình **inspect và modify** cấu trúc, giá trị, kiểu tại runtime. Go's reflection nằm trong `reflect` package.

> Reflection is a powerful concept that violates some of the Go philosophy — use it sparingly, and only when there's no cleaner alternative.

```go
import "reflect"
```

### When to use reflection

| Use case | Why reflection helps |
|---------|----------------------|
| JSON marshal/unmarshal | Không biết struct shape tại compile time |
| ORM field mapping | Map database columns → struct fields bằng tên |
| Validation frameworks | Inspect struct tags để validate |
| Generic printing/formatting | `fmt.Printf("%v")` dùng reflection internals |
| Plugin systems | Load implementation từ tên type |
| Testing mocks | Generate mock objects from interface |

### When NOT to use

```go
// ❌ Over-engineering: type không đổi → không cần reflection
func printName(v any) string {
    t := reflect.TypeOf(v)          // ❌ reflect overhead không cần thiết
    if t.Kind() == reflect.String {
        return v.(string)
    }
    return ""                        // ✅ Just use type assertion
}

// ✅ Nếu type cố định tại compile time → type assertion luôn tốt hơn
func printName(s string) string { return s }
```

## reflect.Type vs reflect.Value

Đây là hai interface cốt lõi của reflection:

```go
reflect.Type  // → "cái gì" (type name, fields, methods)
reflect.Value // → "giá trị gì" (actual data, có thể modify)
```

```go
v := "hello"

t := reflect.TypeOf(v)  // reflect.Type: string
val := reflect.ValueOf(v) // reflect.Value: "hello"

// TypeOf(nil) → nil Type
// ValueOf(nil) → zero Value
```

### Type information

```go
type User struct {
    Name string `json:"name" validate:"required"`
    Age  int    `json:"age" validate:"min=0"`
   Addr string // no tags
}

t := reflect.TypeOf(User{})

// Basic info
t.Name()    // "User"
t.Kind()    // reflect.Struct
t.Size()    // size in bytes (e.g., 48)
t.NumField() // 3

// Field access
for i := 0; i < t.NumField(); i++ {
    f := t.Field(i)
    fmt.Println(f.Name, f.Type, f.Tag.Get("json"), f.Tag.Get("validate"))
    // Name  string  json:"name"  validate:"required"
    // Age   int     json:"age"   validate:"min=0"
    // Addr  string <invalid>     -
}
```

### Value information

```go
val := reflect.ValueOf(User{Name: "Alice", Age: 30})

val.Kind()     // reflect.Struct
val.Type()     // same as reflect.TypeOf(User{}) — same Type

val.FieldByName("Name")          // reflect.Value of "Alice"
val.FieldByIndex([]int{1})        // reflect.Value of Age field (index path)
val.FieldByName("NonExistent")   // zero Value

val.CanSet() // false — ValueOf tạo non-addressable value
             // ⚠️ Chỉ có thể Set() nếu value là addressable (passed as pointer)
```

## Kind vs Type — hai lớp identity

```go
val := int64(42)

t := reflect.TypeOf(val)  // int64
k := reflect.KindOf(val)   // reflect.Int64 (underlying kind)

// Type = what you declare
t.Kind()                  // reflect.Int64
t.Name()                  // "int64"

// Kind = low-level category (structure of the type)
k == reflect.Int64        // true
k == reflect.Int          // false — different!
```

### Kind table

| Kind | Go type |
|------|---------|
| `reflect.Bool` | `bool` |
| `reflect.Int/Int8/Int16/Int32/Int64` | `int`, `int8`, ... |
| `reflect.Uint/Uint8/...` | `uint`, `uint8`, ... |
| `reflect.Float32/Float64` | `float32`, `float64` |
| `reflect.Complex64/Complex128` | `complex64`, `complex128` |
| `reflect.String` | `string` |
| `reflect.Slice` | `[]T` |
| `reflect.Map` | `map[K]V` |
| `reflect.Struct` | `struct` |
| `reflect.Ptr` | `*T` |
| `reflect.Array` | `[N]T` |
| `reflect.Func` | `func` |
| `reflect.Interface` | `interface` |
| `reflect.Chan` | `chan T` |
| `reflect.Bool` | `bool` |

## Setting values — Addressability

**Quy tắc quan trọng**: Chỉ có thể `Set()` khi value là **addressable** — tức là passed as pointer.

```go
// ❌ Cannot set — value is a copy, not addressable
val := reflect.ValueOf(42)
val.SetInt(100) // ❌ Panic: unaddressable value

// ✅ Can set — value came from a pointer
ptr := reflect.ValueOf(&42)
elem := ptr.Elem()        // elem là addressable
elem.SetInt(100)         // ✅ OK

// ✅ Common pattern: pass pointer
val := reflect.ValueOf(&user).Elem()
val.FieldByName("Name").SetString("Bob")
```

### Dereferencing pointer types

```go
ptr := reflect.ValueOf(&user)  // *User
ptr.Kind()                     // reflect.Ptr

if ptr.Kind() == reflect.Ptr {
    elem := ptr.Elem()         // dereference: User
    elem.SetString(...)       // work on the dereferenced value
}
```

## StructTag — reading struct field tags

```go
type User struct {
    Name    string `json:"name" validate:"required" db:"user_name"`
    Age     int    `json:"age" validate:"min=0"`
    PrivPw  string `json:"-"`              // skip in JSON
}

t := reflect.TypeOf(User{})

field := t.FieldByName("Name")
field.Tag                  // reflect.StructTag = `json:"name" validate:"required" db:"user_name"`

field.Tag.Get("json")     // "name"
field.Tag.Get("validate") // "required"
field.Tag.Get("db")       // "user_name"
field.Tag.Get("unknown")  // "" (not found)

// Lookup — đúng hơn, có thêm exists flag
jsonName, ok := field.Tag.Lookup("json")  // "name", true

// Parse tag options (e.g., "omitempty,string")
val := field.Tag.Get("json")
parts := strings.Split(val, ",")
name := parts[0]        // "name"
hasOption := func(opt string) bool {
    for _, p := range parts[1:] {
        if p == opt { return true }
    }
    return false
}
hasOption("omitempty") // false
```

### Walk struct fields recursively

```go
// Recursively flatten all fields from embedded structs
func walkFields(t reflect.Type) []reflect.StructField {
    var fields []reflect.StructField
    for i := 0; i < t.NumField(); i++ {
        f := t.Field(i)
        if f.Anonymous {
            // Embedded struct — walk its fields
            fields = append(fields, walkFields(f.Type)...)
        } else {
            fields = append(fields, f)
        }
    }
    return fields
}
```

## Interface / Convertibility

### Extract from reflect.Value → interface{}

```go
val := reflect.ValueOf("hello")
i := val.Interface()     // any = "hello"
s := i.(string)          // type assertion

// ⚠️ val.Interface() luôn trả về copy không đổi
// Để modify, phải dùng pointer + Set()
```

### Type conversion via reflection

```go
val := reflect.ValueOf(42)

// CanConvertTo — kiểm tra có convert được không
val.CanConvert(reflect.TypeOf(int64(0))) // true

// Convert — thực hiện convert
converted := val.Convert(reflect.TypeOf(int64(0)))
converted.Interface().(int64) // 42
```

### Value comparison

```go
v1 := reflect.ValueOf([]int{1, 2})
v2 := reflect.ValueOf([]int{1, 2})

v1.Interface() == v2.Interface() // false (different slices)
reflect.DeepEqual(v1.Interface(), v2.Interface()) // true ✅

// reflect.Value.Equal (Go 1.21+)
v1.Equal(v2) // true ✅
```

## Inspecting functions & methods

```go
type Greeter struct{}

func (g Greeter) Greet(name string) string { return "Hello " + name }
func (g *Greeter) Reset()                  {}

t := reflect.TypeOf(Greeter{})

// NumMethod — chỉ counting exported methods theo receiver type
t.NumMethod() // 1 (Greet) — Greeter value receiver

t2 := reflect.TypeOf(&Greeter{})
t2.NumMethod() // 2 (Greet + Reset) — *Greeter pointer receiver

// Method lookup (by index)
m := t.Method(0)
m.Name        // "Greet"
m.Type        // func(Greeter, string) string
m.Func        // reflect.Value of the method function

// Method lookup (by name)
m, ok := t.MethodByName("Greet")
// Call method via Func
args := []reflect.Value{reflect.ValueOf(g), reflect.ValueOf("Alice")}
result := m.Func.Call(args) // ["Hello Alice"]
```

## MakeFunc — runtime-defined functions

```go
// MakeFunc tạo một function tại runtime
// Function nhận []reflect.Value, trả về []reflect.Value

double := reflect.MakeFunc(func(args []reflect.Value) []reflect.Value {
    v := args[0]
    if v.Kind() == reflect.Int {
        return []reflect.Value{v.Add(v)} // v * 2
    }
    return []reflect.Value{v}
})

fn := double.Interface().(func(int) int)
fn(5)  // 10
fn(21) // 42
```

### Use case: generic validator from struct tags

```go
type Request struct {
    Name  string `validate:"required"`
    Age   int    `validate:"min=0"`
    Email string `validate:"email"`
}

func Validate(v any) error {
    rv := reflect.ValueOf(v)
    if rv.Kind() != reflect.Struct {
        return errors.New("not a struct")
    }

    t := rv.Type()
    for i := 0; i < t.NumField(); i++ {
        f := t.Field(i)
        tag := f.Tag.Get("validate")
        if tag == "" { continue }

        val := rv.Field(i)
        rules := strings.Split(tag, ",")
        for _, rule := range rules {
            if err := applyRule(rule, val); err != nil {
                return fmt.Errorf("%s: %w", f.Name, err)
            }
        }
    }
    return nil
}
```

## reflect.DeepEqual — deep comparison

```go
a := []int{1, 2, 3}
b := []int{1, 2, 3}
c := &[]int{1, 2, 3}

reflect.DeepEqual(a, b) // true
reflect.DeepEqual(a, c) // false (different types)

// ⚠️ Caveats
reflect.DeepEqual(nil, "")           // false (different types)
reflect.DeepEqual([]int(nil), []int{}) // false (nil ≠ empty slice)
reflect.DeepEqual(map[string]int{}, nil) // false
```

## Trade-off: Reflection vs Code Generation vs Generics

| Approach | Pros | Cons | Use when |
|---------|------|------|---------|
| **Reflection** | Runtime flexibility, no code generation | Slow (~50ns/access), no compile-time safety, hard to read | Frameworks (ORM, DI), plugins, testing mocks |
| **Code generation** | Fast, compile-time safe, readable | Build complexity, generated files | IDL-based (protobuf, sqlc, wire) |
| **Generics** | Type-safe at compile, fast | Only works with known types | Collections, utilities |

```go
// ❌ Reflection cho điều đã có generics → over-engineering
func Contains[T comparable](slice []T, target T) bool {
    for _, v := range slice { if v == target { return true } }
    return false
} // ✅ Generics: type-safe, fast, readable

// ✅ Reflection cho ORM-like field mapping
func MapRowToStruct(row map[string]any, v any) error {
    rv := reflect.ValueOf(v).Elem()
    t := rv.Type()
    for i := 0; i < t.NumField(); i++ {
        f := t.Field(i)
        if dbTag := f.Tag.Get("db"); dbTag != "" {
            if val, ok := row[dbTag]; ok {
                rv.Field(i).Set(reflect.ValueOf(val))
            }
        }
    }
    return nil
} // Không generics nào làm được việc này
```

## Common Patterns

### Create instance from type name (plugin pattern)

```go
func NewByName[T any](typeName string) (T, error) {
    // Tìm type trong registry
    fn, ok := registry[typeName]
    if !ok {
        return *new(T), fmt.Errorf("unknown type: %s", typeName)
    }
    return fn().(T)
}

// Ví dụ: factory pattern với reflect
var typeRegistry = map[string]func() any{}

func Register(name string, factory func() any) {
    typeRegistry[name] = factory
}

// Tạo instance mà không biết type tại compile time
func CreateInstance(name string) any {
    fn := typeRegistry[name]
    if fn == nil { return nil }
    return fn()
}
```

### Copy between structs (field-by-field mapping)

```go
func CopyFields(dst, src any) {
    dv := reflect.ValueOf(dst).Elem()
    sv := reflect.ValueOf(src).Elem()

    for i := 0; i < dv.NumField(); i++ {
        df := dv.Type().Field(i)
        sf := sv.FieldByName(df.Name)
        if sf.IsValid() && sf.Type() == df.Type {
            dv.Field(i).Set(sf)
        }
    }
}
```

## 📌 Tóm tắt

```
Reflection
├── reflect.Type vs reflect.Value
│   ├── Type → metadata (name, fields, methods)
│   ├── Value → actual data (CanSet requires addressable)
│   └── Kind vs Type — category vs declaration
├── Setting values
│   ├── Addressability rule — must pass pointer
│   ├── Elem() — dereference pointer
│   └── SetInt/SetString/Set() methods
├── StructTag — read struct field metadata
│   ├── Tag.Get(key)
│   ├── Tag.Lookup(key) — with exists flag
│   └── Recursive field walk for embedded structs
├── Interface & Convertibility
│   ├── Interface() → extract as any
│   ├── CanConvertTo / Convert
│   └── DeepEqual vs Value.Equal
├── Functions & Methods
│   ├── NumMethod / Method / MethodByName
│   └── MakeFunc — runtime-defined functions
├── Trade-offs
│   ├── Pros: runtime flexibility, framework-enabling
│   ├── Cons: ~50ns/access, no compile-time safety
│   └── vs CodeGen: ORM, testing → reflection; else → codeGen/generics
```
