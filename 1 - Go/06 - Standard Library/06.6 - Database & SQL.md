# 06.6 Database & SQL

## database/sql Overview

`database/sql` là abstract layer trên các database drivers cụ thể. Nó handle connection pooling, prepared statements, transaction management, và error wrapping.

```go
import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"   // register driver
    _ "github.com/lib/pq"               // PostgreSQL
    _ "github.com/mattn/go-sqlite3"     // SQLite
)
```

### Setup connection

```go
db, err := sql.Open("mysql", "user:password@tcp(localhost:3306)/mydb?parseTime=true")
if err != nil {
    return fmt.Errorf("open connection: %w", err)
}
defer db.Close()

// Verify connection
if err := db.Ping(); err != nil {
    return fmt.Errorf("ping: %w", err)
}
```

## Connection Pool — internals

`database/sql` có built-in connection pool. Hiểu cách nó hoạt động là critical cho production performance.

```go
// Connection pool configuration
db.SetMaxOpenConns(25)      // tối đa bao nhiêu connections đồng thời
db.SetMaxIdleConns(5)       // tối đa bao nhiêu idle connections giữ alive
db.SetConnMaxIdleTime(5 * time.Minute) // idle connection timeout (Go 1.15+)
db.SetConnMaxLifetime(30 * time.Minute) // connection max age (reconnect sau)

// ⚠️ MaxOpenConns nên match với database server's max connections
// Nếu nhiều app instances: pool_per_instance × instances ≤ server_limit

// ⚠️ Connection exhaustion pattern
conn, err := db.Conn(context.Background())
if err != nil {
    return fmt.Errorf("get connection: %w", err)
}
defer conn.Close()

// ⚠️ Luôn luôn close connection khi xong
// Nếu không, connection quay về pool nhưng không được reset
// → stale state, stale prepared statements
```

### Connection lifecycle

```
Query flow:
  app → db.Conn() → pool.acquire() [có thể block nếu pool đầy]
    → conn.QueryContext() → execute on DB
    → conn.Close() → pool.release() [reset connection, return to pool]
```

### Reset behavior

```go
// Khi connection return vào pool:
// 1. Ping để verify connection còn alive (optional với CheckConn)
// 2. Reset prepared statements (driver-dependent)
// 3. Rollback uncommitted transaction (nếu có)
// 4. Reset idle timeout clock

// ⚠️ Nếu dùng connection cho nhiều queries không liên quan nhau,
// connection có thể mang stale state:
// - Session variables (SET NAMES, timezone)
// - Temporary tables
// - Locks
```

## Null Handling — nullable columns

Database columns có thể NULL — Go's zero value không tương ứng với SQL NULL.

### Null types

```go
type User struct {
    ID        int64
    Name      string
    Email     sql.NullString   // NULL → NullString{Valid: false}
    DeletedAt sql.NullTime    // NULL → NullTime{Valid: false}
    Age       sql.NullInt64    // NULL → NullInt64{Valid: false}
    Active    sql.NullBool     // NULL → NullBool{Valid: false}
    Salary    sql.NullFloat64  // NULL → NullFloat64{Valid: false}
}

// Scan vào Null types
row := db.QueryRowContext(ctx, "SELECT email FROM users WHERE id = ?", id)
var email sql.NullString
if err := row.Scan(&email); err != nil {
    return err
}
if email.Valid {
    fmt.Println("Email:", email.String)
} else {
    fmt.Println("No email set")
}
```

### Modern approach: pointer

```go
type User struct {
    ID        int64
    Name      string
    Email     *string   // NULL → nil pointer
    DeletedAt *time.Time
    Age       *int64
}

// Scan
var email *string
if err := row.Scan(&email); err != nil { ... }
if email != nil {
    fmt.Println("Email:", *email)
}

// ⚠️ Pointer approach: đơn giản hơn Null types
// ⚠️ Nhưng cần chú ý: nil pointer → SQL NULL, nhưng ""
// vẫn là "" (empty string), không phải NULL
```

### NullString vs *string comparison

| Aspect | `sql.NullString` | `*string` (pointer) |
|--------|-----------------|---------------------|
| Scan NULL | `Valid: false` | `nil` |
| Scan value | `Valid: true, String: "x"` | `*string("x")` |
| JSON marshal | ❌ phải implement | ✅ `omitempty` works (nil → omit) |
| Generic ORM | ⚠️ phải handle Valid | ✅ cleaner |
| DB nullability | ✅ explicit | ⚠️ implicit via nil |

## Query — read data

```go
// QueryContext — đọc nhiều rows
rows, err := db.QueryContext(ctx,
    "SELECT id, name, email FROM users WHERE age > ? LIMIT 100",
    18)
defer rows.Close()

if err != nil {
    return fmt.Errorf("query: %w", err)
}

var users []User
for rows.Next() {
    var u User
    if err := rows.Scan(&u.ID, &u.Name, &u.Email); err != nil {
        return fmt.Errorf("scan: %w", err)
    }
    users = append(users, u)
}
if err := rows.Err(); err != nil { // check rows error
    return err
}
```

### QueryRow — single row

```go
var name string
var age int
err := db.QueryRowContext(ctx,
    "SELECT name, age FROM users WHERE id = ?", id).
    Scan(&name, &age)

// QueryRowContext không có rows.Err() — chỉ Scan() error
if err == sql.ErrNoRows {
    return nil, fmt.Errorf("user %d not found", id)
}
if err != nil {
    return err
}
```

### Column introspection

```go
// Inspect columns tại runtime (dynamic queries)
columns, err := rows.ColumnTypes()
for _, col := range columns {
    fmt.Println(col.Name(), col.DatabaseTypeName(), col.ScanType())
}
// name  VARCHAR  string
// age   INT      int64
```

## Exec — write data

```go
// ExecContext — INSERT, UPDATE, DELETE (không trả về rows)
result, err := db.ExecContext(ctx,
    "UPDATE users SET email = ? WHERE id = ?",
    "new@example.com", userID)

rowsAffected, _ := result.RowsAffected()
lastInsertID, _ := result.LastInsertId() // MySQL/SQLite; PostgreSQL → 0
```

### Transaction patterns

```go
// Transaction — đảm bảo atomic operations
tx, err := db.BeginTx(ctx, nil)  // nil = default isolation
if err != nil {
    return fmt.Errorf("begin transaction: %w", err)
}
defer tx.Rollback()  // Rollback nếu không gọi Commit

// Dùng tx thay vì db cho tất cả operations trong transaction
_, err = tx.ExecContext(ctx, "INSERT INTO orders (user_id) VALUES (?)", userID)
if err != nil {
    return err  // deferred Rollback sẽ được gọi
}
_, err = tx.ExecContext(ctx, "UPDATE inventory SET stock = stock - 1 WHERE id = ?", itemID)
if err != nil {
    return err
}

if err := tx.Commit(); err != nil {
    return fmt.Errorf("commit: %w", err)
}
```

### Transaction with options

```go
// Serializable isolation
tx, err := db.BeginTx(ctx, &sql.TxOptions{
    Isolation: sql.LevelSerializable,
})

// Read-only transaction (cho SELECT queries)
tx, err := db.BeginTx(ctx, &sql.TxOptions{
    ReadOnly: true,
})
```

### Savepoint (PostgreSQL)

```go
// Dùng savepoint để partial rollback trong transaction
_, err = tx.ExecContext(ctx, "SAVEPOINT sp1")
if err != nil { return err }
_, err = tx.ExecContext(ctx, "INSERT INTO temp VALUES (?)", value)
if err != nil {
    tx.ExecContext(ctx, "ROLLBACK TO SAVEPOINT sp1") // partial rollback
} else {
    tx.ExecContext(ctx, "RELEASE SAVEPOINT sp1")
}
```

## Prepared Statements

```go
// Prepare once, execute many — hiệu quả cho repeated queries
stmt, err := db.PrepareContext(ctx,
    "SELECT name FROM users WHERE department_id = ?")
if err != nil {
    return fmt.Errorf("prepare: %w", err)
}
defer stmt.Close()

// Execute với nhiều params
depts := []int{1, 2, 3, 4, 5}
for _, dept := range depts {
    row := stmt.QueryRowContext(ctx, dept)
    var name string
    row.Scan(&name) // ...
}
```

### Prepared statements trong transactions

```go
// ⚠️ Trong transaction, prepared statements được isolate
// Không dùng prepared statements từ tx khác hoặc từ db
tx, _ := db.BeginTx(ctx, nil)
defer tx.Rollback()

txStmt, _ := tx.PrepareContext(ctx, "INSERT INTO logs (msg) VALUES (?)")
defer txStmt.Close()

txStmt.ExecContext(ctx, "message 1")
tx.Commit()

// ⚠️ stmt (tạo từ db.Prepare) không thể dùng trong tx và ngược lại
```

## sqlx — ergonomic extension

`sqlx` (github.com/jmoiron/sqlx) wrap `database/sql` để cung cấp named parameter và automatic struct mapping.

```go
import "github.com/jmoiron/sqlx"

// Named parameter — dùng tên thay vì ?
rows, err := db.QueryxContext(ctx,
    "SELECT * FROM users WHERE department_id = <dept>",
    map[string]any{"dept": deptID})

// Scan vào struct (automatic field mapping)
var users []User
db.SelectContext(ctx, &users, "SELECT * FROM users WHERE active = ?", true)

// Get — scan 1 row vào struct
var user User
db.GetContext(ctx, &user, "SELECT * FROM users WHERE id = ?", id)

// Scan raw map
var m map[string]any
row.MapScan(m)

// Null handling
type User struct {
    Name    string     `db:"name"`
    Deleted sql.NullTime `db:"deleted_at"`
}
```

## Common Pitfalls

### Connection exhaustion

```go
// ❌ BAD: query không có context, không handle error
for _, id := range ids {
    rows, _ := db.Query("SELECT ...")  // ❌ error ignored
    for rows.Next() { rows.Scan(...) }
    rows.Close()                        // ⚠️ Close có thể fail
}
// Nếu query fail giữa chừng, rows.Close() không được gọi → leak

// ✅ GOOD
for _, id := range ids {
    rows, err := db.QueryContext(ctx, "SELECT ...")
    if err != nil { return err }
    defer rows.Close()
    for rows.Next() {
        if err := rows.Scan(...); err != nil { return err }
    }
    if err := rows.Err(); err != nil { return err }
}
```

### nil pointer scan

```go
// ❌ Panic: Scan nil pointer (pointer not initialized)
var ptr *string
row.Scan(ptr) // ❌ panic: scan destination is nil pointer

// ✅ Init pointer
ptr := new(string)
row.Scan(ptr) // ✅ *ptr = "value"
```

### Iteration safety

```go
// ❌ BAD: iterate rows sau khi đã Close parent connection
rows, _ := db.QueryContext(ctx, "SELECT ...")
for rows.Next() {
    // ❌ Bad: db.Close() trong goroutine khác có thể close connection
    // và rows.Next() sẽ fail
}

// ✅ GOOD: use context with timeout
ctx, cancel := context.WithTimeout(ctx, 5*time.Second)
defer cancel()

rows, err := db.QueryContext(ctx, "SELECT ...")
```
