# 06.1 Structured Logging (slog)

## Overview

`slog` (structured logging) được thêm vào **Go 1.21** standard library — thay thế `log` cho production logging.

> `log` = unstructured text → `slog` = structured key-value → production observability

---

## log vs slog

| | log | slog |
|---|---|---|
| Format | Plain text | Structured (key-value) |
| Output | `os.Stderr` only | Configurable handler |
| Log levels | ❌ No built-in levels | ✅ Debug/Info/Warn/Error |
| JSON output | ❌ Manual | ✅ Built-in JSON handler |
| Context | ❌ | ✅ `slog.FromContext(ctx)` |

```go
// log — unstructured
log.Printf("user %s logged in from %s", userID, ip)
// → "2024/01/01 10:00:00 user 123 logged in from 192.168.1.1"

// slog — structured
slog.Info("user logged in", "userID", userID, "ip", ip)
// → {"time":"2024-01-01T10:00:00Z","level":"INFO","msg":"user logged in","userID":"123","ip":"192.168.1.1"}
```

## Logger với Levels

### Tạo Logger

```go
import "log/slog"

// Text handler (mặc định, dễ đọc)
log := slog.New(slog.NewTextHandler(os.Stdout, nil))

// JSON handler (production standard)
jsonLog := slog.New(slog.NewJSONHandler(os.Stdout, nil))

// Set default global logger
slog.SetDefault(jsonLog)
```

### Log Levels

```go
slog.Debug("debug message", "key", "value")
slog.Info("info message", "count", 42)
slog.Warn("warning message", "latency", "200ms")
slog.Error("error message", "err", err)
```

### Cấu hình Level

```go
// Chỉ log Warn trở lên
handler := slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
    Level: slog.LevelWarn,
})
logger := slog.New(handler)
logger.Info("this is ignored")    // NOT logged
logger.Warn("this is logged")    // logged
```

## Structured Attributes

### Basic types

```go
slog.Info("request completed",

slog.Info("user action",

)
```

### Nested output (JSON)

```json
{"time":"...","level":"INFO","msg":"user action","user":{"id":"123","name":"Alice","role":"admin"},"action":"login"}
```

## Logger với Context

### Tạo logger từ context

```go
// Set logger vào context
ctx := slog.WithContext(context.Background(),
    slog.With("service", "auth"),
    slog.With("version", "1.0"),
)

// Inject vào request
func Middleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        ctx := slog.WithContext(r.Context(),
            slog.With("request_id", uuid.NewString()),
        )
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}

// Dùng trong handler
func handler(w http.ResponseWriter, r *http.Request) {
    slog.Info("handling request", "path", r.URL.Path)
}
```

## HTTP Server với slog

```go
func main() {
    // JSON logger cho production
    logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))

    server := &http.Server{
        Addr:         ":8080",
        Handler:      nil,
        ErrorLog:     slog.NewLogLogger(logger.Handler(), slog.LevelError),
    }

    // Hoặc dùng slog default
    slog.SetDefault(logger)

    // Log startup
    slog.Info("server starting", "addr", server.Addr)

    if err := server.ListenAndServe(); err != nil {
        slog.Error("server failed", "err", err)
    }
}
```

## Common patterns

### Error logging với stack

```go
if err != nil {
    slog.Error("operation failed",
        "err", err,
        "operation", "db_query",
        "table", "users",
    )
}
```

### Request/Response logging (middleware)

```go
func LoggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()

        wrapped := &responseWriter{ResponseWriter: w, status: http.StatusOK}
        next.ServeHTTP(wrapped, r)

        slog.Info("http request",
            "method", r.Method,
            "path", r.URL.Path,
            "status", wrapped.status,
            "duration", time.Since(start),
        )
    })
}

type responseWriter struct {
    http.ResponseWriter
    status int
}

func (w *responseWriter) WriteHeader(code int) {
    w.status = code
    w.ResponseWriter.WriteHeader(code)
}
```

### Development vs Production

```go
// Development: text, debug level
// Production: JSON, info level

import "os"

var logger *slog.Logger

func init() {
    if os.Getenv("ENV") == "production" {
        logger = slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
            Level: slog.LevelInfo,
        }))
    } else {
        logger = slog.New(slog.NewTextHandler(os.Stdout, &slog.HandlerOptions{
            Level: slog.LevelDebug,
        }))
    }
}
```

## vs other logging libraries

| Library | Notes |
|---------|-------|
| `slog` (Go 1.21+) | ✅ Standard library, production-ready |
| `log/slog` | Same thing — import path |
| `zerolog` | 3rd party, faster than slog, similar API |
| `zap` | Uber's logger, very fast, structured |
| `logrus` | Popular but slower, deprecated in favor of slog/zap |

> **Recommendation**: Dùng `slog` cho internal projects; `zap`/`zerolog` cho high-throughput services.

## Custom Handler — Extending slog

```go
// Implement slog.Handler interface để custom behavior
type Handler interface {
    Enabled(context.Context, Level) bool
    Handle(context.Context, Record) error
    WithAttrs(attrs []Attr) Handler
    WithGroup(name string) Handler
}

// Example: Add caller info to every log
type callerHandler struct {
    slog.Handler
}

func (h *callerHandler) Handle(ctx context.Context, r slog.Record) error {
    // Add caller location to every record
    _, file, line, _ := runtime.Caller(3)
    r.AddAttrs(slog.String("caller", fmt.Sprintf("%s:%d", filepath.Base(file), line)))
    return h.Handler.Handle(ctx, r)
}

// Wrap existing handler
base := slog.NewJSONHandler(os.Stdout, nil)
logger := slog.New(&callerHandler{Handler: base})
```

## slog Performance Internals

```go
// slog dùng sync.Pool nội bộ để giảm allocation
// Nhưng vẫn có overhead khi:
// 1. Level không enabled — args vẫn được evaluated!

// ❌ Expensive: format string evaluated dù Debug disabled
slog.Debug("request data: " + string(jsonBytes))
//                          ↑ allocation ngay cả khi debug disabled

// ✅ Lazy: dùng slog.LogAttrs (skip evaluation nếu disabled)
slog.LogAttrs(ctx, slog.LevelDebug, "request",
    slog.String("data", string(jsonBytes)),  // ✅ only if enabled
)

// ✅ Check level first for expensive ops
if slog.Default().Enabled(ctx, slog.LevelDebug) {
    slog.Debug("expensive debug", "data", computeExpensiveDebugInfo())
}
```

### Performance Comparison

slog sử dụng [[03.3 - Concurrency Tools#sync.Pool]] nội bộ để giảm allocation.

| Library | ns/op (simple log) | Allocations |
|---------|---------------------|-------------|
| `log` (stdlib) | ~250ns | 1 alloc |
| `slog` (stdlib) | ~400-600ns | 1-3 allocs |
| `zap` (uber) | ~100-200ns | 0 allocs |
| `zerolog` | ~80-150ns | 0 allocs |

> **Kết luận**: Với throughput < 100K logs/sec, `slog` là đủ. Với high-frequency logging (tracing, metrics), dùng `zap` hoặc `zerolog`.

## slog vs zap vs zerolog — Trade-off

| | `slog` | `zap` | `zerolog` |
|---|---|---|---|
| Dependency | ✅ None (stdlib) | ⚠️ External | ⚠️ External |
| Performance | ⚠️ Good | ✅✅ Excellent | ✅✅ Excellent |
| Allocations | ⚠️ Some | ✅ Zero (Sugared) | ✅ Zero |
| API ergonomics | ✅ Simple | ⚠️ Verbose (SugaredLogger) | ✅ Chain API |
| Context support | ✅ Built-in | ✅ | ✅ |
| Custom handler | ✅ interface | ✅ Core/Sink | ✅ Hook |
| JSON output | ✅ | ✅ | ✅ |
| Use case | Internal, standard | High-perf prod | High-perf prod |

```go
// zap (zero allocation logger)
import "go.uber.org/zap"
logger, _ := zap.NewProduction()
defer logger.Sync()
logger.Info("user logged in",
    zap.String("userID", "123"),
    zap.Int("attempt", 1),
)

// zerolog (chain API)
import "github.com/rs/zerolog/log"
log.Info().
    Str("userID", "123").
    Int("attempt", 1).
    Msg("user logged in")
```

## Production Log Architecture Patterns

Production build config (ENV, log level) — liên quan đến [[07.2 - Build & Tooling#Build Flags]].

```go
// Pattern 1: Request-scoped logger via context
func Middleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        reqLogger := slog.With(
            "request_id", uuid.NewString(),
            "method", r.Method,
            "path", r.URL.Path,
        )
        ctx := context.WithValue(r.Context(), loggerKey{}, reqLogger)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}

func LoggerFromCtx(ctx context.Context) *slog.Logger {
    if l, ok := ctx.Value(loggerKey{}).(*slog.Logger); ok {
        return l
    }
    return slog.Default()
}

// Pattern 2: Multi-handler (stdout + file + alert system)
type multiHandler struct{ handlers []slog.Handler }

func (h *multiHandler) Handle(ctx context.Context, r slog.Record) error {
    for _, handler := range h.handlers {
        if handler.Enabled(ctx, r.Level) {
            if err := handler.Handle(ctx, r.Clone()); err != nil {
                return err
            }
        }
    }
    return nil
}

// Pattern 3: Sampling logger (high-frequency logs)
type samplingHandler struct {
    slog.Handler
    rate int64  // log 1 out of N
    count int64
}

func (h *samplingHandler) Handle(ctx context.Context, r slog.Record) error {
    n := atomic.AddInt64(&h.count, 1)
    if n % h.rate == 0 {
        return h.Handler.Handle(ctx, r)
    }
    return nil
}
```

## Best Practices

```go
// 1. Luôn log với structured attrs, không format string
slog.Info("user action",
    "user_id", userID,    // ✅ key-value
    "action", "login",
    "ip", ip,
)
// KHÔNG: slog.Info(fmt.Sprintf("user %s logged in from %s", userID, ip))

// 2. Err attr phải là "err" key (convention)
slog.Error("operation failed", "err", err, "op", "db.query")

// 3. Dùng slog.With cho shared attributes (service-level)
logger := slog.Default().With(
    "service", "auth-service",
    "version", version.String(),
)
// Mỗi log từ logger này sẽ có service + version tự động

// 4. SetDefault ở main(), không ở init()
func main() {
    logger := slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
        Level: slog.LevelInfo,
    }))
    slog.SetDefault(logger)
    // từ đây toàn bộ code dùng slog.Info() sẽ được structured
}

// 5. Development vs Production config
func newLogger() *slog.Logger {
    if os.Getenv("ENV") != "production" {
        return slog.New(slog.NewTextHandler(os.Stderr, &slog.HandlerOptions{
            Level: slog.LevelDebug,
        }))
    }
    return slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
        Level: slog.LevelInfo,
    }))
}
```
