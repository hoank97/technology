# 06.3 JSON & Encoding

## encoding/json Overview

Go's `encoding/json` là standard library cho JSON serialization/deserialization. Package này sử dụng **reflection** để inspect struct fields tại runtime — đây là lý do tại sao struct fields phải là exported (viết hoa).

```go
// Marshal → struct/primitive → JSON bytes
data, err := json.Marshal(v)

// Unmarshal → JSON bytes → struct/primitive
err = json.Unmarshal(data, &v)
```

---

## Struct Tags — control field behavior

Struct field tags là string literals sau field declaration, format: `key:"value"`.

### Các tag keys phổ biến nhất

```go
type Config struct {
    // Field name trong JSON → "server_name" thay vì "ServerName"
    ServerName string `json:"server_name"`

    // Bỏ qua field không marshal/unmarshal
    Internal   string `json:"-"`

    // Marshal như string (number → "123", []byte → base64)
    Port       int    `json:"port,string"`   // 8080 → "8080"
    Secret     []byte `json:"secret,string"` // []byte → base64

    // omitempty — bỏ khỏi output khi zero value
    Timeout    int    `json:"timeout,omitempty"` // 0 → field not present

    // inline — flatten nested struct (chỉ dùng được 1 level)
    Metadata   Meta   `json:",inline"`
}

// Kết hợp nhiều options
Port int `json:"port,string,omitempty"`
```

### Tag reference table

| Tag | Example | Behavior |
|-----|---------|----------|
| `json:"name"` | `` json:"port" `` | Rename field trong JSON |
| `json:"-"` | `` json:"-" `` | Never encode/decode |
| `json:",omitempty"` | `` json:",omitempty" `` | Skip zero values |
| `json:",string"` | `` json:",string" `` | Marshal as JSON string |
| `json:",inline"` | `` json:",inline" `` | Flatten embedded struct's fields |

### Field name vs tag priority

```go
type T struct {
    Field string `json:"custom_name"` // ✅ dùng "custom_name"
    Other string                   // ✅ dùng "Other" (field name)
}

// Key được chọn theo thứ tự:
// 1. `json:"..."` tag nếu có
// 2. Field name (PascalCase) nếu không có tag
// 3. "-" → skip hoàn toàn
```

## Marshal / Unmarshal — core operations

### Basic usage

```go
type User struct {
    Name  string `json:"name"`
    Age   int    `json:"age"`
    Email string `json:"email,omitempty"`
}

user := User{Name: "Alice", Age: 30}

// Marshal → []byte
data, err := json.Marshal(user)      // compact: {"name":"Alice","age":30}
data2, _ := json.MarshalIndent(user, "", "  ") // pretty: 2-space indent

// Unmarshal → struct
var u User
err := json.Unmarshal(data, &u)

// Handle null JSON → pointer
type Config struct {
    Host *string `json:"host,omitempty"` // null → u.Host == nil
}
```

### Empty values & omitempty behavior

```go
type Item struct {
    Count   int     `json:"count,omitempty"`  // 0     → omit
    Name    string  `json:"name,omitempty"`   // ""    → omit
    Active  bool    `json:"active,omitempty"` // false → omit
    Scores  []int   `json:"scores,omitempty"` // nil   → omit
    Meta    *Meta   `json:"meta,omitempty"`   // nil   → omit
}

// ⚠️ Caveat: slices/pointers/empty slices có behavior khác nhau
type T struct {
    Items []string `json:"items,omitempty"`
}
t := T{Items: []string{}} // []{} — empty slice KHÔNG bị omit (≠ nil)
```

### Unknown / arbitrary keys

```go
// Dùng map[string]any để decode JSON với keys không predictable
var m map[string]any
data := []byte(`{"id": 123, "extra": "value"}`)
json.Unmarshal(data, &m)
// m = {"id": 123, "extra": "value"} — mọi key đều được giữ

// Hoặc dùng RawMessage để defer decode
type Response struct {
    Status  string          `json:"status"`
    Payload json.RawMessage `json:"payload"` // chưa decode
}
```

## Decoder vs Unmarshal — streaming vs in-memory

| Aspect | `Unmarshal` | `Decoder` |
|--------|------------|-----------|
| Input | Full `[]byte` | `io.Reader` (stream) |
| Memory | Load all vào memory | Process chunk by chunk |
| Large JSON | ❌ Full file in memory | ✅ Streaming |
| Incremental | ❌ | ✅ Decode part → process → next part |
| Decoder options | No | Can configure (DisallowUnknownFields, etc.) |

### Decoder for streaming / large files

```go
// ❌ Bad: load entire file into memory
data, _ := os.ReadFile("huge.json")
var config Config
json.Unmarshal(data, &config)

// ✅ Good: stream from file
f, _ := os.Open("huge.json")
defer f.Close()

dec := json.NewDecoder(f)
dec.DisallowUnknownFields() // reject unexpected keys

for dec.More() {
    var item Item
    dec.Decode(&item)
    process(item)
}
```

### Encoder for streaming output

```go
// Encoder → write JSON incrementally to writer
enc := json.NewEncoder(os.Stdout)
enc.SetIndent("", "  ") // pretty print

items := []Item{{Name: "A"}, {Name: "B"}}
for _, item := range items {
    enc.Encode(item) // mỗi Encode ghi một JSON object + newline
}
// Output:
// {
//   "name": "A"
// }
// {
//   "name": "B"
// }
```

### Encoder vs Marshal

```go
// Marshal — tạo full JSON []byte trong memory
data, _ := json.Marshal(config) // ⚠️ toàn bộ JSON trong memory

// Encoder — write trực tiếp ra writer (stream)
enc := json.NewEncoder(w)
enc.Encode(config)              // ghi trực tiếp, không allocate full JSON
```

## Custom MarshalJSON / UnmarshalJSON

### Custom marshal

```go
type IP struct{ Quad [4]byte }

func (ip IP) MarshalJSON() ([]byte, error) {
    return []byte(fmt.Sprintf(`"%d.%d.%d.%d"`,
        ip.Quad[0], ip.Quad[1], ip.Quad[2], ip.Quad[3])), nil
}
// Marshal → "192.168.1.1" thay vì {"Quad":[192,168,1,1]}
```

### Custom unmarshal

```go
func (ip *IP) UnmarshalJSON(data []byte) error {
    var s string
    if err := json.Unmarshal(data, &s); err != nil {
        return err
    }
    // Parse "192.168.1.1"
    parts := strings.Split(s, ".")
    if len(parts) != 4 {
        return fmt.Errorf("invalid IP format")
    }
    for i, p := range parts {
        n, _ := strconv.Atoi(p)
        ip.Quad[i] = byte(n)
    }
    return nil
}
```

### Marshaler / Unmarshaler interfaces

```go
type Marshaler interface {
    MarshalJSON() ([]byte, error)
}
type Unmarshaler interface {
    UnmarshalJSON([]byte) error
}

// ⚠️ Pointer receiver — json.Unmarshal requires *Unmarshaler
// Khi Unmarshal gọi với non-pointer, nó tạo pointer mới → sao chép struct
```

### Encoder / Decoder — custom type handling

```go
// json.Encoder / Decoder cũng có SetEscapeHTML, SetIndent options
enc := json.NewEncoder(buf)
enc.SetEscapeHTML(false) // < → < (không escape, dùng khi biết không có XSS)
// ⚠️ Chỉ set false khi biết chắc output không bao giờ chứa user input
```

## Advanced Topics

### RawMessage — deferred parsing

```go
type Event struct {
    Type string          `json:"type"`
    Data json.RawMessage `json:"data"` // chưa parse, giữ nguyên JSON bytes
}

var e Event
json.Unmarshal(body, &e)

// Sau đó parse Data dựa vào Type
switch e.Type {
case "user_created":
    var u UserPayload
    json.Unmarshal(e.Data, &u)
case "order_placed":
    var o OrderPayload
    json.Unmarshal(e.Data, &o)
}
```

### Number — arbitrary precision for large numbers

```go
// Standard unmarshal: number → float64 (IEEE 754, precision limit)
var v any
json.Unmarshal([]byte("9007199254740993"), &v)
// v = 9.007199254740993e+15 — mất precision

// ✅ Dùng json.Number → giữ nguyên string, parse khi cần
var v2 any
dec := json.NewDecoder(bytes.NewReader(body))
dec.UseNumber()
dec.Decode(&v2)
// v2 = json.Number("9007199254740993") — giữ precision

n := v2.(json.Number)
i64, _ := n.Int64()     // parse as int64
f64, _ := n.Float64()   // parse as float64
str := n.String()        // "9007199254740993"
```

### Encode / Decode time

```go
// Default: RFC3339 string
type Event struct {
    Created time.Time `json:"created"`
}
// {"created":"2024-01-15T10:30:00Z"}

// Custom format: Unix timestamp
type Event2 struct {
    Created int64 `json:"created"`
    // Application converts: time.Now().Unix()
}

// Marshal method for custom time format
func (t Time) MarshalJSON() ([]byte, error) {
    return []byte(fmt.Sprintf(`%d`, t.Unix())), nil
}
```

### nil vs empty slice vs empty string

```go
type T struct {
    Names []string `json:"names,omitempty"`
    Desc  string   `json:"desc,omitempty"`
}

t1 := T{}
json.Marshal(t1) // {} — both omitted

t2 := T{Names: []string{}}
json.Marshal(t2) // {"names":[]} — nil slice = omit, empty slice = show

t3 := T{Desc: ""}
json.Marshal(t3) // {} — empty string = omit với omitempty
```

## Performance Comparison

| Library | Speed | Allocation | Notes |
|---------|-------|-----------|-------|
| stdlib `encoding/json` | ~1x | High | Generics-free, reflect-heavy, Go 1.22+ improved |
| `json-iterator/go` | ~3-5x | Lower | Drop-in replacement, configurable |
| `sonic` (by byteDance) | ~8-15x | Lowest | Unsafe + JIT, only Linux/amd64, no GC issues |
| `goccy/go-json` | ~5-10x | Low | Pure Go, no reflection, complex setup |

```go
// Benchmarks (approximate, typical API call):
// stdlib:    ~1,500 ns/op, ~3 allocs/op
// json-iterator: ~300 ns/op, ~1 alloc/op
// sonic:     ~150 ns/op, ~0.5 alloc/op

// ⚠️ Chọn library dựa trên:
// - 80/20 rule: stdlib đủ cho hầu hết cases
// - High-throughput (payroll, trading, real-time): consider json-iterator/sonic
// - Constraints: pure Go? cross-platform? → stdlib hoặc go-json
// - Maintenance overhead: thêm dependency cho 10x perf có worth không?
```

## Common Pitfalls

```go
// ❌ Quên pointer khi Unmarshal
var u User
json.Unmarshal(data, u)     // ❌ Compile error

// ✅
json.Unmarshal(data, &u)    // ✅

// ❌ Mutate original slice khi Unmarshal vào shared array
data := []byte(`[1,2,3]`)
var nums []int
json.Unmarshal(data, &nums) // nums = [1 2 3]

// ⚠️ Nếu reuse nums, phải Reset()
nums = nums[:0]
json.Unmarshal(data2, &nums) // append vào nums cũ

// ❌ Bỏ qua error
_ = json.Unmarshal(data, &v) // ❌

// ✅
if err := json.Unmarshal(data, &v); err != nil {
    return fmt.Errorf("decode config: %w", err)
}
```
