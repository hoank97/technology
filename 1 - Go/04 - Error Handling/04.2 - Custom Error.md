# 04.2 Custom Error

## 1. Sentinel Error

```go
import "errors"

var ErrNotFound = errors.New("not found")
var ErrTimeout = errors.New("timeout")
```

→ Check bằng `errors.Is()`:

```go
if errors.Is(err, ErrNotFound) {
    // handle not found
}
```

---

## 2. Custom Error Type

```go
type MyError struct {
    Msg  string
    Code int
}

func (e *MyError) Error() string {
    return fmt.Sprintf("%s (code=%d)", e.Msg, e.Code)
}
```

→ Extract bằng `errors.As()`:

```go
var myErr *MyError
if errors.As(err, &myErr) {
    fmt.Println(myErr.Code)
}
```

---

## 3. Wrap Error (Context)

Thêm context với `%w`:

```go
_, err := io.ReadAll(r)
if err != nil {
    return fmt.Errorf("read config: %w", err)
}
```

→ `errors.Is()` và `errors.As()` tìm xuyên qua chain:

```bash
# Debug: in ra full chain
fmt.Println(err)  // read config: file not found
```

---

## 4. With Stack (pkg/errors)

```go
import "github.com/pkg/errors"

return errors.Wrap(err, "read config")
// Output: read config: file not found: ...
// Có cả stack trace
```

## 5. errors.Join (Go 1.20+)

Kết hợp nhiều errors thành một:

```go
import "errors"

func validate(u *User) error {
    var errs []error
    if u.Name == "" {
        errs = append(errs, errors.New("name required"))
    }
    if u.Age < 0 {
        errs = append(errs, errors.New("age cannot be negative"))
    }
    if u.Email == "" {
        errs = append(errs, errors.New("email required"))
    }

    // Trả về nil nếu không có lỗi nào
    // Trả về joined error nếu có ≥1 lỗi
    return errors.Join(errs...)
}
```

### Khi nào dùng?

| Pattern | Khi nào |
|---------|---------|
| `errors.Join(errs...)` | Gom **nhiều independent errors** (validation) |
| `fmt.Errorf("msg: %w", err)` | Wrap **một error** với context |
| `errors.Join(fmt.Errorf("msg: %w", e1), e2)` | Kết hợp cả hai |

### errors.Is với Join

```go
err := errors.Join(ErrNotFound, ErrTimeout)
errors.Is(err, ErrNotFound)  // true
errors.Is(err, ErrTimeout)   // true

// Unwrap chain
fmt.Println(err) // "not found\ntimeout"
```

### errors.As với Join

```go
var perr *ProcessError
if errors.As(err, &perr) {
    fmt.Println(perr.Code)
}
```

### fmt.Errorf với multiple %w (Go 1.20+)

```go
// ✅ Go 1.20+
return fmt.Errorf("validation failed: %w", errors.Join(err1, err2))

// ❌ Pre-1.20 — chỉ wrap được 1 error
return fmt.Errorf("validation failed: %w", err1)
```

### Use case: Collect validation errors

```go
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return e.Field + ": " + e.Message
}

func ValidateAll(data *Data) error {
    var errs []error

    if data.Name == "" {
        errs = append(errs, &ValidationError{Field: "name", Message: "required"})
    }
    if data.Value < 0 {
        errs = append(errs, &ValidationError{Field: "value", Message: "must be positive"})
    }

    return errors.Join(errs...)
}
```

## Opaque Errors (Information Hiding)

Đôi khi bạn muốn **ẩn lý do chi tiết** khỏi caller — "opaque error":

```go
// Thay vì expose sentinel/type → chỉ expose behavior
type temporary interface {
    Temporary() bool
}

// Caller check behavior, không phải type
func isRetryable(err error) bool {
    var t temporary
    return errors.As(err, &t) && t.Temporary()
}
```

> **Design principle**: "Be conservative in what you send, liberal in what you accept" — xuất ít error types nhất có thể, nhưng cung cấp đủ thông tin để caller quyết định.

## Error Design Patterns — Góc Nhìn Đa Chiều

### Pattern 1: Sentinel (Simple flag)

```go
// Tốt cho: simple yes/no conditions
var ErrNotFound = errors.New("not found")
var ErrAlreadyExists = errors.New("already exists")
```

### Pattern 2: Custom struct (Rich metadata)

```go
// Tốt cho: cần HTTP code, request ID, retry hint, ...
type APIError struct {
    StatusCode int
    RequestID  string
    Message    string
    Retryable  bool
}

func (e *APIError) Error() string {
    return fmt.Sprintf("[%d] %s (req=%s)", e.StatusCode, e.Message, e.RequestID)
}

func (e *APIError) Temporary() bool { return e.Retryable }
```

### Pattern 3: Error với behavior interface

```go
// Tốt cho: generic retry/timeout handling
type TimeoutError interface {
    error
    Timeout() bool
}

type NotFoundError interface {
    error
    NotFound() bool
}
```

### Trade-off: Chọn Error Type Nào?

| Scenario | Sentinel | Custom Struct | Behavior Interface |
|----------|----------|---------------|-------------------|
| Đơn giản, binary outcome | ✅ | ❌ overkill | ❌ overkill |
| Cần metadata (code, ID) | ❌ | ✅ | ❌ |
| Generic behavior check | ❌ | ⚠️ | ✅ |
| Public API (library) | ⚠️ tight coupling | ⚠️ | ✅ most flexible |
| Internal package | ✅ | ✅ | ⚠️ |

## pkg/errors vs stdlib — Khi nào dùng cái nào?

```go
// stdlib (Go 1.13+) — đủ dùng cho hầu hết cases
import "fmt"
return fmt.Errorf("open file: %w", err)

// github.com/pkg/errors — khi cần stack trace
import "github.com/pkg/errors"
return errors.Wrap(err, "open file")  // captures stack trace at wrap point

// Lấy stack trace
if err != nil {
    fmt.Printf("%+v\n", err)  // prints with stack trace
}
```

### Trade-off: stdlib vs pkg/errors

| | `fmt.Errorf %w` | `pkg/errors.Wrap` |
|---|---|---|
| Stack trace | ❌ | ✅ Captured |
| Performance | ✅ Fast | ⚠️ Stack capture overhead |
| Dependency | ✅ None | ⚠️ External |
| errors.Is/As | ✅ Works | ✅ Works |
| Use case | General | Debugging complex flows |

> **Recommendation**: Dùng `errors.Join` + `fmt.Errorf %w` (Go 1.20 stdlib) cho internal. Chỉ dùng `pkg/errors` khi stack trace thực sự cần thiết.

## Error Chain Design — Best Practices

```go
// ✅ Thêm function context khi wrap cross package boundary
// package db
func (r *Repo) GetUser(id int) (*User, error) {
    row, err := r.db.QueryRow("SELECT ...", id)
    if err != nil {
        return nil, fmt.Errorf("Repo.GetUser id=%d: %w", id, err)
    }
    // ...
}

// package service
func (s *Service) GetUser(id int) (*User, error) {
    u, err := s.repo.GetUser(id)
    if err != nil {
        return nil, fmt.Errorf("Service.GetUser: %w", err)  // add layer context
    }
    return u, nil
}

// Final error chain:
// "Service.GetUser: Repo.GetUser id=42: sql: no rows in result set"
// → Clear stack trace via message WITHOUT pkg/errors overhead
```

## 📌 Tóm tắt

```
Custom Error Types
│
├── Sentinel (errors.New)
│   ├── Simple binary outcome
│   ├── errors.Is() để so sánh
│   └── ⚠️ Không carry metadata
│
├── Custom Error Struct
│   ├── Rich metadata: code, ID, message, retry hint
│   ├── errors.As() để extract
│   └── Implement optional interfaces: Temporary(), NotFound()
│
├── Behavior Interface Pattern
│   ├── Define behavior không phải type: Timeout(), Retryable()
│   └── ✅ Most flexible for public APIs
│
├── Error Wrapping
│   ├── fmt.Errorf("context: %w", err) → preserves chain
│   ├── errors.Join (Go 1.20+) → multi-error
│   └── pkg/errors.Wrap → giữ stack trace (có overhead)
│
├── errors.Is vs errors.As
│   ├── Is: so sánh identity (sentinel)
│   └── As: extract type từ chain
│
└── Design Trade-offs
    ├── Ít error types xuất ra public API → opaque errors
    ├── Thêm context khi cross package boundary
    └── Đừng wrap error đã đủ context (double wrapping)
```
