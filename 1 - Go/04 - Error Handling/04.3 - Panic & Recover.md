## When to Panic

### Dùng được khi:

Khi nào dùng panic thay vì error — xem thêm [[04.1 - Error is a Value]].

1. **Programmer error** — violated invariants
   ```go
   if index < 0 || index >= len(slice) {
       panic("index out of bounds") // Bug nghiêm trọng
   }
   ```

2. **Init failure** — không thể tiếp tục
   ```go
   if os.Getenv("API_KEY") == "" {
       panic("API_KEY must be set") // App không thể chạy
   }
   ```

3. **Truly unrecoverable** — memory corruption, stack overflow

### Không dùng khi:

- Expected errors (user input, network)
- Library code

## Recover

`recover()` **bắt panic** và ngăn crash:

```go
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered:", r)
        }
    }()
    panic("oops!")
}
```

### Chỉ hoạt động trong defer

```go
// ❌ Không work — panic xảy ra trước
recover()
panic("oops!")
```

### Recover in goroutine

recover() trong goroutine — liên quan đến [[03.1 - Goroutine & Scheduler#Goroutine Lifecycle]].

```go
go func() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered in goroutine:", r)
        }
    }()
    panic("oops")
}()
```

## HTTP Server Pattern

Nơi recover hợp lý:

```go
func handleRequest(w http.ResponseWriter, r *http.Request) {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("Panic recovered: %v", r)
            http.Error(w, "Internal Server Error", 500)
        }
    }()
    // handler code
}
```

---

## ⚠️ Cautions

1. **Không dùng recover thay error handling**
2. **State sau panic có thể corrupted**
3. **Chỉ recover có chủ đích** — trong framework/server

## Panic Internals — Stack Unwinding

```go
// Khi panic xảy ra:
// 1. Current function stops immediately
// 2. Deferred functions in that function RUN (LIFO)
// 3. Propagate up the call stack
// 4. Each level's deferred functions run
// 5. If no recover() → crash + print stack trace

func A() {
    defer fmt.Println("A deferred") // (3) runs third
    B()
}
func B() {
    defer fmt.Println("B deferred") // (2) runs second
    C()
}
func C() {
    defer fmt.Println("C deferred") // (1) runs first
    panic("C panicked!")
}
// Output:
// C deferred
// B deferred
// A deferred
// panic: C panicked!
// goroutine 1 [running]: ...
```

## Runtime Panics — Built-in Cases

```go
// Go runtime tự động panic trong các trường hợp:

// 1. Nil pointer dereference
var p *int
_ = *p  // panic: nil pointer dereference

// 2. Index out of bounds
s := []int{1, 2, 3}
_ = s[5]  // panic: index out of range [5] with length 3

// 3. Type assertion fail (without ok form)
var i interface{} = "hello"
_ = i.(int)  // panic: interface conversion: interface is string, not int
// ✅ Safe form:
n, ok := i.(int)  // ok = false, no panic

// 4. Send on closed channel
ch := make(chan int)
close(ch)
ch <- 1  // panic: send on closed channel

// 5. Negative slice index
s := []int{1, 2, 3}
_ = s[-1]  // panic: index out of range

// 6. Stack overflow (infinite recursion)
func infinite() { infinite() }
// panic: stack overflow
```

## Panic vs Error — Góc Nhìn Đa Chiều

| | `panic` | `error` |
|---|---|---|
| Khi nào | Programmer error, invariant violated | Expected failure |
| Recovery | `recover()` trong defer | `if err != nil` |
| Performance | Fast (unwinding, không stack alloc) | ⚠️ Allocation per error |
| Caller awareness | ❌ (unexpected) | ✅ (in signature) |
| Cross-boundary | ❌ Dangerous (corruptible state) | ✅ Safe |
| Library code | ❌ Never | ✅ Always |
| Init failure | ✅ OK | ⚠️ Program can't recover anyway |

> **Go Philosophy**: panics should NOT cross package boundaries. Nếu library panic → bug trong library.

## recover() — Advanced Patterns

### Pattern 1: Convert panic → error (thường dùng trong library)

```go
// Safe wrapper: convert internal panic → error ở boundary
func safeExecute(fn func()) (err error) {
    defer func() {
        if r := recover(); r != nil {
            switch x := r.(type) {
            case error:
                err = x
            default:
                err = fmt.Errorf("panic: %v", r)
            }
        }
    }()
    fn()
    return nil
}

// Usage
err := safeExecute(func() {
    panic("something went wrong")
})
fmt.Println(err) // "panic: something went wrong"
```

### Pattern 2: Panic với stack trace

```go
func safeGo(fn func()) {
    go func() {
        defer func() {
            if r := recover(); r != nil {
                buf := make([]byte, 4096)
                n := runtime.Stack(buf, false)
                slog.Error("goroutine panicked",
                    "recovered", r,
                    "stack", string(buf[:n]),
                )
            }
        }()
        fn()
    }()
}
```

### Pattern 3: Must pattern (thường thấy trong init code)

Must pattern dùng panic — liên quan đến [[01.2 - Control Flow & Functions#init() Function]].

```go
// Must panics nếu err != nil — chỉ dùng ở startup/init
func Must[T any](v T, err error) T {
    if err != nil {
        panic(err)
    }
    return v
}

// Dùng ở package-level hoặc main():
var tmpl = Must(template.ParseFiles("layout.html"))
var db   = Must(sql.Open("postgres", dsn))
// → Nếu fail thì không nên tiếp tục, panic là đúng
```

## Panic: Khi nào hợp lý?

```
Panic là ĐÚNG khi:
  ├── Init failure: program không thể tiếp tục (must-pattern)
  ├── Programmer invariant violated:
  │   ├── Invalid state mà user code không thể cause  
  │   └── e.g., internal slot out of range sau logic đã validate
  └── Unrecoverable: memory corruption, stack overflow

Panic là SAI khi:
  ├── Network error (expected)
  ├── User input invalid (expected)
  ├── File not found (expected)
  ├── Library code (caller can't expect it)
  └── Thay thế for error handling (antipattern)
```
