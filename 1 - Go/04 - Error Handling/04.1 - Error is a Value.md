## error Interface — Internals

`error` trong Go là một **interface** đơn giản nhất có thể:

```go
// Built-in interface — chỉ 1 method
type error interface {
    Error() string
}

// errors.New trả về *errorString — implements error
type errorString struct{ s string }
func (e *errorString) Error() string { return e.s }

// Mọi type có method Error() string đều là error
type MyError struct{ Code int }
func (e *MyError) Error() string {
    return fmt.Sprintf("error code %d", e.Code)
}
```

> **Góc nhìn đa chiều**: Go coi error như *giá trị thông thường* (value), không phải exception. Điều này buộc programmer phải xử lý error một cách tường minh, không "quên" như try/catch.

## Check Error

```go
// ❌ Ignore error — nguy hiểm, mất thông tin
data, _ := json.Marshal(v)

// ✅ Handle + propagate với context
data, err := json.Marshal(v)
if err != nil {
    return fmt.Errorf("marshal user %d: %w", userID, err)
}

// ✅ errors.Is — so sánh đúng cách (xuyên chuỗi wrap)
if errors.Is(err, os.ErrNotExist) {
    return handleNotFound()
}

// ✅ errors.As — extract type cụ thể
var netErr *net.OpError
if errors.As(err, &netErr) {
    fmt.Println("network op:", netErr.Op)
}
```

## Sentinel Errors — `errors.New`

```go
// Package-level sentinel errors — exported để caller compare
var (
    ErrNotFound   = errors.New("not found")
    ErrPermission = errors.New("permission denied")
    ErrTimeout    = errors.New("timeout")
)

// Kiểm tra với errors.Is (hoạt động xuyên wrap chain)
if errors.Is(err, ErrNotFound) {
    // handle not found
}
```

| | Sentinel `errors.New` | Custom Error Type |
|---|---|---|

| Carry metadata | ❌ String only | ✅ Struct fields |
| Comparison | `errors.Is` | `errors.As` |
| Extensibility | ❌ Fixed message | ✅ |
| Use case | Simple flags | Rich error context |

## Error Wrapping với `%w`

```go
// fmt.Errorf + %w — wrap để giữ nguyên gốc
func readConfig(path string) error {
    f, err := os.Open(path)
    if err != nil {
        return fmt.Errorf("readConfig %q: %w", path, err)
    }
    defer f.Close()
    // ...
}

// Caller có thể unwrap để check nguyên nhân gốc
err := readConfig("/etc/app.conf")
if errors.Is(err, os.ErrNotExist) {
    fmt.Println("config file not found") // ✅ works through wrap chain
}
fmt.Println(err)
// "readConfig "/etc/app.conf": open /etc/app.conf: no such file or directory"
```

### Wrapping Chain Mechanics

```
fmt.Errorf("readConfig: %w", err)
    → wrappedError { msg: "readConfig: ...", cause: err }

errors.Is(wrappedErr, os.ErrNotExist):
    → check wrappedErr == os.ErrNotExist? No
    → check wrappedErr.Unwrap() == os.ErrNotExist? Yes → true
```

### ⚠️ Khi không nên wrap

```go
// Đừng wrap nếu error không cần propagate nguyên nhân
// (ví dụ: boundary giữa tầng service và domain)
return fmt.Errorf("user not found") // ❌ mất info

// Nên:
// - Wrap khi cross pkg boundary để có context
// - Không wrap khi error đã rõ ràng hoặc bạn muốn ẩn implementation
```

## Handle vs Ignore — Patterns

```go
// ✅ Pattern 1: Handle + return (most common)
result, err := compute()
if err != nil {
    return fmt.Errorf("compute: %w", err)
}

// ✅ Pattern 2: Log + fallback (khi có thể tiếp tục)
data, err := cache.Get(key)
if err != nil {
    slog.Warn("cache miss", "key", key, "err", err)
    data = fetchFromDB(key) // fallback
}

// ✅ Pattern 3: Retry on transient errors
for attempt := 0; attempt < 3; attempt++ {
    err = connect(addr)
    if err == nil {
        break
    }
    if !isRetryable(err) {
        return err
    }
    time.Sleep(backoff(attempt))
}

// ❌ Pattern: Ignore error — chỉ dùng khi có lý do rõ ràng
_ = f.Close()  // ❌ nên log nếu Close fail
// ✅ defer func() { if err := f.Close(); err != nil { log.Warn(...) } }()
```

## `if err != nil` — Góc Nhìn Đa Chiều

Go bị chỉ trích vì pattern `if err != nil` lặp đi lặp lại. Đây là các góc nhìn:

| Quan điểm | Lập luận |
|-----------|---------|
| ✅ Explicit | Error rõ ràng, không thể bỏ qua. Control flow tường minh. |
| ✅ No hidden control flow | Không bị "bắn ra" ngoài unexpected như exception |
| ⚠️ Verbose | Tốn dòng code, đặc biệt khi chain nhiều calls |
| ⚠️ Copy-paste | Dễ quên hoặc copy sai khi handle mỗi chỗ |
| Go 2 direction | Proposals cho `?` operator đang bàn (không ship) |

```go
// Go hiện tại — verbose nhưng rõ ràng
a, err := stepA()
if err != nil { return err }
b, err := stepB(a)
if err != nil { return err }
c, err := stepC(b)
if err != nil { return err }

// vs Rust — ? operator (Go không có)
// let c = step_c(step_b(step_a()?)?)?;
// Concise nhưng ẩn control flow

// Go pattern để giảm repetition: errgroup / pipeline
```

## Best Practices

```go
// 1. Thêm context vào error khi propagate
return fmt.Errorf("service.CreateUser: %w", err)

// 2. Không log VÀ return — chọn một
// ❌ double handling
log.Error("failed", "err", err)
return err  // caller cũng sẽ handle

// ✅ return lên tầng có context đủ để log
return fmt.Errorf("createUser %d: %w", id, err)

// 3. Sentinel errors phải có prefix theo package
var ErrInvalidInput = errors.New("mypackage: invalid input")

// 4. Kiểm tra error type trước khi access fields
var appErr *AppError
if errors.As(err, &appErr) {
    fmt.Println(appErr.HTTPCode)
}
```

## 📌 Tóm tắt

```
Error is a Value
│
├── error interface { Error() string }
│   ├── Mọi type có Error() string implement error
│   └── nil error = thành công
│
├── Check: if err != nil
│   ├── Explicit — không thể bỏ qua
│   └── vs Exception: tường minh, không hidden control flow
│
├── errors.Is(err, target)
│   ├── So sánh xuyên wrap chain (Unwrap)
│   └── Dùng cho sentinel errors
│
├── errors.As(err, &target)
│   ├── Extract type cụ thể từ chain
│   └── Dùng cho custom error struct
│
├── fmt.Errorf("context: %w", err)
│   ├── Wrap = thêm context + giữ original
│   └── Tạo chain: wrapping → errors.Is/As có thể traverse
│
├── Handle Patterns
│   ├── Return + wrap (propagate up)
│   ├── Log + fallback (recoverable)
│   └── Retry (transient errors)
│
└── Trade-offs
    ├── Verbose nhưng explicit
    ├── No hidden control flow vs Exception
    └── Sentinel vs Custom: simple flag vs rich metadata
```
