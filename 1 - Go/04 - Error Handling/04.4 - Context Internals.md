# 04.4 Context Internals

## Overview

`context` package cung cấp deadline, cancellation, và value propagation giữa các goroutines. Nó là **cầu nối** giữa request lifecycle và goroutine lifecycle.

```go
import "context"
```

### Context hierarchy

```
Background() → empty root context
    ├── WithCancel(parent) → cancelFunc
    ├── WithDeadline(parent, time)
    ├── WithTimeout(parent, duration)
    └── WithValue(parent, key, value)
```

> Mỗi context mới được tạo từ một parent → tạo thành cây. Khi parent bị cancel, tất cả children cũng bị cancel.

---

## Context Internals — Structs

Go không expose internal struct (trong internal/context package), nhưng concept tương đương:

```go
// cancelCtx — supports cancellation
type cancelCtx struct {
    contextKey  // "cancel" key
    mu       sync.Mutex
    done     chan struct{}  // closed when cancelled
    children map[cancelCtx]struct{}  // child contexts
    err      error          // set when cancelled
}

// deadlineCtx — cancelCtx + deadline
type deadlineCtx struct {
    cancelCtx
    deadline time.Time
}

// timerCtx — deadlineCtx + timeout
type timerCtx struct {
    deadlineCtx
    timer *time.Timer  // auto-cancel when deadline expires
}

// valueCtx — stores key-value pair
type valueCtx struct {
    contextKey  // "values" key (type-safe)
    value  any
    parent context.Context
}
```

## WithCancel — Cancellation propagation

```go
ctx, cancel := context.WithCancel(parent)
defer cancel() // ⚠️ Always pair — caller responsible for cancel

// Thực thi một operation bị cancel khi parent được cancel
go func() {
    select {
    case <-ctx.Done():
        return  // context cancelled
    case result := <-dataCh:
        process(result)
    }
}()

// Sau khi cancel được gọi:
// ctx.Err() == context.Canceled
```

### Channel semantics

```go
// ctx.Done() trả về <-chan struct{}
// Đây là closure, được closed khi context cancelled

// ⚠️ nil channel — never blocks, never returns
// Never store context in a struct field that might be nil

// ✅ Pattern: defer cancel để cleanup
func operation(ctx context.Context) error {
    ctx, cancel := context.WithCancel(ctx)
    defer cancel()

    // Nếu operation return trước khi done, defer cancel đảm bảo cleanup
    result, err := doWork(ctx)
    return err // cancel được gọi ngay khi function exit
}
```

## WithTimeout / WithDeadline — Time-bound cancellation

```go
// WithTimeout = shortcut cho WithDeadline(parent, time.Now().Add(duration))
ctx, cancel := context.WithTimeout(parent, 5*time.Second)
defer cancel()

// ⚠️ Timeout cho database query
rows, err := db.QueryContext(ctx, "SELECT ...")
// Nếu query chạy > 5s → ctx.Done() → rows.Close() → return error
```

### Timeout vs Deadline

```go
// Timeout — relative time
ctx, cancel := context.WithTimeout(parent, 100*time.Millisecond)

// Deadline — absolute time
deadline := time.Date(2024, 1, 15, 10, 0, 0, 0, time.UTC)
ctx, cancel := context.WithDeadline(parent, deadline)
```

### Automatic cancel behavior

```go
// TimerCtx auto-cancels khi deadline đến
// Điều này xảy ra background — goroutine đang block trên ctx.Done()
// sẽ được unblocked mà không cần caller gọi cancel

ctx, cancel := context.WithTimeout(context.Background(), 100*time.Millisecond)
defer cancel()

select {
case <-time.After(200 * time.Millisecond):
    fmt.Println("timeout expired")
    fmt.Println(ctx.Err()) // context.DeadlineExceeded
case <-ctx.Done():
    fmt.Println(ctx.Err()) // context.DeadlineExceeded
}
```

## WithCancelCause (Go 1.20+) — Cancellation with reason

```go
// CancelCause — lưu reason cho cancellation
ctx, cancel := context.WithCancelCause(parent)

// Khi cancel:
cancel(errors.New("database connection failed"))

// Đọc cause:
fmt.Println(ctx.Err())          // context canceled
fmt.Println(context.Cause(ctx)) // database connection failed
```

### Use case: propagate root cause up the stack

```go
func dbOperation(ctx context.Context) error {
    conn, err := db.Conn(ctx)
    if err != nil {
        cancelWithCause(ctx, fmt.Errorf("db pool exhausted: %w", err))
        return err
    }
    return nil
}

func cancelWithCause(ctx context.Context, err error) {
    // CancelCause requires Go 1.20+
    // For older versions: wrap in ctx.Err() string
    if c, ok := ctx.(interface{ Cancel(err error) }); ok {
        c.Cancel(err)
    }
}
```

## WithValue — Type-safe key-value propagation

### Key uniqueness requirement

```go
// ❌ Collision: dùng string key
ctx := context.WithValue(ctx, "user_id", "123")  // ❌ BAD
// Package A và B cùng dùng "user_id" → collision!

// ✅ Type-safe: tạo unexported key type
type userIDKey struct{} // unexported type — unique by identity
type requestIDKey struct{}

ctx = context.WithValue(ctx, userIDKey{}, "123")     // ✅ type-safe
ctx = context.WithValue(ctx, requestIDKey{}, "req-1")

// Đọc:
uid := ctx.Value(userIDKey{}).(string)  // ✅ type-safe
rid := ctx.Value(requestIDKey{}).(string)
```

### Key lookup algorithm

```go
// Value lookup tree:
ctx.Value(key):
  1. Check this context's key → match → return value
  2. Walk up parent chain → repeat
  3. Not found → return nil

// ⚠️ Shadowing: nếu child có cùng key type → child wins
childCtx := context.WithValue(parentCtx, key, "child-value")
parentCtx.Value(key)  // → "parent-value"
childCtx.Value(key)   // → "child-value" (shadows parent)
```

### Best practices

```go
// 1. Define key types in package — private scoped
package auth

type key struct{} // unexported, only this package can create/use it

// 2. Getter function với type-safe return
func UserID(ctx context.Context) (string, bool) {
    v := ctx.Value(key{})
    if v == nil { return "", false }
    return v.(string), true
}

// 3. Never store non-copyable objects (connections, goroutines)
// Context value copied by reference — doesn't affect lifetime
type ctxKey struct{}
ctx = context.WithValue(ctx, ctxKey{}, myDBPool)
// ⚠️ myDBPool won't be garbage-collected just because ctx is alive
// ⚠️ ctx doesn't extend lifetime of stored values — it only references them
```

## Context hierarchy lifecycle

```go
// Parent cancels → all children receive cancellation
root, cancelRoot := context.WithCancel(context.Background())
defer cancelRoot()

child1, cancel1 := context.WithCancel(root)
defer cancel1() // child1's own cancel

child2, cancel2 := context.WithTimeout(root, 5*time.Second)
defer cancel2()

// root cancel → child1.Done() AND child2.Done() both unblock
// child1 cancel → child1.Done() unblocks, root and child2 unchanged
// child2 timeout → child2.Done() unblocks, root and child1 unchanged
```

### Garbage collection of contexts

```go
// Context value memory management:
// - Value stored in valueCtx node
// - Node kept alive while context tree alive
// - When root context cancelled → entire tree eligible for GC
// - ⚠️ Values are not destroyed on cancel — GC timing is non-deterministic

// ✅ For deterministic cleanup: use explicit cleanup functions
// instead of relying on context cancellation
```

## Common patterns

### HTTP middleware — attach request context

```go
func tracingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        traceID := generateTraceID()
        ctx := context.WithValue(r.Context(), traceIDKey{}, traceID)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}

// In handler:
traceID := r.Context().Value(traceIDKey{}).(string)
```

### Goroutine-safe context propagation

```go
// ✅ CORRECT: pass context to goroutine
go func() {
    select {
    case <-ctx.Done():
        return // properly receives cancellation
    case <-time.After(1 * time.Hour):
        // This won't run if parent cancels (unless you want it to)
    }
}()

// ❌ WRONG: capture ctx in closure that outlives function
var bgCtx context.Context // ❌ global context
func() {
    bgCtx = ctx // ❌ context might be cancelled before use
}()
```

### Error handling with context

```go
// Wrap context errors with additional info
if ctx.Err() == context.DeadlineExceeded {
    return fmt.Errorf("operation timed out after 5s: %w", ctx.Err())
}
if ctx.Err() == context.Canceled {
    return fmt.Errorf("operation cancelled: %w", ctx.Err())
}

// Full error chain
err := db.QueryContext(ctx, "SELECT ...")
if ctx.Err() != nil {
    return fmt.Errorf("db query failed (ctx error): %w", ctx.Err())
}
if err != nil {
    return fmt.Errorf("db query failed: %w", err)
}
```

## Context usage rules

| Rule | Reason |
|------|--------|
| Pass context as first parameter | Convention — code reviews know where to look |
| Never store in struct fields | Can become nil, breaks cancellation propagation |
| Check `ctx.Err()` before returning | Avoid masking root cause |
| Always `defer cancel()` in functions | Prevent goroutine leaks |
| Use typed keys, not strings | Prevent package key collision |
| Don't use context for values that outlive request | Use explicit dependencies instead |

```
│   ├── deadlineCtx — cancelCtx + deadline time
│   ├── timerCtx — deadlineCtx + auto-cancel timer
│   └── valueCtx — key/value stored in node, parent pointer

│   ├── WithCancel → cancelFunc → ctx.Done() closes
│   ├── WithTimeout / WithDeadline → auto-cancel at time
│   └── WithCancelCause (Go 1.20+) → track root cause
├── Value propagation
│   ├── Key uniqueness: unexported key type (no string collision)
│   ├── Lookup: walk parent chain, child shadows parent
│   └── ⚠️ Context doesn't extend value lifetime — GC only
├── Lifecycle
│   ├── Root cancel → all children cancelled
│   ├── Individual child cancel → only that branch
│   └── Timer/timeout → automatic background cancel
└── Rules
    ├── First param, never struct field
    ├── Always defer cancel(), check ctx.Err()
    └── Typed keys, no string keys
```
