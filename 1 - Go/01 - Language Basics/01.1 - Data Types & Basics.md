# 01.1 Data Types & Basics



- **Dynamic size** — kích thước có thể thay đổi
- **Là một struct**:

```go
type slice struct {
    ptr *T   // pointer tới underlying array
    len int  // số phần tử đang dùng
    cap int  // sức chứa tối đa
}
```

- `len`: số phần tử đang dùng
- `cap`: sức chứa tối đa trước khi realloc
- **Reference semantics** — points to an underlying array

### Cheap to pass around

```go
func g(s []int) {
    s[0] = 100
}

func main() {
    s := []int{1, 2, 3}
    g(s)
    fmt.Println(s) // [100 2 3] — caller thấy thay đổi
}
```

### ⚠️ Risk

- Nếu function modify elements → caller thấy thay đổi
- Nếu `append` triggers reallocation → local pointer thay đổi → **caller header vẫn trỏ old array**

## Nil vs Empty

| Type | Ý nghĩa |
|------|---------|
| `nil slice` (`var s []int`) | Chưa init, không có underlying data |
| `empty slice` (`s := []int{}`) | Đã init, nhưng chứa 0 phần tử |

---

## Value Type vs Reference Type

| | Value Type | Reference Type |
|---|---|---|
| Copy | Copy toàn bộ dữ liệu | Copy pointer → shared underlying data |
| Khi truyền | Tạo bản copy mới | Chia sẻ data gốc |
| Ví dụ | Array, int, struct | Slice, map, channel, pointer |

## Struct Embedding vs Inheritance

Go **không có inheritance**. Thay vào đó dùng:

### Struct Embedding

```go
type Animal struct {
    Name string
}

func (a Animal) Speak() {
    fmt.Println("Animal speaks")
}

type Dog struct {
    Animal // embedded struct — như "extends"
}
```

- Gọi `Dog{Name: "Buddy"}.Speak()` → `Animal.Speak()`
- **Shadowing**: nếu Dog có field trùng tên với Animal → dùng Dog's field

### So sánh

| | Inheritance (Java/C++) | Struct Embedding (Go) |
|---|---|---|
| Syntax | `class Dog extends Animal` | `Dog Animal` |
| Multiple | Không | Có (multiple embedding) |
| Method override | Override keyword | Shadowing (không polymorphism) |

## Interface{}

- Định nghĩa tập method
- `interface{}` = `any` (Go 1.18+)
- **Mọi type** đều implement `interface{}`

```go
var x interface{} = "hello"  // string implements interface{}
```

## Type Assertion vs Type Switch

### Type Assertion — check 1 type cụ thể

```go
func handle(x any) {
    s, ok := x.(string)
    if !ok {
        fmt.Println("not a string")
        return
    }
    fmt.Println("string:", s)
}
```

### Type Switch — check nhiều type

```go
switch v := x.(type) {
case int:
    fmt.Println("int:", v)
case string:
    fmt.Println("string:", v)
default:
    fmt.Println("unknown type")
}
```

## Array — Fixed-Size Storage

```go
// Array: kích thước cố định, là value type
var a [5]int              // zero-initialized: [0 0 0 0 0]
b := [3]string{"a", "b", "c"}
c := [...]int{1, 2, 3}   // compiler đếm elements

// Array là VALUE type — copy toàn bộ khi assign/pass
x := [3]int{1, 2, 3}
y := x       // y là bản copy hoàn toàn
y[0] = 99
fmt.Println(x[0]) // 1 — x không bị ảnh hưởng
```

### Array vs Slice — Trade-off

| | Array `[N]T` | Slice `[]T` |
|---|---|---|
| Size | Fixed, compile-time | Dynamic |
| Type | Value type | Reference type (ptr+len+cap) |
| Pass | Copy toàn bộ | Copy header (pointer shared) |
| Allocation | Stack (nếu nhỏ) | Header stack, backing array heap |
| Use case | Crypto keys, fixed buffers, no GC | General collections |

> **Khi nào dùng Array**: khi biết size tại compile-time, muốn tránh heap allocation, ví dụ: `[32]byte` cho SHA-256 key, `[4]byte` cho IPv4.

## Map Internals (hmap)

Go map là **hash table** — không phải tree (như `std::map` của C++):

```go
// Internal representation (simplified)
type hmap struct {
    count      int       // số key-value pairs
    buckets    *bmap     // array of 8-slot buckets
    oldbuckets *bmap     // non-nil khi incremental rehashing
    // ...
}
```

- Mỗi **bucket** chứa tối đa **8 key-value pairs**
- **Load factor ~6.5** → khi avg > 6.5 elements/bucket → grow 2x
- Rehashing là **incremental** (không blocking)

```go
m := make(map[string]int, 100) // pre-allocate 100 slots → giảm rehashing

// ⚠️ Map KHÔNG thread-safe!
// concurrent reads: OK
// concurrent read + write: PANIC (fatal error: concurrent map read and map write)

// Zero value access — không panic
val := m["missing"]          // val = 0
val, ok := m["missing"]      // ok = false

delete(m, "key")             // no-op nếu key không tồn tại
```

### Map iteration — Non-deterministic

```go
// ⚠️ Range order là RANDOM — by design (chống abuse)
for k, v := range m {
    // thứ tự khác nhau mỗi lần chạy
}

// ✅ Sorted iteration
keys := make([]string, 0, len(m))
for k := range m {
    keys = append(keys, k)
}
sort.Strings(keys)
for _, k := range keys {
    fmt.Println(k, m[k])
}
```

### Map concurrency trade-offs

| | `map` | `sync.Map` | `map + Mutex` | `map + RWMutex` |
|---|---|---|---|---|
| Thread-safe | ❌ | ✅ | ✅ | ✅ |
| Read perf | ✅✅ | ✅ (R-heavy) | ⚠️ lock | ✅ concurrent reads |
| Write perf | ✅✅ | ⚠️ slower | ✅ | ⚠️ exclusive |
| Use case | Single goroutine | Many goroutines, R>>W | Balanced R/W | Read-heavy |

## String Internals

String trong Go là **immutable byte sequence** (không phải `[]byte`):

```go
// Internal representation
type StringHeader struct {
    Data uintptr  // pointer tới byte array
    Len  int
}
```

```go
s := "hello 🌍"

len(s)           // 10 bytes  (🌍 = 4 bytes UTF-8)
len([]rune(s))   // 7 characters

// String là immutable — compile error nếu modify
// s[0] = 'H'  ❌

// Convert (mỗi convert = 1 allocation)
b := []byte(s)   // copy: string → mutable bytes
s2 := string(b)  // copy: bytes → new string

// ✅ strings.Builder cho string concatenation hiệu quả
var sb strings.Builder
for _, word := range words {
    sb.WriteString(word)
    sb.WriteByte(' ')
}
result := sb.String() // 1 allocation thay vì N
```

### Rune vs Byte

```go
// Iterate by BYTE
for i := 0; i < len(s); i++ {
    fmt.Printf("%x ", s[i])  // raw bytes
}

// Iterate by CHARACTER (rune = Unicode code point)
for i, r := range s {  // range tự decode UTF-8
    fmt.Printf("%d: %c\n", i, r)
}
```

### String Trade-offs

| | `string` | `[]byte` |
|---|---|---|
| Immutable | ✅ Safe to share | ❌ Mutable |
| Pass cost | ✅ Cheap (pointer+len) | ✅ Cheap (3 words) |
| Modify | ❌ (copy required) | ✅ In-place |
| Map key | ✅ | ❌ (not comparable) |
| Network I/O | ⚠️ Convert needed | ✅ Direct |

## Pointer

Go có pointer nhưng **không có pointer arithmetic** (không như C/C++):

```go
x := 42
p := &x           // p là *int, lấy địa chỉ của x
fmt.Println(*p)   // 42 — dereference
*p = 100          // modify x qua pointer
fmt.Println(x)    // 100

// Nil pointer
var ptr *int      // nil
// *ptr = 1       // ❌ panic: nil pointer dereference

// new() — allocate zeroed value, return pointer
p2 := new(int)    // *int, *p2 = 0
```

### When to use Pointer vs Value

| Scenario | Pointer | Value |
|----------|---------|-------|
| Modify caller's data | ✅ | ❌ |
| Large struct (>= ~64B) | ✅ (avoid copy) | ❌ inefficient |
| Nullable (optional field) | ✅ `nil` check | ❌ |
| Small primitive (int, bool) | ❌ overkill | ✅ |
| Immutable / read-only | ❌ | ✅ |
| Map value (mutate in-place) | ✅ | ❌ (copy in map) |

```go
// ⚠️ Map value không thể mutate trực tiếp
type Counter struct{ n int }
m := map[string]Counter{"a": {n: 1}}
// m["a"].n++  ❌ cannot assign to m["a"].n

// ✅ Fix 1: use pointer
mp := map[string]*Counter{"a": {n: 1}}
mp["a"].n++  // OK

// ✅ Fix 2: copy, modify, reassign
c := m["a"]
c.n++
m["a"] = c
```

## Method Sets & Receiver Types

```go
type Counter struct{ count int }

// Value receiver — works on T và *T
func (c Counter) Value() int { return c.count }

// Pointer receiver — chỉ works trên *T
func (c *Counter) Increment() { c.count++ }
```

### Method Set Rules

| Type | Methods available |
|------|--------------------|
| `T` (value) | Chỉ methods với **value receiver** |
| `*T` (pointer) | Value receiver **+** pointer receiver |

```go
c := Counter{}
c.Increment()       // ✅ Go auto-takes address: (&c).Increment()

// Interface matching theo Method Set
type Incrementer interface { Increment() }
var i Incrementer = &Counter{}  // ✅ *Counter implements (has pointer receiver)
// var i Incrementer = Counter{} ❌ Counter does NOT implement
```

### Trade-off: Value Receiver vs Pointer Receiver

| | Value Receiver | Pointer Receiver |
|---|---|---|
| Mutation | ❌ works on copy | ✅ modifies original |
| Nil safe | ❌ panic | ✅ can check `c == nil` |
| Concurrency | ✅ no shared state | ⚠️ cần mutex nếu mutate |
| Interface (T) | ✅ T implements | ❌ only *T implements |
| Small struct | ✅ prefer | — |
| Large struct | — | ✅ avoid copy |

> **Rule of thumb**: Nếu *bất kỳ* method nào cần pointer receiver → dùng pointer receiver cho **TẤT CẢ** methods của type đó (consistency).

## Interface Internals (iface vs eface)

Interface trong Go là **fat pointer** gồm 2 trường:

```go
// Non-empty interface: iface
type iface struct {
    tab  *itab          // type descriptor + method table
    data unsafe.Pointer // pointer tới actual value
}

// Empty interface: eface (any / interface{})
type eface struct {
    _type *_type        // type info only
    data  unsafe.Pointer
}
```

### ⚠️ Interface Nil Trap

```go
func returnsError() error {
    var p *MyError = nil
    return p  // ❌ BUG: trả về NON-NIL error!
              // iface{tab: *MyError_tab, data: nil} ≠ nil error
}

if err := returnsError(); err != nil {
    fmt.Println("got error!") // ← sẽ in dù *MyError là nil
}

// ✅ Fix: return untyped nil
func returnsError() error {
    var p *MyError = nil
    if p == nil {
        return nil  // return true nil interface
    }
    return p
}
```

### Interface Performance

```go
// Interface dispatch có overhead: vtable lookup + indirect call
// Direct call:    ~1ns
// Interface call: ~2-5ns

// ⚠️ Trong hot loop với 10^9 iterations — measurable!
for i := 0; i < 1e9; i++ {
    iface.Method() // indirect call mỗi iteration
}

// ✅ Type assertion để escape interface trong hot path
if concrete, ok := iface.(*ConcreteType); ok {
    for i := 0; i < 1e9; i++ {
        concrete.Method() // direct call — no indirection
    }
}
```

### Trade-off: Interface vs Concrete Type

| | Interface | Concrete Type |
|---|---|---|
| Flexibility | ✅ Polymorphism, DI | ❌ Fixed |
| Performance | ⚠️ Indirect dispatch, boxing | ✅ Direct call |
| Testability | ✅ Easy mock | ⚠️ Harder to mock |
| Escape analysis | ⚠️ Often forces heap | ✅ May stay on stack |
| Compile-time check | ✅ | ✅ Stricter |

> **Go Proverb**: *"Accept interfaces, return concrete types."* — Felgo & Rob Pike

### Compile-time Interface Assertion

```go
// ✅ Best practice: assert interface implementation tại compile time
var _ io.Writer = (*MyWriter)(nil)  // compile error nếu không implement
var _ io.Reader = (*MyReader)(nil)
```
