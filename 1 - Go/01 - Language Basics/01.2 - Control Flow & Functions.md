# 01.2 Control Flow & Functions



## Defer

### How it works

`defer` schedules a function call to run **AFTER** the surrounding function returns.

```go
func main() {
    defer fmt.Println("world")
    fmt.Println("hello")
}
// Output:
// hello
// world
```

### Execution order — LIFO (Stack)

```go
func main() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
    fmt.Println("0")
}
// Output: 0 → 3 → 2 → 1
```

### Defer vs Finally

| | Defer | Finally |
|---|---|---|
| Scope | Function scope | Block scope |
| Used with | Any function | try-catch (other languages) |
| Flexibility | More flexible | Block-scoped |

## Closure

A closure is a **function that captures variables from its surrounding scope**.

### Closures capture variables, not values

```go
func main() {
    x := 10
    f := func() {
        fmt.Println(x) // captures x by reference
    }
    x = 20
    f() // output: 20 — closure thấy giá trị mới nhất
}
```

### ⚠️ Loop pitfall

```go
func wrapOps(ops []func(int) int) []func(int) int {
    result := make([]func(int) int, len(ops))
    for i, op := range ops {
        result[i] = func(x int) int { return op(x) }
        // i và op bị capture → dùng giá trị cuối cùng sau vòng for
    }
    return result
}

// Fix: copy biến trong vòng for
for i, op := range ops {
    i := i // shadow biến loop
    op := op
    result[i] = func(x int) int { return op(x) }
}
```

## Pass by Value or Reference

Go is **pass by value only**. Sometimes the value being copied is a pointer:

```go
// Truyền pointer → pointer được copy, nhưng pointed-to data vẫn shared
func swap(a, b *int) {
    *a, *b = *b, *a // swap qua pointer ✓
}

// Truyền giá trị thường → copy hoàn toàn
func increment(n int) {
    n++ // chỉ thay đổi bản copy local
}
```

## Named Return Value

Go's return values may be **named**. They are treated as variables defined at the top of function:

```go
func split(sum int) (x, y int) {
    x = sum * 4 / 9   // x, y đã được khai báo
    y = sum - x
    return // naked return — return x, y
}
```

- Dùng để **document** ý nghĩa return value
- Tránh naked return trong function dài (vì đọc khó)

## for range Loop Semantics (Go 1.22 change)

### ⚠️ Go < 1.22 — Loop variable semantics

> **CRITICAL**: Trước Go 1.22, loop variable được **reused** across iterations — closure capture gây bug.

```go
// ❌ BUG trong Go < 1.22 — tất cả goroutines share cùng `i`
for i, url := range urls {
    go fetch(url) // BUG: captures loop variable by reference
    // Tất cả goroutines thấy cùng giá trị (url cuối cùng hoặc zero)
}

// ✅ FIX — capture giá trị trong closure
for i, url := range urls {
    url := url  // shadow — copy giá trị hiện tại
    go fetch(url)
}

// Hoặc truyền argument
for i, url := range urls {
    go func(u string) { fetch(u) }(url) // truyền giá trị, không capture
}
```

### Go ≥ 1.22 — Per-iteration scope

> Từ Go 1.22, mỗi iteration có **biến loop riêng** — không cần shadow nữa.

```go
// ✅ Go 1.22+: mỗi goroutine capture giá trị của iteration hiện tại
for i, url := range urls {
    go fetch(url) // CORRECT: mỗi goroutine có url riêng
}

// Pre-1.22: vẫn cần shadow/closure argument
```

## Function as First-Class Citizen

Function trong Go là **first-class value** — có thể gán biến, pass làm argument, return từ function:

```go
// Gán function vào biến
greet := func(name string) string {
    return "Hello, " + name
}
fmt.Println(greet("Go")) // "Hello, Go"

// Pass function làm argument
func apply(fn func(int) int, vals []int) []int {
    result := make([]int, len(vals))
    for i, v := range vals {
        result[i] = fn(v)
    }
    return result
}
doubled := apply(func(x int) int { return x * 2 }, []int{1, 2, 3})
// [2, 4, 6]

// Return function từ function (factory pattern)
func multiplier(factor int) func(int) int {
    return func(x int) int {
        return x * factor  // factor captured từ outer scope
    }
}
double := multiplier(2)
triple := multiplier(3)
fmt.Println(double(5), triple(5)) // 10 15
```

## Variadic Functions

```go
// ...T — nhận zero hoặc nhiều arguments của type T
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

sum()           // 0
sum(1, 2, 3)    // 6

// Spread: truyền slice với ...
nums := []int{1, 2, 3, 4}
sum(nums...)    // 10

// Mixed: fixed + variadic
func logf(level string, format string, args ...any) {
    fmt.Printf("[%s] "+format+"\n", append([]any{level}, args...)...)
}
```

### Variadic Trade-offs

| | Pros | Cons |
|---|---|---|
| Flexibility | ✅ Gọi với 0..N args | ⚠️ Creates `[]T` slice → heap alloc |
| API ergonomics | ✅ Cleaner callsite | ⚠️ Không type-safe nếu dùng `...any` |
| Hot path | ❌ Avoid | ✅ Use fixed-arg overloads |

> **Note**: Với Go 1.21+, `min()`, `max()` là built-in variadic, zero allocation (compiler intrinsic).

## `init()` Function

```go
// init() chạy tự động TRƯỚC main(), không cần gọi
// Không có arguments, không có return value
package mypackage

var config Config

func init() {
    config = loadConfig()
}

func init() {
    // ✅ Nhiều init() được phép trong cùng file/package
    setupLogging()
}
```

### init() Execution Order

```
Import dependency init() → current package init() → main()

Trong package: top-down trong file, alphabetical giữa files
→ Không nên depend vào order giữa các file
```

### Trade-off: init() vs Alternatives

| | `init()` | Constructor func `New...()` | `sync.Once` |
|---|---|---|---|
| Automatic | ✅ | ❌ Caller phải gọi | ❌ Caller trigger |
| Testability | ❌ Hidden side effects | ✅ Explicit | ✅ |
| Error handling | ❌ Không return error | ✅ | ✅ |
| Lazy init | ❌ Always runs | ❌ Eager | ✅ On first use |
| Preference | Legacy/plugin | ✅ Production | ✅ Singleton |

## Blank Identifier `_`

```go
// 1. Bỏ qua giá trị trả về
val, _ := strconv.Atoi("42")  // bỏ qua error ⚠️ dangerous

// 2. Bỏ qua loop index
for _, v := range items { ... }

// 3. Import for side effects only (chỉ chạy init())
import _ "net/http/pprof"       // register pprof handlers
import _ "github.com/lib/pq"    // register postgres driver

// 4. ✅ Compile-time interface assertion
var _ io.Reader = (*MyReader)(nil)  // compile error nếu không implement
var _ io.Writer = (*MyWriter)(nil)
```

## Multiple Return Values — Patterns

```go
// Pattern 1: value + error (most common)
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// Pattern 2: comma-ok idiom (type assertion, map lookup, channel)
val, ok := m["key"]          // map
ch, ok := <-channel          // channel
concrete, ok := iface.(Type) // type assertion

// Pattern 3: Named return + defer for cleanup/error wrapping
func readFile(path string) (content []byte, err error) {
    f, err := os.Open(path)
    if err != nil {
        return nil, fmt.Errorf("open: %w", err)
    }
    defer func() {
        if cerr := f.Close(); cerr != nil && err == nil {
            err = cerr  // ✅ modify named return từ defer
        }
    }()
    return io.ReadAll(f)
}
```

### Trade-off: Named Return vs Explicit Return

| | Named Return | Explicit Return |
|---|---|---|
| Naked return | ✅ Allowed | ❌ |
| Defer can modify | ✅ | ❌ |
| Documentation | ✅ Self-documenting signature | ⚠️ Needs comment |
| Clarity (short func) | ✅ OK | ✅ OK |
| Clarity (long func) | ❌ Confusing | ✅ Prefer explicit |
| Pitfall | ⚠️ Shadowing via `:=` | — |

```go
// ⚠️ Named return shadowing pitfall
func tricky() (result int, err error) {
    if result, err := compute(); err != nil {  // ❌ shadows named return!
        return 0, err
    }
    // result từ compute() bị lost — named return result vẫn = 0
    return
}

// ✅ Fix: không dùng := khi muốn assign named return
func correct() (result int, err error) {
    result, err = compute()  // = không shadow
    return
}
```

## Defer — Cost & Trade-offs

```go
// Go 1.14+: "open-coded defer" — compiler eliminates overhead
// cho defer không nằm trong loop và số lượng nhỏ
func read() error {
    f, _ := os.Open("file")
    defer f.Close()  // ✅ near-zero overhead (open-coded)
    return process(f)
}

// ⚠️ Defer trong loop: mỗi iteration tạo defer frame
for i := 0; i < 1e6; i++ {
    f, _ := os.Open(name(i))
    defer f.Close()  // ❌ 1e6 deferred calls! không release ngay
}

// ✅ Fix: dùng anonymous function để release trong loop
for i := 0; i < 1e6; i++ {
    func() {
        f, _ := os.Open(name(i))
        defer f.Close()  // OK — runs at end of anonymous func, not main func
    }()
}
```

### Defer argument evaluation timing

```go
x := 10
defer fmt.Println(x)  // ✅ argument evaluated NOW → prints 10
x = 20
// Output: 10

defer func() {
    fmt.Println(x)  // ✅ closure evaluates at EXECUTION time → prints 20
}()
x = 20
// Output: 20
```
