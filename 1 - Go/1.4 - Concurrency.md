## Go Scheduler (G-M-P Model)

### Components

| | Mô tả |
|---|---|
| **G** (Goroutine) | Goroutine đang chạy, chứa stack & instruction pointer |
| **M** (Machine) | OS thread thực sự |
| **P** (Processor) | Resource để schedule G (số lượng = GOMAXPROCS) |

### Rules

1. G chạy trên M
2. M cần P để execute Go code
3. GOMAXPROCS = số lượng P

```go
runtime.GOMAXPROCS(4) // Enable parallelism với 4 CPU cores
```

---

### Missions

#### Multiplexing

Reflex N goroutines → M OS threads → P processors

#### Work Stealing

Khi P hết việc → steal G từ P khác → **Load balance**

#### Preemption

Preemption là ability của Go runtime để **pause một running goroutine** để others can run.

- Without preemption: long running goroutines can **starve** others
- Scheduler becomes **unfair**

**Go 1.14+**: sysmon gửi signal để preempt G chạy > 10ms

#### Hand off (Advanced)

Kỹ thuật nâng cao — khi M bị block (syscall), scheduler có thể "handoff" P sang M khác.

---

## gopark

`gopark` là runtime operation để **park (block) current goroutine**:

- Goroutine không runnable
- **Không consume CPU**
- Nằm trong waiting state

**Tự động gọi khi:**
- Channel receive → không có data
- Channel send → buffer full, không có receiver
- Mutex.Lock() → lock không available
- `runtime.Gosched()` → chủ động yield

## Context Switch Cost

### Cheap, nhưng có limit

- Mỗi goroutine có **stack** (dù nhỏ)
- **Stack scanning** during GC → overhead
- Goroutine không hoàn toàn miễn phí

### So sánh

| | Goroutine | OS Thread |
|---|---|---|
| Switch location | User-space | Kernel |
| Syscall | ❌ Không | ✅ Có |
| Stack size | ~2KB | 1–8MB |
| Speed | ~ns | ~μs |
