# 03.2 Channel & Select

## Channel

### Core concept

Channel là một **hchan struct** được bảo vệ bởi mutex. Nó chứa:

- **Ring buffer** — circular buffer để lưu messages
- **sendq** — linked list of goroutines waiting to send
- **recvq** — linked list of goroutines waiting to receive

```
hchan struct
├── buf     unsafe.Pointer  // ring buffer
├── sendq   waitq           // waiting senders
├── recvq   waitq           // waiting receivers
├── lock    mutex           // bảo vệ tất cả fields
└── ...
```

**Sending = copy operation** từ một goroutine's stack sang another (hoặc buffer).

---

## Unbuffered vs Buffered

### Unbuffered

```go
ch := make(chan int) // size = 0
```

- Send **block** cho đến khi có receiver
- Receive **block** cho đến khi có sender
- **Synchronous rendezvous** — "handshake" giữa sender và receiver

### Buffered

```go
ch := make(chan int, 10) // buffer size = 10
```

- Send block khi buffer **full**
- Receive block khi buffer **empty**
- Vẫn block nếu buffer đầy/trống

---

## Channel Direction

```go
// Send-only
func sender(ch chan<- int) {
    ch <- 42
}

// Receive-only
func receiver(ch <-chan int) {
    v := <-ch
}

// Bidirectional (default)
func both(ch chan int) {
    ch <- 1
    <-ch
}
```

---

## Closed Channel

| Operation | Khi closed |
|-----------|-----------|
| **Read** `v, ok := <-ch` | `ok = false` nếu buffer rỗng |
| **Write** `ch <- v` | **Panic** |
| **Close** `close(ch)` | OK, nhưng **chỉ 1 lần** |

```go
// Safe receive pattern
for v, ok := <-ch; ok; v, ok = <-ch {
    fmt.Println(v)
}
```

## Select

### How it works

Select cho phép goroutine **wait on multiple channel operations** cùng lúc.

```go
select {
case v := <-ch1:
    fmt.Println("Received from ch1:", v)
case ch2 <- 42:
    fmt.Println("Sent to ch2")
case <-time.After(time.Second):
    fmt.Println("Timeout")
}
```

### Blocking vs Non-blocking

| Type | Code | Behavior |
|------|------|----------|
| **Blocking** | `select {}` | Block vĩnh viễn |
| **Non-blocking** | `select { default: ... }` | Execute default immediately |
| **With timeout** | `case <-time.After(...)` | Block với timeout |

```go
// Non-blocking
select {
case v := <-ch:
    fmt.Println(v)
default:
    fmt.Println("No data available")
}
```

---

## Fairness

When multiple channels have trigger → **Go chose randomly a channel to get data**.

→ **Không guaranteed fairness** → có thể cause starvation.

---

## Nil Channel in Select

### Behavior

| Channel state | Read `<-ch` | Send `ch <- v` |
|---------------|-------------|----------------|
| Open (has data) | Returns data | Sends data |
| Open (empty) | **Block** | **Block** |
| **nil** | **Block forever** | **Block forever** |
| Closed (empty) | Returns zero, `ok=false` | **Panic** |

### ⚠️ Key insight

> `nil` channel trong `select` → **case bị IGNORED** (không bao giờ được chọn).

```go
// ✅ Dynamically enable/disable a case
select {
case v := <-ch1:
    fmt.Println("ch1:", v)
case v, ok := <-ch2:
    if !ok {
        fmt.Println("ch2 closed")
        break
    }
    fmt.Println("ch2:", v)
default:
    fmt.Println("No data")
}

// ✅ Disable a case by setting to nil
ch := chan int(nil) // nil channel

select {
case <-time.After(100 * time.Millisecond):
    fmt.Println("timeout")
case v := <-ch: // NEVER fires — ch is nil
    fmt.Println("never")
}
// → prints "timeout" after 100ms
```

### Use case: Conditional signal

```go
// Có thể dynamic control channel participation trong select
func worker(jobs <-chan int, results chan<- int, stop <-chan struct{}) {
    for {
        select {
        case <-stop:          // always checked
            return
        case j := <-jobs:    // checked only if jobs != nil
            results <- j * 2
        }
        // jobs == nil → case bị ignored → loop continue forever?
        // ⚠️ Cẩn thận: cần exit condition khác
    }
}

// Usage
jobs := make(chan int, 10)
stop := make(chan struct{})

go worker(jobs, results, stop) // jobs != nil → process
close(stop)                      // stop signal → worker exits
```

## errgroup for Pipelines

### Package

`golang.org/x/sync/errgroup` — quản lý goroutines với **shared error propagation** và **cancellation**.

```go
import "golang.org/x/sync/errgroup"

func fetchAll(ctx context.Context, urls []string) ([]string, error) {
    g, ctx := errgroup.WithContext(ctx)

    results := make([]string, len(urls))

    for i, url := range urls {
        i, url := i, url // pin loop variables
        g.Go(func() error {
            resp, err := http.Get(url)
            if err != nil {
                return fmt.Errorf("fetch %s: %w", url, err)
            }
            defer resp.Body.Close()
            body, err := io.ReadAll(resp.Body)
            results[i] = string(body)
            return nil
        })
    }

    if err := g.Wait(); err != nil {
        return nil, err // first error cancels all
    }
    return results, nil
}
```

### vs sync.WaitGroup

| Feature | errgroup | sync.WaitGroup |
|---------|----------|----------------|
| Error propagation | ✅ First error | ❌ |
| Context cancellation | ✅ Automatic on first error | ❌ |
| Early exit | ✅ Cancel on error | ❌ (must check manually) |
| Return value | ✅ Collected results | ❌ |
| Simplicity | ⚠️ | ✅ (for simple tasks) |

### Bounded Concurrency với semaphore

```go
func boundedFetch(ctx context.Context, urls []string, concurrency int) error {
    g, ctx := errgroup.WithContext(ctx)
    sem := make(chan struct{}, concurrency) // semaphore

    for _, url := range urls {
        url := url
        g.Go(func() error {
            sem <- struct{}{}         // acquire
            defer func() { <-sem }()   // release

            resp, err := http.Get(url)
            if err != nil {
                return err
            }
            resp.Body.Close()
            return nil
        })
    }

    return g.Wait()
}
```

## Common Patterns

### Fan-out / Fan-in

```go
// Fan-out: nhiều goroutine đọc từ 1 channel
for i := 0; i < n; i++ {
    go worker(in)
}

// Fan-in: 1 goroutine gộp nhiều channel
merge := func(cs ...<-chan int) <-chan int {
    var wg sync.WaitGroup
    out := make(chan int)
    for _, c := range cs {
        wg.Add(1)
        go func(ch <-chan int) {
            for v := range ch { out <- v }
            wg.Done()
        }(c)
    }
    go func() { wg.Wait(); close(out) }()
    return out
}
```

### Pipeline

```go
gen := func(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums { out <- n }
        close(out)
    }()
    return out
}

sq := func(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        for n := range in { out <- n * n }
        close(out)
    }()
    return out
}
```

## 📌 Tóm tắt

```
Channel & Select
│
├── Channel (hchan)
│   ├── Ring buffer + sendq + recvq + mutex
│   ├── Unbuffered: synchronous rendezvous
│   ├── Buffered: async với back-pressure
│   ├── Direction: chan-, -chan
│   ├── Closed: read OK (ok=false), write panic
│   ├── Nil: block forever (skipped in select)
│   └── Send = copy operation
│
├── Select
│   ├── Đợi nhiều channel cùng lúc
│   ├── Blocking: không có default
│   ├── Non-blocking: có default
│   ├── Timeout: time.After
│   ├── Nil channel: case SILENTLY SKIPPED
│   └── Random khi multiple ready
│
└── errgroup
    ├── goroutine pool với shared error
    ├── WithContext: cancel on first error
    ├── Semaphore pattern: bounded concurrency
    └── vs WaitGroup: error-aware
```
