# 02.3 The unsafe Package

## Overview

`unsafe` package cho phép绕过 Go's type safety để thực hiện những thao tác không thể làm được bằng safe Go. Tên nó nói lên tất cả: **sử dụng có thể gây undefined behavior, break compatibility, và bugs khó debug**.

```go
import "unsafe"
```

> `unsafe` được maintain bởi Go team — nó không phải third-party package hay "hack" nào đó. Nhưng code dùng nó KHÔNG có Go 1 compatibility guarantee cho các internal details.

### Các functions

```go
unsafe.Sizeof(x any)  uintptr  // kích thước của type (không tính data thực)
unsafe.Alignof(x any) uintptr // alignment requirement
unsafe.Offsetof(x any) uintptr // offset của field trong struct
unsafe.Pointer            // generic pointer type
```

## unsafe.Pointer — Conversions across types

`unsafe.Pointer` là **typed pointer** nhưng cho phép convert giữa các pointer types. Đây là cầu nối an toàn duy nhất trong unsafe.

```go
// Pointer conversion rules (3 rules của Go):
// 1. *T1 → unsafe.Pointer → *T2  (any pointer type ↔ unsafe.Pointer)
// 2. uintptr → unsafe.Pointer → *T  (integer → pointer)
// 3. unsafe.Pointer → *T  (back to typed pointer)

// ✅ Chiều 1: Convert []byte → string (zero-copy)
func ByteSliceToString(b []byte) string {
    // ⚠️ Chỉ dùng khi b đã immutable và sẽ không bị modify sau đó
    return *(*string)(unsafe.Pointer(&b))
}

// ✅ Chiều 2: Convert string → []byte (read-only view)
func StringToByteSlice(s string) []byte {
    return *(*[]byte)(unsafe.Pointer(&s))
}

// ⚠️ Cả hai đều tạo zero-copy view, không allocate mới
// ⚠️ Điều này chỉ an toàn khi data backing không bị thay đổi
```

### String internals để hiểu conversion

```go
// Go string internals (immutable, no pointer to slice header)
type string struct {
    ptr *byte  // pointer to backing array (read-only)
    len int
}

// Go slice internals
type slice struct {
    ptr   *byte  // pointer to backing array
    len   int
    cap   int
}

// ByteSliceToString:
//   s.ptr = b.ptr
//   s.len = b.len   (string len = cap của slice)

// ⚠️ Caveat: string backed by slice's array
// Nếu slice reallocates → string pointer trỏ vào memory không valid
```

## Sizeof, Alignof, Offsetof

### Sizeof — size of a type

```go
unsafe.Sizeof(int64(0))    // 8 bytes (platform-independent)
unsafe.Sizeof(bool(false))  // 1
unsafe.Sizeof(string(""))  // 16 (ptr+len on 64-bit)
unsafe.Sizeof([]int(nil)) // 24 (ptr+len+cap on 64-bit)

// ⚠️ Sizeof không đếm data mà type trỏ đến (nếu có)
// Nên nếu type là con trỏ, nó chỉ trả về kích thước pointer (8 bytes)
unsafe.Sizeof([]byte{})    // 24 — slice header, không phải []byte content
```

### Alignof — alignment requirement

```go
unsafe.Alignof(int64(0))   // 8
unsafe.Alignof(int32(0))   // 4
unsafe.Alignof(float64(0)) // 8
unsafe.Alignof(bool(false)) // 1
unsafe.Alignof(string(""))   // 8 (max of fields)
unsafe.Alignof(struct{}{})  // 1 (empty struct = 0 bytes, alignment = 1)

// Alignof dùng để tính padding khi reorder struct fields
```

### Offsetof — field offset in struct

```go
type Header struct {
    Magic   uint32
    Version uint16
    Flags   uint16
}

unsafe.Offsetof(Header{}.Magic)   // 0
unsafe.Offsetof(Header{}.Version) // 4 (có 2 bytes padding)
unsafe.Offsetof(Header{}.Flags)   // 6 (có 0 bytes padding)
```

## Common Production Patterns

### Pattern 1: Zero-copy string ↔ []byte conversion

```go
// ✅ Standard approach: allocate new []byte
s := "hello"
b := []byte(s) // allocation + copy

// ✅ Zero-copy approach: view same memory
b := unsafe.Slice((*byte)(unsafe.StringData(s)), len(s))
// b là view vào string's backing array — không copy data

// ⚠️ Rules:
// 1. Không bao giờ write vào b nếu s cần immutability guarantee
// 2. s phải được keep alive cho đến khi b không còn dùng
// 3. Chỉ dùng cho hot paths (JSON parsing, networking) cần perf
```

### Pattern 2: Memory layout inspection (field order optimization)

```go
// Kiểm tra struct size trước và sau khi reorder fields
type BadOrder struct {
    a bool   // 1 byte
    _ [7]byte // padding
    b int64  // 8 bytes
    // Total: 16 bytes

type GoodOrder struct {
    b int64  // 8 bytes
    a bool   // 1 byte
    _ [7]byte // padding
    // Total: 16 bytes — nhưng b được aligned tốt hơn
}

fmt.Println(unsafe.Sizeof(BadOrder{}))  // 16
fmt.Println(unsafe.Sizeof(GoodOrder{})) // 16 — same size, but GoodOrder aligns better

// Alignment matters more for nested arrays/structs
fmt.Println(unsafe.Alignof(BadOrder{}))  // 8
fmt.Println(unsafe.Alignof(GoodOrder{})) // 8
```

### Pattern 3: Binary protocol parsing (zero-copy)

```go
// Zero-copy parsing binary format (e.g., protocol buffer headers)
type PacketHeader struct {
    Size    uint32
    Version uint16
    Flags   uint16
}

func ParseHeader(data []byte) (*PacketHeader, error) {
    if len(data) < 8 {
        return nil, errors.New("header too short")
    }
    // ⚠️ Reinterpret bytes as PacketHeader without allocation
    return (*PacketHeader)(unsafe.SliceData(data)), nil
}

// ⚠️ Only safe khi:
// 1. data is aligned (slice data from network buffer)
// 2. No unaligned access on the platform
// 3. Data layout matches CPU endianness
```

### Pattern 4: etcd / zerolog-style field access

```go
// Một số high-performance library dùng unsafe.Pointer để:
// 1. Tránh allocation khi không cần thiết
// 2. Access fields bằng offset thay vì reflection

// Ví dụ đơn giản:
type Config struct {
    size int
    data []byte
}

// Lấy field "size" mà không dùng reflection
func ConfigSize(c *Config) int {
    // size là field đầu tiên nên offset = 0
    // ⚠️ Chỉ work vì biết layout chính xác của Config
    return *(*int)(unsafe.Pointer(c))
}
```

## When to Use unsafe

| Use case | safe alternative | unsafe appropriate? |
|---------|-----------------|-------------------|
| Zero-copy string ↔ []byte | `[]byte(s)` / `string(b)` | ✅ Yes — high perf hot paths |
| Binary protocol parsing | `encoding/binary` | ⚠️ Only if profiling shows it matters |
| JSON encoder internals | `encoding/json` | ❌ No — standard lib is good enough |
| Custom allocators | `sync.Pool`, `runtime` | ⚠️ Rare, only for specific use cases |
| Accessing private API internals | — | ❌ No — breaks on every release |
| Type assertion without interface | Type assertion | ❌ No |
| Faster type comparison | `reflect.DeepEqual` | ❌ No |

## Safety Rules (strict)

```go
// ❌ NEVER convert uintptr → unsafe.Pointer → *T in separate steps
ptr := (*int)(unsafe.Pointer(uintptr(0x1234))) // ❌ Unsafest pattern

// ⚠️ uintptr is just an integer — GC can move data while uintptr is held
var p *int = new(int)
ptr := uintptr(unsafe.Pointer(p)) // ⚠️ DANGER: uintptr breaks the pointer chain
// GC doesn't know ptr exists — p's memory could be collected/moved here

// ✅ Safe: keep pointer and uintptr in same expression
ptr := unsafe.Pointer(p)

// ❌ NEVER store unsafe.Pointer in uintptr variable
up := unsafe.Pointer(p)
up2 := unsafe.Pointer(uintptr(up) + 8) // ⚠️ Risky

// ✅ Safe pattern: atomic pointer operations
var ptr atomic.UnsafePointer
ptr.Store(unsafe.Pointer(p))
```

### The golden rule

```go
// unsafe.Pointer chỉ an toàn khi:
// 1. Pointer được giữ alive bởi Go's GC (không lưu trong uintptr biến)
//
// 2. Conversions tuân thủ 3 rules:
//    a) *T ↔ unsafe.Pointer ↔ *T (any type ↔ any type)
//    b) uintptr ↔ unsafe.Pointer (trong cùng expression)
//    c) unsafe.Pointer → *T (convert về typed pointer)
//
// 3. Alignment correct cho platform (8-byte values phải aligned)

```
