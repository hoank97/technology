# 02.2 Memory Model & Race Detector

## Overview

Go Memory Model định nghĩa **khi nào một goroutine đọc được giá trị ghi bởi goroutine khác** — đảm bảo thứ tự và tính nhất quán.

> Formalized kể từ Go 1.17, nhưng các nguyên tắc áp dụng từ Go 1.0.

---

## Happens-Before

### Định nghĩa

**Happens-Before** là quan hệ thứ tự giữa 2 operation trong cùng chương trình:

- Nếu **A happens-before B** → A chắc chắn hoàn thành trước khi B bắt đầu
- B đảm bảo thấy kết quả của A

### Trong một Goroutine

```go
a := 1          // 1
b := 2          // 2
fmt.Println(a + b) // 3 — thấy a=1, b=2
```

Trong cùng goroutine: **program order** (thứ tự code) chính là happens-before.

### Cross Goroutines

> **CRITICAL**: Không có gì đảm bảo một goroutine thấy giá trị ghi bởi goroutine khác — TRỪ khi có synchronization point tạo ra happens-before.

```go
// ❌ UNSAFE — data race (undefined behavior)
var a int
go func() { a = 1 }()
fmt.Println(a) // Có thể in 0 hoặc 1 hoặc crash
```

```go
// ✅ SAFE — channel tạo happens-before
var a int
var ch = make(chan int)
go func() {
    a = 1
    ch <- 1 // (1) ghi xong, (2) gửi signal
}
<-ch          // (3) nhận signal
fmt.Println(a) // Chắc chắn in 1
```

---

## Synchronizes-With

Các operation tạo ra **synchronization point**, tạo happens-before relationship:

| Operation | Details |
|-----------|---------|
| **Channel send** → **Channel receive** | Sender happens-before receiver nhận được giá trị |
| **Channel close** → **Receive from closed** | Close happens-before receive trả về zero + ok=false |
| **Mutex Unlock** → **Mutex Lock** | Unlock happens-before lock tiếp theo acquire |
| **RWMutex RUnlock** → **RLock** | Tương tự cho read lock |
| **Once call** → **subsequent calls** | Once hoàn thành trước tất cả call tiếp theo |
| **sync.Pool** | `Put` → `Get` không guaranteed (Pool có thể bị clear) |
| **`runtime.Gosched()`** | Voluntary yield — yields execution slot |

### Ví dụ: Channel as Memory Barrier

```go
var msg string
done := make(chan bool)

go func() {
    msg = "hello"   // (1) ghi trước
    done <- true    // (2) signal
}()

<-done
fmt.Println(msg)    // (3) đọc — chắc chắn thấy "hello"
```

### Ví dụ: Mutex as Synchronizer

```go
var counter int
var mu sync.Mutex

go func() {
    mu.Lock()
    counter++   // (1) ghi trong critical section
    mu.Unlock() // (2) unlock
}()

mu.Lock()
print(counter) // (3) lock — thấy giá trị ghi bởi goroutine kia
mu.Unlock()
```

## Data Race

### Định nghĩa

**Data race** xảy ra khi:

1. 2+ goroutines cùng access cùng memory
2. Ít nhất 1 goroutine **write**
3. Không có happens-before relationship giữa 2 access

### Kết quả

> Data race = **undefined behavior** — chương trình có thể crash, in giá trị sai, hoặc "hoạt động" một cách ngẫu nhiên.

### Ví dụ

```go
// ❌ DATA RACE
var shared int

go func() {
    shared = 1 // write
}()

go func() {
    fmt.Println(shared) // read — có thể thấy 0
}()
```

## Race Detector

### Cách dùng

```bash
go run -race main.go        # chạy với race detector
go test -race ./...          # test với race detector
go build -race ./cmd/app     # build binary với race detector
```

### Output mẫu

```
WARNING: DATA RACE
Write at 0x00c000010090 by goroutine 7:
  main.func·001()
      /path/main.go:8 +0x3c

Previous read at 0x00c000010090 by goroutine 6:
  main.func·002()
      /path/main.go:12 +0x30

Goroutine 7 (running):
  ...
Goroutine 6 (running):
  ...
```

### Cách đọc output

- **Write** = goroutine gây race (ghi sau cùng theo wall clock)
- **Previous read** = goroutine đọc trước, không thấy giá trị mới
- Thể hiện **happens-before violation** — không có synchronization giữa 2 goroutines

### Programmatic Check

```go
import "runtime"

if race.Enabled {
    // race-specific code
}

// Hoặc kiểm tra binary có race detector
// go build -race → binary ~3x lớn hơn, chậm hơn
```

### Khi nào dùng

| Situation | Recommendation |
|-----------|---------------|
| Development | ✅ Luôn chạy `-race` |
| Unit tests | ✅ `go test -race` |
| CI/CD | ✅ Production-critical paths |
| Production | ❌ Quá chậm — chỉ dùng cho debugging |

## Safe Patterns — Guaranteed by Memory Model

### Initialized Struct

```go
var u struct {
    mu  sync.Mutex
    val int
}

// ✅ SAFE — zero value struct có mu unlocked, val=0
u.mu.Lock()
u.val = 42
u.mu.Unlock()
```

### Once

```go
var once sync.Once
var data string

func setup() { data = "initialized" }

func main() {
    go setup()
    once.Do(setup)
    fmt.Println(data) // ✅ SAFE — đảm bảo "initialized"
}
```

### Single Writer

```go
// ✅ SAFE — 1 writer, N readers, reader sau writer với sync
var msg string
var mu sync.RWMutex

func write(v string) {
    mu.Lock()
    msg = v
    mu.Unlock()
}

func read() string {
    mu.RLock()
    defer mu.RUnlock()
    return msg
}
```

## Common Anti-patterns

### ❌ Lazy Initialization (Race)

```go
var singleton *Service

func GetService() *Service {
    if singleton == nil {        // read 1
        mu.Lock()
        if singleton == nil {   // read 2 (trong lock)
            singleton = &Service{} // write
        }
        mu.Unlock()
    }
    return singleton
}
```

→ Double-checked locking cần mutex bao quanh **toàn bộ** check hoặc dùng `sync.Once`.

### ✅ Safe Singleton

```go
var once sync.Once
var singleton *Service

func GetService() *Service {
    once.Do(func() {
        singleton = &Service{}
    })
    return singleton
}
```

## Kết hợp: Memory Model + Race Detector

| Tool | Purpose |
|------|---------|
| **Memory Model** | Hiểu quy tắc — khi nào đọc thấy giá trị write |
| **Race Detector** | Phát hiện vi phạm — tự động find data races |
| **Mutual依赖** | Memory Model cho biết **cách fix**, Race Detector cho biết **ở đâu fix** |

> **Best practice**: Luôn hiểu Memory Model để viết code đúng từ đầu; dùng Race Detector để verify.

## 📌 Tóm tắt

```
Memory Model & Race Detector
│
├── Happens-Before
│   ├── Trong goroutine: program order
│   ├── Cross goroutines: CẦN synchronization point
│   └── Không có sync = undefined behavior
│
├── Synchronizes-With (tạo happens-before)
│   ├── Channel send → receive
│   ├── Mutex Unlock → Lock
│   ├── Once call → subsequent calls
│   └── RWMutex RUnlock → RLock
│
├── Data Race
│   ├── ≥2 goroutines access, ≥1 write, no sync
│   └── = undefined behavior
│
├── Race Detector
│   ├── go run -race / go test -race
│   ├── Find all race instances (not just first)
│   └── Warning: slows 3x, binary 3x larger
│
└── Safe Patterns
    ├── sync.Once cho singleton
    ├── Mutex/RWMutex cho shared state
    ├── Channel cho signaling
    └── Atomic cho single primitive
```
